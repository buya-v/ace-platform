# T101 — Database Connection Layer — Shared PostgreSQL Client Package

## Status: COMPLETE

## Summary

Created `src/shared/` Go module (`github.com/garudax-platform/shared`) with a `internal/db` package providing PostgreSQL connection pooling, health checks, transaction helpers, and a migration runner. Uses `github.com/jackc/pgx/v5` (pgxpool) as the sole external dependency.

## Deliverables

| File | Description |
|---|---|
| `src/shared/go.mod` | Go module with pgx/v5 dependency |
| `src/shared/internal/db/pool.go` | `Config` struct from env vars, `Pool` wrapper over `*pgxpool.Pool`, validation, `NewPool()`, `Close()`, `Raw()` |
| `src/shared/internal/db/health.go` | `HealthCheck()` returning `HealthStatus` with latency and pool stats |
| `src/shared/internal/db/tx.go` | `RunInTx(ctx, pool, fn)` with begin/commit/rollback + panic recovery |
| `src/shared/internal/db/migrate.go` | `Migrate(ctx, pool, fs.FS)` — reads `.sql` files, tracks applied migrations in `schema_migrations` table |
| `src/shared/internal/db/pool_test.go` | 27 unit tests covering config, validation, DSN, env parsing, migration file collection |

## Design Decisions

1. **Single external dependency**: Only `pgx/v5` is imported. Follows the existing zero-dep-where-possible pattern.
2. **Config from environment**: `DefaultConfig()` reads `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_SSL_MODE`, `DB_MAX_CONNS`, `DB_MIN_CONNS` with sensible defaults for local dev.
3. **Validation before connect**: `validateConfig()` checks all fields before attempting a connection, providing clear error messages.
4. **fs.FS for migrations**: The migration runner accepts an `fs.FS`, allowing services to use `embed.FS` for compiled-in migrations or `os.DirFS` for external files.
5. **schema_migrations table**: Simple tracking table with filename + applied_at timestamp. Each migration runs in its own transaction.
6. **Panic recovery in RunInTx**: If the TxFunc panics, the transaction is rolled back before the panic is re-raised.
7. **Exported test helpers**: `CollectMigrationFiles()` and `ApplyMigrationTx()` are exported for downstream services to use in their own tests.

## Suggested Follow-ups

- **T102+ (service integration)**: Each service should add `require github.com/garudax-platform/shared v0.0.0` to its `go.mod` with a `replace` directive pointing to `../shared`. Then import `github.com/garudax-platform/shared/internal/db` and replace in-memory stores with PostgreSQL-backed implementations.
- **Integration tests**: When a PostgreSQL instance is available (e.g., Docker Compose), add integration tests that exercise `NewPool`, `HealthCheck`, `RunInTx`, and `Migrate` against a real database. Target 80%+ coverage.
- **Connection string auth**: For production, consider supporting `DB_URL` as a single connection string override (e.g., from a secrets manager) in addition to the individual env vars.
