## Decisions Made

1. **Handler ordering matters**: New specific handlers (buy/sell regex, bust/disable/suspend patterns) are placed BEFORE the catch-all keyword checks (`containsAny(lower, "margin")`, `containsAny(lower, "participant")`) to prevent early match interception.

2. **Resolve alert before alerts list**: The `resolve alert <id>` regex check is placed before the `containsAny(lower, "alert")` list check, so "resolve alert 123" routes to the write endpoint, not the list endpoint.

3. **Reports endpoints may not exist**: `GET /api/v1/reports/market-summary` and `GET /api/v1/reports/large-traders` are not in routes.go. The handlers gracefully fall back with a helpful message if the endpoint returns non-2xx.

4. **Cancel order vs mass cancel**: `cancel order <id>` (specific) is checked before `mass cancel` / `cancel all` to ensure specificity.

5. **Instrument alias loop preserved**: The per-alias loop for candles/trades/last-trade is placed before the existing price/book alias loop to avoid redundancy.
