# T119 — Matching Engine Auction Phases

## Status: DONE

## Summary

Implemented proper market phases and auction uncrossing in the matching engine. Three new files created, one existing file modified, and the engine wired up with PhaseManager.

## Files Created

- `src/matching-engine/internal/orderbook/auction.go` — AuctionEngine with uncrossing algorithm
- `src/matching-engine/internal/orderbook/phases.go` — PhaseManager with lifecycle: PRE_OPEN -> OPENING_AUCTION -> CONTINUOUS -> CLOSING_AUCTION -> POST_CLOSE
- `src/matching-engine/internal/orderbook/auction_test.go` — 18 new tests covering auction and phase logic

## Files Modified

- `src/matching-engine/internal/orderbook/orderbook.go` — Added auction-phase order queueing (no matching during auction/pre-open), accept orders in BookStatePreOpen, reject non-limit orders during auction
- `src/matching-engine/internal/engine/engine.go` — Added PhaseManager to bookEntry, TransitionPhase() and GetPhase() methods

## Decisions Made

1. **Auction uncrossing is a pure function**: `RunAuction()` takes bid/ask levels and reference price, returns trades. No side effects on the order book — the caller (PhaseManager.TransitionTo) handles state updates.

2. **During auction/pre-open phases, only limit orders are accepted**: Market orders, FOK, and IOC are rejected since they don't make sense without immediate matching. Orders are added to the book but not matched.

3. **Phase transitions are validated**: Only the legal sequence PRE_OPEN -> OPENING_AUCTION -> CONTINUOUS -> CLOSING_AUCTION -> POST_CLOSE (and POST_CLOSE -> PRE_OPEN for next day) is allowed.

4. **Equilibrium price selection**: Maximizes matched volume. Tiebreaker: closest to reference price (last trade / previous close). If still tied: higher price wins. This matches standard exchange auction mechanics.

5. **Time priority preserved in auction fills**: Orders are sorted by sequence number before matching at equilibrium price.
