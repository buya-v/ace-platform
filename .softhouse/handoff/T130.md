## Decisions Made
1. Orders are submitted directly to the matching engine (port 8081) rather than through the gateway, matching the existing pattern in the original script. This bypasses auth middleware which simplifies order submission.
2. Warehouse operations go directly to the warehouse service (port 8088) since the gateway only proxies a subset of warehouse endpoints (receipts, deliveries, inventory) but not facilities or inspections.
3. The script uses `declare -A TRADE_COUNT` (associative array) to track trades per instrument from the matching engine's exec_type response field.

## Prerequisites / Blockers
- The matching engine must have all 6 instruments registered via the `INSTRUMENTS` environment variable. The current `docker-compose.yml` only registers `WHT-HRW-2026M07-UB` by default. To enable multi-instrument trading, set:
  ```
  INSTRUMENTS=WHT-HRW-2026M07-UB,CRN-YEL-2026M09-UB,SBN-NO2-2026M11-UB,BRL-MALT-2026M07-UB,CSH-RAW-2026M09-UB,LVS-CATTLE-2026M10-UB
  ```
  in the matching engine's environment (docker-compose.yml line 111). Note: `docker-compose.yml` is not committed to git.
