# T120 — Matching Engine Additional Order Types

## Status: COMPLETE

## Summary

Enhanced the matching engine with three new order type features:

### 1. Iceberg Orders (`iceberg.go`)
- Orders with `DisplayQty` and `TotalQty` fields; only `DisplayQty` is visible on the book
- When a display slice is fully filled, hidden reserve auto-replenishes the next slice
- If remaining hidden qty < display qty, the smaller amount is shown
- Replenished orders lose time priority (removed from front of queue, re-enqueued at back with new timestamp and sequence number)
- Validation: iceberg orders must be limit orders; display qty must be < total qty
- Exec reports correctly show PARTIAL_FILL during replenishment and FILL only when entire iceberg is exhausted

### 2. GTD Expiry (`expiry.go`)
- `ExpiryChecker` monitors GTD orders and cancels them when `ExpireAt` time passes
- `StartExpiryChecker(ctx, book, callback)` runs a background goroutine checking every second
- `ExpireOrders()` can also be called directly (for testing or manual sweeps)
- Expired orders produce EXPIRED execution reports and invoke the callback
- Injectable `nowFunc` for deterministic testing
- Correctly removes empty price levels after expiry

### 3. Stop Order Monitor (`stop_monitor.go`)
- `StopMonitor` holds stop orders in a separate map (not on the main book)
- `AddStopOrder()` validates and accepts stop-limit and stop-market orders with PENDING_NEW status
- `OnTrade(lastPrice)` checks all pending stops: buy stops trigger when price >= stop, sell stops when price <= stop
- Triggered stop-limit orders become limit orders; stop-market orders become market orders
- Designed to run AFTER match completes (no latency impact on hot path)
- `CancelStopOrder()` allows cancellation before trigger

## Decisions Made

1. **Iceberg replenishment timing**: Dequeue happens BEFORE qty update to avoid uint64 underflow in PriceLevel.TotalQty
2. **Exec type for iceberg fills**: Uses PARTIAL_FILL while hidden qty remains, FILL only when entire iceberg exhausted (FIX protocol compliant)
3. **Stop monitor is external to OrderBook**: StopMonitor is a separate struct that wraps an OrderBook, keeping the hot matching path clean
4. **GTD expiry uses injectable clock**: `nowFunc` field allows deterministic testing without time.Sleep
