## Decisions Made
1. Used Zod schemas for tool parameter validation (required by @modelcontextprotocol/sdk's `server.tool()`)
2. GatewayClient accepts both `access_token` and `AccessToken` JWT response fields (known casing inconsistency in auth service)
3. `.mcp.json` stores only GARUDAX_GATEWAY_URL — credentials (GARUDAX_ADMIN_EMAIL, GARUDAX_ADMIN_PASSWORD) should be set via environment to avoid committing secrets
4. Tools return raw JSON from gateway — no transformation layer — so Claude sees exactly what the API returns
