## Implementation Decisions

### ParseIntent priority order
All 14 rules are evaluated in strict order using compiled `regexp.MustCompile` vars (initialized once at package load). First match wins. This matches the task spec exactly. The keyword fallback (`keywordAction`) checks `settlement`/`settle` BEFORE `health`/`status` to prevent "settlement status" matching the health case.

### ResolveInstrumentAlias — two-pass matching
1. **Exact match** against `prefixAliases` map (O(1)) — handles all documented aliases.
2. **Prefix match** against `prefixGroups` ordered slice — handles "whe"→wheat, "cas"→cashmere, "bar"→barley, "cat"→cattle, etc.
3. **Full ID passthrough** — if input contains '-', it is uppercased and returned as-is (e.g., "WHT-HRW-2026M07-UB" or custom IDs).

Deterministic ordering in `prefixGroups` ensures "wh" → wheat (not wht, which would also prefix-match but shorter aliases come after longer ones in the list).

### ResolveDate
Handles: `today`, `yesterday` (case-insensitive), ISO `YYYY-MM-DD`, compact `YYYYMMDD`. Unknown strings pass through unchanged.

### ExtractNumber
Regex-based; extracts first numeric token including sign and decimal. Returns `(0, false)` on no match.
