## Summary

## Changes Made

### Part A — Watch Lists

**types.go** — appended `WatchList` struct with ID, Name, OwnerID, InstrumentIDs, ClientIDs, FirmIDs, CreatedAt, UpdatedAt fields (all JSON-tagged).

**store.go** — appended `WatchListStore` interface (Create, Get, ListByOwner, Update, Delete) and `InMemoryWatchListStore` implementation (thread-safe, map-backed).

**handlers_watchlist.go** (new file) — implements:
- `POST /watchlists` — create a watch list (requires name + owner_id)
- `GET /watchlists?owner_id=X` — list watch lists by owner
- `GET /watchlists/{id}` — get a single watch list
- `PUT /watchlists/{id}` — update name/instrument_ids/client_ids/firm_ids (partial update)
- `DELETE /watchlists/{id}` — delete (returns 204)

### Part B — Trade Capture Reports

**handlers_trade_capture.go** (new file) — implements:
- `GET /api/v1/securities/trade-capture-reports?firm_id=X&date=YYYY-MM-DD` — lists all trades where the buying or selling participant belongs to firm_id and trade_date matches date. Response includes TradeCaptureReport with firm_id, date, trades[], total_buy_qty, total_sell_qty, total_buy_value, total_sell_value, net_position.
- Uses `participantStore.List(FirmID=...)` to resolve firm-to-participant membership; nil-safe fallback when participantStore is not configured.

### Part C — IP Restrictions

**types.go** — appended `IPRestriction` struct with ParticipantID, AllowedIPs []string, Enabled bool (all JSON-tagged).

**store.go** — appended `IPRestrictionStore` interface (Get, Set, Delete) and `InMemoryIPRestrictionStore` implementation (upsert semantics on Set).

**handlers_ip_restriction.go** (new file) — implements:
- `GET /ip-restrictions/{participant_id}` — retrieve allow-list
- `PUT /ip-restrictions/{participant_id}` — upsert allow-list and enabled flag
- `DELETE /ip-restrictions/{participant_id}` — remove restriction record (returns 204)

### Part D — Password Policy

**types.go** — appended `PasswordPolicy` struct with TenantID, MinLength, RequireUppercase, RequireLowercase, RequireDigit, RequireSpecial, MaxAgeDays (all JSON-tagged).

**store.go** — appended `PasswordPolicyStore` interface (Get, Set) and `InMemoryPasswordPolicyStore` implementation.

**handlers_password_policy.go** (new file) — implements:
- `GET /password-policy` — returns policy for current tenant (defaults to 8-char, upper+lower+digit required, 90-day max age if not configured)
- `PUT /password-policy` — upsert policy for current tenant
- Tenant resolved from middleware context or X-GarudaX-Tenant header.

## Summary

Built the complete `fix-gateway` Go service implementing the FIX protocol gateway per `docs/fix-gateway-spec.md`. Zero external dependencies, follows the established zero-dep Go module pattern.

## Deliverables

### Part A — Scaffold (`src/fix-gateway/`)
- `go.mod`: `github.com/garudax-platform/fix-gateway`, go 1.25.0, zero deps
- `cmd/fix-gateway/main.go`: HTTP admin on :8091, health on :9091, env-configured
- `Dockerfile`: multi-stage `golang:1.25-alpine` build, `alpine:3.19` runtime, EXPOSE 8091 9091 9878

### Part B — FIX Parser (`internal/fix/parser.go`)
- `FIXMessage` struct with `Fields map[int]string`
- `ParseMessage(data []byte)`: splits by SOH (0x01) or pipe (|), parses tag=value pairs
- `BuildMessage(msgType, senderCompID, targetCompID, seqNum, fields)`: constructs wire-format message with BeginString, BodyLength, CheckSum
- `CalculateChecksum(data []byte)`: sum bytes mod 256, formatted %03d
- `GetTag`, `GetIntTag`, `GetFloatTag` helpers
- 60+ tag constants covering session, order, execution, and market data tags
- Message type constants for all FIX 4.4 session and application messages

### Part C — FIX Mapper (`internal/fix/mapper.go`)
- `MapNewOrderSingle(msg)` → `InternalOrder` struct with full field mapping and validation
- `MapExecutionReport(orderID, execID, execType, ordStatus, side, qty, price, leavesQty, cumQty)` → `*FIXMessage`
- Side mapping: 1=BUY, 2=SELL, 5=SHORT_SELL, 6=SHORT_SELL (exempt)
- OrdType mapping: 1=MARKET, 2=LIMIT, 3=STOP, 4=STOP_LIMIT
- TimeInForce mapping: 0=DAY, 1=GTC, 3=IOC, 4=FOK, 6=GTD

### Part D — Session Manager (`internal/session/manager.go`)
- `SessionState` enum: DISCONNECTED, LOGON_SENT, ACTIVE, LOGOUT_SENT
- `Session` struct: SenderCompID, TargetCompID, TenantID, State, InSeqNum/OutSeqNum uint64, HeartbeatInterval, LastRecvTime
- `SessionManager`: thread-safe with sync.RWMutex
- Methods: CreateSession, GetSession, ProcessLogon, ProcessLogout, IncrementOutSeq, ValidateInSeq, ListSessions, SessionCount

### Part E — Broker Store (`internal/broker/`)
- `FIXBroker` struct: ID, CompID, TenantID, Name, Status, Config map, CreatedAt, UpdatedAt
- `BrokerStore` interface: Create, GetByID, GetByCompID, Update, List
- `InMemoryStore`: seeded with MSE-BROKER-001 (CompID MSE001, tenant mse-equities, status ACTIVE)
- HTTP handlers: POST/GET /api/v1/fix/brokers, GET/PUT /api/v1/fix/brokers/{id}, GET /api/v1/fix/sessions

### Part F — Server (`internal/server/server.go`)
- Admin HTTP server on :8091 with broker/session management routes
- Health HTTP server on :9091 with /healthz, /readyz, /metrics
- Graceful shutdown with 10s timeout

### Part G — Docker + Gateway Integration
- `docker-compose.yml`: fix-gateway service entry with ports 9878/8091/9091, depends_on postgres+kafka+gateway
- Gateway: `FixGatewayAddr` config field, reverse proxy routes for `/api/v1/fix/brokers` and `/api/v1/fix/sessions`
- Gateway env var: `FIX_GATEWAY_ADDR: fix-gateway:8091`
