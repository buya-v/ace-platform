# T1 Handoff ŌĆö Securities Matching Engine

## Summary

Implemented the securities matching engine in 4 parts: types, store interfaces, matching engine, and server wiring.

## Changes

### Part A ŌĆö Types (`src/securities-service/internal/types/types.go`)
- Added `TradeStatus` type with 4 constants: `TRADE_PENDING`, `TRADE_CONFIRMED`, `TRADE_SETTLED`, `TRADE_FAILED`
- Added `SecurityTrade` struct with ID, BuyOrderID, SellOrderID, InstrumentID, Price, Quantity, TradeDate, SettlementDate, Status, CreatedAt ŌĆö all with `json:"snake_case"` tags
- Added `Position` struct with ID, ParticipantID, InstrumentID, Quantity, AvgCost, MarketValue, UnrealizedPnl, UpdatedAt ŌĆö all with json tags

### Part B ŌĆö Store Interfaces (`src/securities-service/internal/store/store.go`, `pg_store.go`)
- Added `TradeStore` interface: Create, Get, ListByInstrument
- Added `PositionStore` interface: GetOrCreate, Update, List
- Added `InMemoryTradeStore` with sync.RWMutex + map[string]*SecurityTrade
- Added `InMemoryPositionStore` with sync.RWMutex + map[string]*Position (key: "participantID:instrumentID")
- Added `Update(*SecurityOrder) error` to `OrderStore` interface
- Implemented `Update` on both `InMemoryOrderStore` and `PgOrderStore`

### Part C ŌĆö Matching Engine (`src/securities-service/internal/engine/engine.go`)
- `MatchingEngine` struct with instrumentStore, orderStore, tradeStore, positionStore fields
- `NewMatchingEngine` constructor
- `MatchOrder(order)` implementing full price-time priority matching:
  - Validates instrument ACTIVE status
  - Lists PENDING + PARTIALLY_FILLED orders on opposite side
  - Sorts by price-time priority (ascending price for sells, descending for buys, then CreatedAt)
  - Price crossing: MARKET always crosses; LIMIT checks incoming >= resting (buy) or <= (sell)
  - Trades at resting order's price, quantity = min(remaining, resting)
  - Creates SecurityTrade with UUID, TradeDate=today, SettlementDate=T+2, Status=TRADE_PENDING
  - Updates both orders: FILLED if fully filled, PARTIALLY_FILLED if partial
  - Updates positions: buyer +qty with weighted average cost, seller -qty
  - Continues until incoming fully filled or no more crosses

### Part D ŌĆö Server Wiring
- Added `engine *engine.MatchingEngine` field to `Server` struct
- Updated `New()` constructor signature to accept `*engine.MatchingEngine`
- Modified `handleSubmitOrder`: after storing, calls `engine.MatchOrder()`, returns `{"order": {...}, "trades": [...]}`
- Updated `main.go`: creates `InMemoryTradeStore`, `InMemoryPositionStore`, `MatchingEngine`, passes to `Server`
- Updated all existing tests to use new `New()` signature and parse `{"order":..., "trades":...}` response

## Decisions Made
- Matching engine errors are non-fatal in the handler ŌĆö if matching fails, the order is still stored as PENDING and returned with empty trades array
- `nil` engine is supported (engine field check in handler) for backward compatibility in tests
- Position key uses "participantID:instrumentID" composite string for the in-memory store
- AvgCost for buyer uses weighted average: `((oldQty * oldAvgCost) + (tradeQty * tradePrice)) / newQty`
- Seller's AvgCost is preserved (not recalculated on sell)
