## What Was Done

### Part A тАФ types.go: new domain types
- `CorporateActionType` string enum: CA_DIVIDEND, CA_STOCK_SPLIT, CA_RIGHTS_ISSUE, CA_MERGER
- `CorporateActionStatus` enum: ANNOUNCED, PROCESSING, COMPLETED, CANCELLED
- `CorporateAction` struct with full json tags including `Details map[string]interface{}`
- `EntitlementStatus` enum: PENDING, PAID
- `Entitlement` struct with json tags
- `FRCReport` struct with json tags

### Part B тАФ store.go: new stores
- `CorporateActionFilters` struct (InstrumentID, ActionType)
- `CorporateActionStore` interface: Create, Get, List(filters), UpdateStatus
- `EntitlementStore` interface: Create, ListByAction, ListByParticipant
- `InMemoryCorporateActionStore` тАФ thread-safe with sync.RWMutex, follows existing store patterns exactly
- `InMemoryEntitlementStore` тАФ same pattern
- Modified `InMemoryPositionStore.List` to return all positions when `participantID == ""` (needed for CA processing to find all holders of an instrument without knowing participant IDs up front тАФ consistent with the zero-value filter convention used by InstrumentStore.List)

### Part C тАФ handlers_corporate_actions.go (new file)
- GET/POST `/api/v1/securities/corporate-actions`
- GET `/api/v1/securities/corporate-actions/{id}`
- POST `/api/v1/securities/corporate-actions/{id}/process`
- DIVIDEND: fetches all positions for instrument, creates Entitlement per holder (qty * dividend_amount from Details)
- STOCK_SPLIT: adjusts all positions for instrument by split_ratio from Details
- RIGHTS_ISSUE / MERGER: marks COMPLETED with "manual processing required" note
- Status transitions: ANNOUNCED тЖТ PROCESSING тЖТ COMPLETED (with rollback to ANNOUNCED on error)

### Part D тАФ handlers_reports.go (new file)
- GET `/api/v1/securities/reports/frc?type=...&date=...`
- DAILY_SUMMARY: iterates all instruments, collects trades per instrument, filters by date prefix, sums count/volume/value
- LARGE_TRADER: all positions with Quantity > 1000 shares
- SUSPICIOUS_ACTIVITY: placeholder returning empty list with note

### Part E тАФ server.go and main.go wiring
- Added `tradeStore`, `positionStore`, `corporateActionStore`, `entitlementStore` to `Server` struct
- `New()` extended from 7-arg to 11-arg signature
- `registerRoutes` adds corporate-actions and FRC report routes
- `main.go` creates new stores and passes them to `server.New`
- Updated all 4 call sites in `handlers_instrument_test.go` to match new signature
