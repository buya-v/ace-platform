### Part C — IOC/FOK constants (`types.go`)

The existing `TimeInForce` type already declares `TimeInForceIOC = "IOC"` and
`TimeInForceFOK = "FOK"`. Added typed alias constants so matching-engine code
can use the `TIF_` prefix convention without duplicating string values:

```go
const (
    TIF_IOC = TimeInForceIOC
    TIF_FOK = TimeInForceFOK
)
```

### Store additions (`store.go`)

| Interface | In-Memory impl | Key behaviour |

| `TradeCorrectionStore` | `InMemoryTradeCorrectionStore` | slice-backed; `Create` appends; `ListByTrade` filters by TradeID |

## Key Decisions

- `TIF_IOC`/`TIF_FOK` are typed as `TimeInForce` (not `string`) via alias — usable anywhere `TimeInForce` is expected.
- `TickTableStore.Get` returns `ErrNotFound` (consistent with all other stores) rather than `nil, nil`.
- `ThrottleStore.CheckAndIncrement` returns `(false, error)` on throttle — the error carries the rate-limit message, letting callers log it without needing a separate sentinel value.
- Deep copy of `[]TickTier` on both `Get` and `Set` prevents slice aliasing bugs.
- `InMemoryTradeCorrectionStore` uses a slice (not a map) because corrections are append-only and always queried by `TradeID` (no direct ID lookup needed).

- **Persistence**: all three stores are in-memory; a future task should add PostgreSQL-backed implementations behind the same interfaces.
