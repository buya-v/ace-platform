## Summary

Implemented FIX protocol TCP server (Part A), heartbeat monitoring with inactivity detection (Part B), order router to securities service (Part C), and wired everything into main.go with graceful shutdown (Part D).

### Part A ‚Äî FIX TCP Server (`internal/server/tcp.go`)

`FIXServer` struct with: listener, sessionManager, brokerStore, orderRouter, connections map, mu sync.RWMutex.

- `Start(addr)` ‚Äî `net.Listen("tcp", addr)`, accept loop in goroutine
- `handleConnection(conn)` ‚Äî read loop, parse FIX messages via `fix.ParseMessage`, dispatch by MsgType:
  - **Logon (A)**: validates CompID via `brokerStore.GetByCompID`, checks broker status is ACTIVE, creates session via sessionManager, sends Logon ack with swapped sender/target CompIDs
  - **NewOrderSingle (D)**: maps via `fix.MapNewOrderSingle`, routes via `orderRouter.SubmitOrder` with tenant header, sends ExecutionReport (New or Rejected)
  - **OrderCancelRequest (F)**: routes cancel via `orderRouter.CancelOrder`, sends ExecutionReport (Canceled) or OrderCancelReject
  - **Heartbeat (0)**: updates session LastRecvTime
  - **TestRequest (1)**: responds with Heartbeat containing TestReqID
  - **Logout (5)**: closes session, sends Logout ack, closes connection
- `Stop()` ‚Äî closes listener, signals all heartbeat goroutines via `stopHB` channels, closes all connections

### Part B ‚Äî Heartbeat Monitor (in `tcp.go`)

`heartbeatLoop()` goroutine started after successful logon:
- Sends Heartbeat every `HeartbeatInterval` (default 30s) to the connection
- Tracks `LastRecvTime`; if no message in 2x interval (60s), sends TestRequest
- If still no response after another interval (30s), disconnects the connection and logs out the session

### Part C ‚Äî Order Router (`internal/router/order_router.go`)

- `OrderRouter` struct with `securitiesServiceURL` string, `httpClient *http.Client` (10s timeout)
- `NewOrderRouter(url)` constructor
- `SubmitOrder(order, tenantID)` ‚Äî POST to `{url}/api/v1/securities/orders` with `X-GarudaX-Tenant` header, returns response body map
- `CancelOrder(orderID, tenantID)` ‚Äî DELETE to `{url}/api/v1/securities/orders/{orderID}` with tenant header

### Part D ‚Äî main.go Wiring

- Creates `OrderRouter` with `SECURITIES_SERVICE_URL` env (default `http://securities-service:8089`)
- Creates `FIXServer` with logger, sessionMgr, brokerStore, orderRouter, targetCompID
- Starts TCP on `FIX_LISTEN_ADDR` env (default `:9878`)
- Graceful shutdown: stops FIX TCP server first, then HTTP admin/health servers
