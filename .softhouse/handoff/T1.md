# T1 Handoff — Securities Admin UI Scaffold

## Summary

Added Securities module scaffolding to `src/admin-ui`: 7 new API functions, 3 sidebar entries, 3 route registrations, 3 placeholder page components, and 3 empty CSS modules. TypeScript compiles cleanly and all 404 existing tests pass.

## What Was Done

### Part A — API functions (`src/admin-ui/src/services/api.ts`)
Added 7 exported functions following the existing `apiFetch` + `URLSearchParams` pattern:

- `fetchSecuritiesInstruments(filters?, signal?)` — GET `/securities/instruments` with `asset_class` and `trading_status` query params
- `createSecuritiesInstrument(data)` — POST `/securities/instruments`
- `updateInstrumentStatus(id, status, reason?)` — PUT `/securities/instruments/:id/status`
- `fetchSecuritiesOrders(filters?, signal?)` — GET `/securities/orders` with `instrument_id` and `status` query params
- `submitSecuritiesOrder(data)` — POST `/securities/orders`
- `cancelSecuritiesOrder(id)` — POST `/securities/orders/:id/cancel`
- `fetchSecuritiesPositions(filters?, signal?)` — GET `/securities/positions` with `participant_id` query param

### Part B — Sidebar entries (`src/admin-ui/src/components/Sidebar.tsx`)
Appended 3 entries to the `operationsNav` array after the Tickets entry:

- `Securities` → `/dashboard/securities` (ChartBarIcon)
- `Sec. Orders` → `/dashboard/securities-orders` (BookIcon)
- `Sec. Positions` → `/dashboard/securities-positions` (DollarIcon)

### Part C — Routes (`src/admin-ui/src/App.tsx`)
Imported the 3 new page components and registered routes inside the admin-only `RoleGuard` block:

- `securities` → `<SecuritiesInstrumentsPage />`
- `securities-orders` → `<SecuritiesOrdersPage />`
- `securities-positions` → `<SecuritiesPositionsPage />`

### Part D — Placeholder pages and CSS modules

## Decisions Made

- Used `any` return types for securities API functions since no domain types exist yet; downstream tasks should define proper types and replace `any`.
- `updateInstrumentStatus` uses PUT (not PATCH) to match the status-replacement semantics.
- `cancelSecuritiesOrder` uses POST to `/cancel` sub-resource, consistent with the existing `haltTrading`/`resumeTrading` pattern.
- The 3 sidebar items are appended to `operationsNav` after Tickets — they render in the Operations collapsible section.
- Placeholder pages contain no logic or CSS imports; they are pure stubs for routing verification.
