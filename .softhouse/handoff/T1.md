# T1 Handoff — MCP Tools for Securities

## Summary

Created `src/admin-bot/mcp-server/src/tools/securities.ts` with 7 MCP tools covering
the full securities lifecycle: list/get/create instruments, submit/list/cancel orders,
and list positions. Updated `index.ts` to import and register the new tools. TypeScript
compiles with zero errors (`npx tsc --noEmit` clean pass in both the worktree and the
main repo).

## What Was Done

### New file: `src/admin-bot/mcp-server/src/tools/securities.ts`

Exports `registerSecuritiesTools(server: McpServer, client: GatewayClient): void`
with 7 tools registered via `server.tool()`:

| Tool | Method + Path | Key params |
|---|---|---|
| `list_securities_instruments` | GET `/api/v1/securities/instruments` | `asset_class?`, `trading_status?` |
| `get_securities_instrument` | GET `/api/v1/securities/instruments/{id}` | `instrument_id` |
| `create_securities_instrument` | POST `/api/v1/securities/instruments` | `ticker`, `name`, `asset_class`, `lot_size`, `tick_size`, `currency` |
| `submit_securities_order` | POST `/api/v1/securities/orders` | `instrument_id`, `side`, `order_type`, `quantity`, `price?` |
| `list_securities_positions` | GET `/api/v1/securities/positions` | `participant_id?` |
| `list_securities_orders` | GET `/api/v1/securities/orders` | `instrument_id?`, `status?` |
| `cancel_securities_order` | DELETE `/api/v1/securities/orders/{id}` | `order_id` |

All responses are formatted as human-readable aligned text tables (not raw JSON).
List tools handle both bare arrays and `{ instruments: [...] }` / `{ orders: [...] }` /
`{ positions: [...] }` envelope shapes to be resilient to backend response format
variations.

### Modified: `src/admin-bot/mcp-server/src/index.ts`

Added:
```typescript
import { registerSecuritiesTools } from "./tools/securities.js";
// ...
registerSecuritiesTools(server, client);
```

Placed after `registerDailyAdminTools` in the registration block.

## Decisions Made

- **Dual-shape response handling**: list tools accept both bare array and enveloped
  `{ instruments/orders/positions: [...] }` shapes since the backend spec is not yet
  finalized for the securities module.
- **MARKET order price**: `price` is optional on `submit_securities_order`; omitting
  it is valid for MARKET orders and the tool displays "MARKET" in output.
- **`asset_class` enum on create**: constrained to `EQUITY | BOND | ETF` matching the
  task spec; list filter uses the same enum for safety.
- **DELETE for cancel**: used `DELETE /api/v1/securities/orders/{id}` as specified;
  this is consistent with REST conventions for order cancellation.
- **Table formatting**: columns are padded with `padEnd`/`padStart` for alignment;
  a `─` divider line separates header from data rows.
