## Summary

Rewired the GarudaX trading web-ui from commodity exchange (`ace-commodities`) endpoints to the securities-service endpoints (`/api/v1/securities/*`). All API calls now target the securities-service, the `X-GarudaX-Tenant: mse-equities` header is injected into every authenticated request, and the Instrument type has been updated from commodity-style fields to securities-style fields.

## Changes Made

### `src/web-ui/src/services/api.ts`
- Added `'X-GarudaX-Tenant': 'mse-equities'` header to `apiRequest()`.

### `src/web-ui/src/types/instrument.ts`
- Replaced `Instrument` interface: removed `instrumentId`, `symbol`, `commodityName`, `deliveryMonth`, `deliveryLocation`, `status`. Added `id`, `isin`, `ticker`, `name`, `asset_class`, `security_type`, `exchange_code`, `lot_size` (number), `tick_size` (number), `currency`, `listing_date`, `trading_status`, `outstanding_shares`, `created_at`, `updated_at`.
- Simplified `InstrumentDetail` to extend `Instrument` without additional commodity-specific fields.

### `src/web-ui/src/components/InstrumentSelector.tsx`
- Endpoint: `/instruments?status=active` -> `/securities/instruments`
- Response: `{ instruments }` -> `{ data }`
- Removed ticker polling to `/market-data/ticker`
- Fields: `instrumentId` -> `id`, `symbol` -> `ticker`, `commodityName` -> `name`

### `src/web-ui/src/pages/Trading.tsx`
- Instrument fetch: `/securities/instruments`, parse `data` array
- `selectedInstrument?.instrumentId` -> `selectedInstrument?.id`
- Trade history: `/trades/history` -> `/securities/trades`, parse `{ data }`
- Close position: `/orders` -> `/securities/orders`
- Added orders list polling from `GET /securities/orders` with new "Orders" tab
- Commodity-specific polling kept with 15s interval and silent errors

### `src/web-ui/src/hooks/useOrders.ts`
- Submit endpoint: `/orders` -> `/securities/orders`

### `src/web-ui/src/components/InstrumentInfo.tsx`
- Endpoint: `/instruments/{id}` -> `/securities/instruments/{id}`
- Replaced commodity fields with securities fields (ISIN, Asset Class, Security Type, etc.)

### `src/web-ui/src/components/Chart.tsx`
- Added "No chart data available" message for empty candle data

### `src/web-ui/src/components/TradeHistory.tsx`
- Filter dropdown: `inst.instrumentId` -> `inst.id`, `inst.symbol` -> `inst.ticker`

### `src/web-ui/src/__tests__/tradingReducer.test.ts`
- Updated test instrument to match new Instrument interface

## Not Changed (Intentional)
- `MarketContext.tsx` -- uses Instrument type generically, works with updated type
- `types/trade.ts` -- Position.instrumentId, TradeRecord.instrumentId are trade-internal fields
- `types/order.ts` -- already uses snake_case `instrument_id`
- `Positions.tsx` -- uses Position.instrumentId (trade type)
