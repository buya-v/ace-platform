# T1: Dynamic WebSocket URL Configuration

## Summary
Successfully implemented dynamic WebSocket URL generation in `src/admin-ui/src/config.ts` to support both HTTP and HTTPS environments. The hardcoded `ws://localhost:8080/api/v1/ws` has been replaced with a runtime-computed URL that detects the current protocol and host.

## Changes Made

### File: `src/admin-ui/src/config.ts`
- **Previous implementation**: Static `WS_BASE_URL` with hardcoded `ws://localhost:8080/api/v1/ws`
- **New implementation**:
  - Created `getDefaultWsUrl()` helper function that:
    - Detects protocol from `window.location.protocol`
    - Selects 'wss:' for HTTPS, 'ws:' for HTTP
    - Uses `window.location.host` for the hostname and port
    - Returns properly formatted URL: `${protocol}//${window.location.host}/api/v1/ws`
  - Moved defaults object into `getConfig()` function body so URL is computed at runtime (not module load time)
  - This ensures tests can mock `window.location` before URL computation

### File: `src/admin-ui/src/__tests__/config.test.ts`
- Updated test suite to comprehensively test dynamic URL generation
- Added 6 test cases:
  1. **returns defaults when no runtime config** — validates all default values including dynamic WS URL
  2. **generates wss URL for https protocol** — verifies HTTPS detection works
  3. **generates ws URL for http protocol** — verifies HTTP detection works
  4. **merges runtime config over defaults** — validates `__GARUDAX_CONFIG__` override behavior
  5. **allows runtime config to override WS_BASE_URL** — confirms WS URL can be overridden
  6. **does not contain undefined or NaN values** — UAT compliance check for rendered output

All 6 tests pass.

## Technical Details

### Protocol Detection Logic
```typescript
function getDefaultWsUrl(): string {
  const protocol = window.location.protocol === 'https:' ? 'wss:' : 'ws:';
  return `${protocol}//${window.location.host}/api/v1/ws`;
}
```

### Runtime Config Override
The `__GARUDAX_CONFIG__` override still works perfectly because:
1. Defaults are computed fresh on each `getConfig()` call
2. Runtime config spread-merged over defaults: `{ ...defaults, ...runtime }`
3. If `WS_BASE_URL` is in runtime config, it takes precedence

### URL Examples
- HTTP environment (localhost:3000): `ws://localhost:3000/api/v1/ws`
- HTTPS environment (admin.garudax.mn): `wss://admin.garudax.mn/api/v1/ws`
- With port: `wss://admin.garudax.mn:8443/api/v1/ws` (if port in location.host)

## Implementation Notes

### Why Move Defaults Into getConfig()?
Moving the `defaults` object from module scope into the function body ensures:
1. URL is computed at call time, not module initialization time
2. Tests can mock `window.location` before calling `getConfig()`
3. Each call gets a fresh default based on current window.location

This pattern is safe because:
- No expensive computation (string concatenation only)
- No side effects
- Trivial performance impact
