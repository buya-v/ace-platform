## Decisions Made

1. **Port 8089 / 9089** — next free slot following the existing convention (8088/9088 would overlap with warehouse health pattern; 8089/9089 are clean).

2. **Auth middleware**: Securities routes are all protected (authenticated). No public prefix exemption was added — securities trading data requires authentication per standard exchange security practice.

3. **PATCH + PUT both registered** for `instruments/{id}`: The securities-service `handleInstrument` accepts both PATCH (partial update) and the task spec mentioned PUT. Both are forwarded to the same `UpdateInstrument` RPC method so the backend decides semantics.

4. **Rate limit group `"securities"`** added to the gateway's `groupFn` switch for future rate-limit tuning on securities endpoints.

5. **No `go.mod` changes** were needed — gateway and securities-service are independent zero-dep Go modules. Config is the only cross-cutting change.
