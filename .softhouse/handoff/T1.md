### Part B — Config Loader + Endpoint

**New file:** `src/platform-service/internal/config/loader.go`
- `ConfigLoader` struct with `venuesDir string`
- `NewConfigLoader(venuesDir string) *ConfigLoader` — defaults to `"./venues"` if empty
- `LoadConfig(tenantID string) (map[string]interface{}, error)` — reads `venues/{tenantID}/config.json`, returns parsed JSON; returns descriptive error if file not found

**Modified:** `src/platform-service/internal/server/handlers.go`
- Added `handleTenantConfig` handler for `GET /platform/v1/tenants/{id}/config`
- Returns 404 with `CONFIG_NOT_FOUND` code if config file not found
- Returns 405 for non-GET methods

**Modified:** `src/platform-service/internal/server/server.go`
- Added `configLoader *config.ConfigLoader` field to `Server` struct
- Updated `New()` and `NewWithProvisioner()` to initialise a default ConfigLoader
- Added `NewWithConfig()` constructor for explicit ConfigLoader injection (used by main.go and tests)
- Updated `registerRoutes()` — the single `/platform/v1/tenants/` wildcard handler now checks for `/config` suffix and dispatches to `handleTenantConfig`
- Added `"strings"` import for the route dispatch logic

**Modified:** `src/platform-service/cmd/platform-service/main.go`
- Imports `config` package
- Creates `ConfigLoader` with `venuesDir` from `VENUES_DIR` env var (empty = `"./venues"` default)
- Uses `server.NewWithConfig()` to inject the loader

## Decisions Made

- Route dispatch uses a single `mux.HandleFunc("/platform/v1/tenants/", ...)` that checks for the `/config` suffix — avoids Go 1.22+ pattern syntax while remaining clear
- `NewWithConfig` constructor added rather than mutating existing constructors — preserves backward compatibility with existing callers
- `VENUES_DIR` env var allows deployment-time override of the venues directory (useful in Docker/K8s where venues are volume-mounted)
- ConfigLoader is struct-based (not interface) for simplicity; promote to interface if mocking is needed in tests
