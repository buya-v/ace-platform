## API Surface

| Method | Path | Handler |
|---|---|---|
| GET | `/platform/v1/tenants` | list all tenants |
| POST | `/platform/v1/tenants` | create tenant (validates id slug + name; defaults status=ONBOARDING) |
| GET | `/platform/v1/tenants/{id}` | get by ID, 404 if not found |
| PATCH | `/platform/v1/tenants/{id}` | update name / governance_tier |
| PUT | `/platform/v1/tenants/{id}/status` | change status, validates enum |

Health endpoints on port 9090: `/healthz`, `/readyz`.

## Decisions Made

- **Zero deps** — stdlib only (`crypto/rand`, `encoding/json`, `net/http`, `sync`, `regexp`, `log/slog`). Consistent with zero-dep platform pattern.
- **newUUID** — copied verbatim from securities-service `handlers_instrument.go` (crypto/rand, RFC 4122 v4). Currently reserved for async provisioning correlation IDs.
- **extractLastSegment** — copied verbatim from securities-service. Handles trailing slash and double-segment paths like `/tenants/{id}/status`.
- **Status sub-resource** — uses the same `strings.HasSuffix("/status")` detection as securities-service handles `/instruments/{id}/status`.
- **InMemoryTenantStore** seeded with `ace-commodities` (ACTIVE, STANDARD) and `mse-equities` (ONBOARDING, FLAGSHIP) matching V29__platform_schemas.sql seed rows.
- **Port allocation** — 8090 API / 9090 health. Follows `8089/9089` → `8090/9090` increment from securities-service.
