# T102 — Auth Service PostgreSQL Persistence

## Summary
Replaced auth-service in-memory store with a PostgreSQL-backed implementation. The in-memory store remains as a fallback when `DB_HOST` is not set, preserving backward compatibility for development and testing.

## Deliverables

### 1. Migration: `infrastructure/db/migrations/V10__auth_tables.sql`
- Creates `auth` schema with 5 tables: `users`, `sessions`, `api_keys`, `audit_log`, `pkce_challenges`
- Proper indexes on `email`, `user_id`, `refresh_token_hash`, `key_hash`, `created_at`
- Foreign keys with `ON DELETE CASCADE` from sessions/api_keys/pkce to users
- `audit_log` table included per spec (not yet consumed by application code — ready for future audit middleware)

### 2. PostgreSQL Store: `src/auth-service/internal/store/postgres.go`
- `PostgresStore` struct implements all 15 methods of the `auth.Store` interface
- Uses `database/sql` with `pgx/v5/stdlib` driver (no shared module dependency)
- Handles nullable columns (`locked_until`, `expires_at`, `redirect_uri`) via `sql.NullTime`/`sql.NullString`
- Unique constraint violations mapped to `ErrAlreadyExists`
- Connection pool configured: 25 max open, 5 max idle, 5min max lifetime
- `ConnectPostgres()` helper builds DSN and pings to validate connectivity

### 3. Updated `src/auth-service/cmd/auth-service/main.go`
- When `DB_HOST` env var is set: connects to PostgreSQL via `store.ConnectPostgres()`
- When `DB_HOST` is not set: uses `store.NewInMemoryStore()` (original behavior)
- Added `_ "github.com/jackc/pgx/v5/stdlib"` driver import
- `defer db.Close()` for clean shutdown

### 4. Dependency update: `go.mod` / `go.sum`
- Added `github.com/jackc/pgx/v5 v5.9.1` and transitive deps (`pgpassfile`, `pgservicefile`)
- Go version bumped to 1.25.0 (required by pgx v5.9.1)

## Decisions Made
- Used string-contains check for unique violation detection (`23505` or `unique constraint`) to avoid importing pgx error types directly into the store package
- PKCE challenges table uses `auth_code` as primary key (VARCHAR) rather than UUID, matching the in-memory store's map key
- `ListUsers()` returns `nil` (not error) on DB failure, matching the in-memory store's non-error signature
- `RevokeUserSessions` does not return `ErrNotFound` when no sessions exist (matches in-memory behavior)

## Test Results
- All existing tests pass (they use InMemoryStore, unaffected by changes)
- Build compiles cleanly with `go build ./...`
