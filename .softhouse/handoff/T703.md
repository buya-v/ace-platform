# T703 Handoff — MCP tools: CRUD tools for instruments, commodities, fees, facilities

## Status: DONE

## Summary

Added 12 new MCP tools across 4 existing tool files. All tools follow the established Zod schema + `client.request()` pattern. TypeScript build passes with zero errors.

## Tools Added

### `src/admin-bot/mcp-server/src/tools/market_data.ts` (+3 tools)
- `create_instrument` — POST /api/v1/admin/instruments — creates a tradable instrument with full contract spec (commodity_id, delivery_month/year, contract_size, tick_size, currency, settlement_type)
- `update_instrument` — PUT /api/v1/admin/instruments/{id} — patches status and/or trading_hours; destructures `id` out of args before passing body
- `create_commodity` — POST /api/v1/admin/commodities — creates a commodity with id, name, category, unit

### `src/admin-bot/mcp-server/src/tools/fees.ts` (+3 tools)
- `create_fee_schedule` — POST /api/v1/admin/fees/schedules — name + effective_from (ISO 8601)
- `create_fee_rule` — POST /api/v1/admin/fees/rules — schedule_id, fee_type, participant_tier, rate_bps, optional min_fee/max_fee
- `set_participant_tier` — PUT /api/v1/admin/fees/tiers/{participant_id} — sets tier for a participant

### `src/admin-bot/mcp-server/src/tools/warehouse.ts` (+4 tools)
- `register_facility` — POST /api/v1/admin/warehouse/facilities — name, optional address/capacity_mt/commodities_accepted array
- `issue_receipt` — POST /api/v1/warehouse/receipts — holder_id, commodity_id, quantity, optional facility_id
- `pledge_receipt` — POST /api/v1/warehouse/receipts/{receipt_id}/pledge — pledges a receipt as collateral
- `create_delivery` — POST /api/v1/warehouse/deliveries — receipt_id, optional destination

### `src/admin-bot/mcp-server/src/tools/participants.ts` (+2 tools)
- `screen_participant` — POST /api/v1/screening/check — runs compliance check on one participant
- `batch_screen` — POST /api/v1/screening/batch — runs compliance check on all participants (no args)

## Implementation Decisions

- All tools use `z.string()`, `z.number()`, `z.array(z.string())` with `.optional()` for nullable fields — consistent with existing tools
- `update_instrument` destructures `{ id, ...rest }` so `id` goes in the URL path and only the update fields go in the body
- `pledge_receipt` passes `{}` as body (empty POST body) — the receipt_id is in the URL path
- `batch_screen` uses an empty schema `{}` — no input parameters needed
- `index.ts` required no changes — all new tools auto-register through the existing `registerXxxTools()` function calls
