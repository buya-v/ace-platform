## Decisions Made

### Test architecture
- Each test creates its own isolated `httptest.Server` via a `newMultiServer()` helper that dispatches by `"METHOD /path"` key — avoids shared state between tests and allows parallel execution.
- `routeHandler.called` uses `atomic.Int32` for safe call-count assertion (same pattern as the existing `executor_test.go`).
- `assert()` local helper keeps the file self-contained with no external test dependencies.

### Mock server design
- Routes are declared as a `map[string]*routeHandler` — makes it immediately clear which endpoints each workflow exercises.
- Unmatched routes return 404. The executor's `withAttribution()` helper calls `GET /api/v1/auth/me`; since this falls to 404 the email is empty and replies have no "Executed by" suffix, keeping assertions simple.

### Assertion strategy
- Assertions check OR conditions (`strings.Contains(r.Reply, "✅") || strings.Contains(r.Reply, "HALT")`) to be resilient to minor reply wording changes.
- Multi-step state is verified with `atomic.Bool` flags and ordering slices.
- Request bodies are decoded inside mock handlers to assert payload correctness (KYC rejection reason, risk limit value).
