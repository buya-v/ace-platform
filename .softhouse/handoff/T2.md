# T2 Handoff — W3C Traceparent Header Propagation

## Summary

Implemented W3C Trace Context (traceparent) propagation across the GarudaX platform with zero external dependencies. All 31 new shared tests pass and all existing gateway tests continue to pass.

## Deliverables

### New files

**`src/shared/internal/observability/tracing.go`**
- `TraceContext` struct: `TraceID` (32 hex chars), `SpanID` (16 hex chars), `ParentSpanID`, `Sampled bool`
- `ParseTraceparent(header string) (TraceContext, error)` — strict W3C validation (version check, lengths, lowercase-hex-only, no-all-zero IDs, reserved `ff` version rejected)
- `FormatTraceparent(tc TraceContext) string` — produces `00-<trace_id>-<span_id>-<flags>`
- `NewTraceContext()` — generates root span (new trace_id + span_id, Sampled=true)
- `NewSpan(tc TraceContext)` — creates child span (same trace_id, new span_id, ParentSpanID set to incoming span_id)
- `TraceFromContext(ctx) TraceContext` — extracts TraceContext from context
- `InjectTraceparent(req *http.Request, tc TraceContext)` — sets traceparent header on outgoing requests
- `TraceMiddleware(logger *slog.Logger) func(http.Handler) http.Handler` — HTTP middleware with full W3C propagation; also stores trace_id via `WithTraceID` for `LoggerFromContext` compatibility

**`src/shared/internal/observability/tracing_test.go`**
- 31 tests covering: parse valid/invalid headers (empty, wrong field count, reserved version, non-hex, all-zero IDs, wrong lengths), ID generation (length, hex format, uniqueness for 100 samples), NewTraceContext/NewSpan semantics, FormatTraceparent round-trip, context store/retrieve, InjectTraceparent no-op on empty, TraceMiddleware (no header, valid header, invalid header fallback, sampled flag propagation, trace_id in context)

### Modified files

**`src/gateway/internal/observability/middleware.go`**
- Updated the existing `TracingMiddleware` to add W3C traceparent support alongside legacy `X-Request-ID`/`X-Trace-ID` headers
- Priority order for trace_id: (1) incoming `traceparent` header, (2) incoming `X-Request-ID`, (3) newly generated
- Sets `traceparent` response header (W3C format) and `X-Request-ID`/`X-Trace-ID` (backward compat)
- Logs `trace_id`, `span_id`, `parent_span_id` on every request_started and request_completed log entry
- Added `parseTraceparent()`, `generateSpanID()`, `isTraceparentHex()`, `isAllZeroString()` helpers (no external deps)
- All 5 pre-existing gateway observability tests continue to pass

## Decisions Made

1. **ParseTraceparent semantic model**: The incoming `parent-id` field is stored as `tc.SpanID` (not `tc.ParentSpanID`). This is because the incoming `parent-id` represents the remote caller's span ID — it is the span we descend from. `NewSpan(tc)` then sets `child.ParentSpanID = tc.SpanID`. This avoids an extra assignment and keeps the semantic chain clear: ParseTraceparent gives you the remote span, NewSpan creates your local span descending from it.

2. **Gateway self-contained**: The gateway keeps its own `internal/observability` package and does not import the shared module. The W3C parsing logic is ~40 lines. This preserves the zero-dep pattern — adding a module dependency would require `go.mod` changes.

3. **Backward compatibility preserved**: The gateway's `TracingMiddleware` still emits `X-Request-ID` and `X-Trace-ID` response headers and still reads `X-Request-ID` from incoming requests. All existing tests pass without modification.

4. **Middleware position unchanged**: `TracingMiddleware` was already wired in `main.go` at the correct position (after RequestID, before Auth). No changes to `main.go` were needed.

## Suggested Follow-ups

- **Proxy propagation**: Wire `InjectTraceparent` in `src/gateway/internal/proxy/httpclient.go` `Do()` method using `TraceFromContext(r.Context())` so all proxied backend requests carry the traceparent forward — enabling end-to-end trace correlation across services.
- **Shared module import**: If the gateway ever adds the shared module as a dependency, the gateway's inline tracing helpers can be removed in favour of `shared/internal/observability` exports.
- **W3C tracestate header**: The optional `tracestate` header (vendor-specific metadata) is not implemented. Can be added to `TraceContext` as `TraceState string` when needed.
- **Sampling policy**: Currently, new root traces are always sampled (`Sampled: true`). A configurable sampling rate (e.g., 10% for high-traffic paths) could be wired in via config.
