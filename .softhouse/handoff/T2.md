## Summary

### Part A ‚Äî TenantMiddleware wired into gateway

The gateway does NOT import `github.com/garudax-platform/shared` ‚Äî it is an independent Go module (`github.com/garudax-platform/gateway`). To avoid a cross-module dependency, the tenant middleware was implemented directly inside the gateway's own middleware package rather than importing from `shared`.

**New file:** `src/gateway/internal/middleware/tenant.go`

Key design choices:
- `TenantID` named string type ‚Äî same as the shared package; prevents accidental mixing with raw strings
- `tenantContextKey{}` unexported struct ‚Äî no external key collision possible
- `tenantHealthPaths` exact-match map for `/healthz`, `/readyz`, `/metrics` ‚Äî no prefix match to prevent accidental bypass
- `TenantFromContext(ctx)` exported helper for handlers that need tenant-aware logic
- Error codes match the shared package: `TENANT_REQUIRED` (401) / `UNKNOWN_TENANT` (403)

**Test file:** `src/gateway/internal/middleware/tenant_test.go` ‚Äî 6 tests:
- `TestTenantMiddleware_ValidTenant` ‚Äî 200 + context injection verified
- `TestTenantMiddleware_MissingHeader` ‚Äî 401 TENANT_REQUIRED
- `TestTenantMiddleware_UnknownTenant` ‚Äî 403 UNKNOWN_TENANT
- `TestTenantMiddleware_HealthBypass` ‚Äî all three bypass paths pass without header
- `TestTenantFromContext_RoundTrip` ‚Äî set/get round-trip
- `TestTenantID_String` ‚Äî Stringer interface

**main.go change:** Tenant middleware instantiated with `[]string{"ace-commodities", "mse-equities"}` and inserted into the chain BEFORE Auth:

```
RequestID ‚Üí Tracing ‚Üí Metrics ‚Üí BodyLimit ‚Üí Tenant ‚Üí Auth ‚Üí RateLimit ‚Üí Router
```

Tenant runs after BodyLimit (so request body reads don't bypass tenant enforcement) and before Auth (so tenant context is available during JWT validation and to all handlers).

## Key Decisions Made

1. **Gateway-local tenant implementation** ‚Äî Rather than adding `github.com/garudax-platform/shared` as a go.mod dependency (which would require a replace directive for a local module path and complicate the build), the tenant logic was reimplemented directly in the gateway's `internal/middleware` package. The API surface (`TenantMiddleware`, `TenantFromContext`, `TenantID`) is identical to the shared package. If the shared module is ever imported by the gateway for other reasons, the gateway-local implementation can be replaced with a thin wrapper.

2. **Tenant middleware position: after BodyLimit, before Auth** ‚Äî Auth needs to see a valid JWT to extract user identity; tenant validation is a prerequisite concern. Placing tenant before auth ensures the tenant claim is always available if auth passes. Placing it after BodyLimit avoids the case where an oversized request body bypasses tenant enforcement.

3. **Exact-match health bypass** ‚Äî `/healthz`, `/readyz`, `/metrics` bypass tenant enforcement using exact path comparison (`map[string]bool`), not prefix matching. This matches the shared package's design (T1 handoff, decision #5) and prevents accidental bypass of paths like `/metricsadmin`.

4. **`auth.` schema not renamed** ‚Äî Auth is platform-level per `platform-architecture.md ¬ß3.4` and the T2 migration handoff (V30). The rename mapping in the task spec did not include `auth.`, and no `auth.` schema references appeared in the grep output from Go source files.

5. **Kafka topic strings not modified** ‚Äî `consumer_test.go` files across all services contain test cases like `{"ace-commodities.clearing.novated", "clearing.novated"}`. The `clearing.` here is a Kafka event type component (topic = `{tenant}.{domain}.{event}`), not a SQL schema name. These were intentionally left unchanged.

## Suggested Follow-ups

- **Domain services adopt `tenant.MustTenant(ctx)`** ‚Äî All domain service handlers should call `middleware.TenantFromContext(ctx)` (or a shared equivalent) at the top of every handler before any DB query. This surfaces missing-tenant bugs at the handler boundary.
- **Observability integration** ‚Äî Log the tenant ID in every structured log line. The gateway's `TracingMiddleware` runs after `TenantMiddleware` so it can call `TenantFromContext` to enrich every span with `tenant_id`.
- **Dynamic tenant registry** ‚Äî The current whitelist `["ace-commodities", "mse-equities"]` is a static compile-time constant. When the Platform Control Plane (Phase 0.7) is available, wire it to a `TenantStore` interface that queries `platform.tenants` with in-memory caching and a TTL refresh.
- **SQL schema is now `ace_*` in code** but V30 migration must be applied to the database before deploying these service binaries. Coordinate deployment: run V30 migration first, then deploy updated service images.
- **`participants.` schema** ‚Äî Not found in any Go source file during the grep pass. If a participants service is added in future (Phase 0.7 control plane or MSE onboarding), its SQL must use `ace_participants.*` from the start.
- **`margin-engine` and `settlement-engine`** ‚Äî Neither had any old schema references in their store files. Their SQL already used qualified names or in-memory stores. No changes needed for these services.
