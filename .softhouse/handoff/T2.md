## Implementation Decisions

### Architecture
- Followed the exact Participants.tsx + CircuitBreakers.tsx patterns for hooks, column defs, and action flows.
- `usePolling(fetchSecuritiesInstruments, 15000)` — polls every 15 s, respects the API's optional `asset_class`/`trading_status` filters.
- The `asset_class` toolbar select drives the polling filter directly (re-fetches from server on change).
- Local `search` filter runs client-side over already-fetched data (ticker + name, case-insensitive).

### State
- `search` — client-side text filter
- `assetClassFilter` — drives server-side query param
- `createModalOpen` / `createForm` / `formErrors` — modal lifecycle
- `statusTarget` — `{ instrument, action: 'HALTED' | 'TRADING' }` — feeds the ConfirmDialog

### Create Modal
- Overlay + inner modal div with `role="dialog" aria-modal="true"`.
- All 7 fields: ticker (text), name (text), asset_class (select), lot_size (number), tick_size (number), currency (text, default MNT), exchange_code (text).
- Submits `parseFloat` coerced lot/tick sizes; ticker uppercased before POST.
- Calls `createSecuritiesInstrument` → toast success → close modal → `refresh()`.
- On error: toast error with API message.

### Status Change
- Single `statusTarget` state shared by both Halt and Resume actions.
- Action button sets `statusTarget` appropriately; ConfirmDialog calls `updateInstrumentStatus(id, action)`.
- "Edit Status" button toggles between HALTED ↔ TRADING (same confirm dialog, different label).

### Columns
- `ticker` — sortable, `mono: true`
- `name` — sortable
- `asset_class` — `render: <StatusBadge status={row.asset_class} />`
- `lot_size` — `align: 'right'`, `mono: true`
- `tick_size` — `align: 'right'`, `mono: true`
- `trading_status` — `render: <StatusBadge status={row.trading_status} />`
- `actions` — Halt/Resume button + Edit Status button

### validateInstrumentForm (pure export)
- All 5 required fields validated; returns `FormErrors` record.
- `lot_size` and `tick_size` must be numeric AND > 0.
- Optional fields (`currency`, `exchange_code`) not validated.

### CSS Module
- Follows CircuitBreakers.module.css exactly for `.overlay`, `.modal`, `.formLabel`, `.formInput`, `.fieldError`, `.modalActions`, `.cancelBtn`, `.saveBtn`.
- Added `.toolbar` (flex row with gap + align-items), `.createBtn` (accent-blue), `.searchInput`, `.select`.
- Added `aria-invalid` red-border style on `.formInput[aria-invalid="true"]`.
- Modal has `max-height: 90vh; overflow-y: auto` to handle the taller 7-field form.
