## Summary

Implemented three features in the securities-service:

### Part A — Tiered Tick Validation (`tick_table.go`)
- New file: `src/securities-service/internal/engine/tick_table.go`
- `ValidateTickSize(price, tickTable, defaultTickSize)` function:
  - If tickTable is nil or has no tiers: validates against defaultTickSize
  - Finds tier where `MinPrice <= price < MaxPrice` (last tier if price >= all MaxPrice)
  - Validates `math.Abs(math.Remainder(price, tier.TickSize)) < 1e-9`
  - Error format: `"price %.2f must be a multiple of tick_size %.4f (tier: %.2f-%.2f)"`
- Wired into `handleSubmitOrder` in `handlers_order.go`:
  - Before the existing default tick_size check, attempts to load tick table from `tickTableStore`
  - If found, uses `ValidateTickSize` with tiered table
  - If not found (ErrNotFound), falls back to instrument default tick_size

### Part B — IOC/FOK Enforcement (`engine.go`)
- **FOK pre-check** (before matching loop):
  - Calculates total available crossing volume on the opposite side
  - If total < order quantity: returns error `"FOK order cannot be fully filled (available: N, requested: M)"`
  - Only proceeds with matching if fully fillable
- **IOC post-check** (after matching loop):
  - If remaining qty > 0 and filled qty == 0: sets status to CANCELLED
  - If remaining qty > 0 and filled qty > 0: sets status to PARTIALLY_FILLED
  - Updates the order in the store
- GTC/DAY orders keep existing behavior (stay as resting if not fully filled)

### Part C — Order Throttle (`handlers_order.go`)
- Added `tickTableStore` and `throttleStore` fields to Server struct
- Updated `server.New()` constructor to accept both new stores
- In `handleSubmitOrder`, before any validation: calls `throttleStore.CheckAndIncrement(participantID, 100)`
- If throttled: responds `429 {"error":{"code":"THROTTLED","message":"rate limit exceeded: max 100 orders/sec"}}`

## Decisions
- Throttle check uses `participantID` from the order body (not firm ID) since that's what's available pre-validation
- Throttle is nil-safe: if `throttleStore` is nil, throttle check is skipped
- FOK pre-check counts only iceberg visible quantity (not hidden) to match the matching loop behavior
- IOC with partial fills gets PARTIALLY_FILLED status (not CANCELLED) to preserve fill history
- Tick table lookup uses store.ErrNotFound as the "not found" signal to fall back to default
