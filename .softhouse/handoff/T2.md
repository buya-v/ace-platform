## Technical Details

### Error Boundary Behavior
1. **Normal operation**: Component passes children through unchanged
2. **Error caught**:
   - State updates to `hasError: true`
   - Error logged to console with details
   - Fallback UI rendered (error card)
3. **User recovery**: Clicking "Reload page" executes `window.location.reload()`

### Why This Placement?
The ErrorBoundary wraps the `<Outlet />` (react-router) rather than wrapping `DashboardInner` because:
- Errors in layout components (Sidebar, TopBar) are usually not recoverable by reload
- Errors in page content (routed components) are typically recoverable via reload
- This provides granular error handling for the main content area

## Implementation Notes

### Class Component Choice
Used a class component (not functional) because:
- React Error Boundaries MUST be class components (no hook equivalent as of React 18)
- `componentDidCatch()` is a lifecycle method available only on class components
- `getDerivedStateFromError()` is required to update state during render

### Icon Import
Used existing `AlertIcon` from `icons.tsx` (already available, no new icons created):
- Size 48px for high visibility
- `--accent-red` color indicates error state
- Consistent with project's icon library

### Z-index Strategy
- ErrorBoundary container: z-index 9998
- Modal dialogs: z-index 1000
- This ensures modals (ConfirmDialog) can appear over error boundary if both occur
