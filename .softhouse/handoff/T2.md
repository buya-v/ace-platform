## Decisions Made

1. **`newTradingParamsServer` helper** builds a fully-wired `httptest.Server` with a live `InMemoryTradingParamSetStore` — mirrors the existing `newTestServerWithStores` pattern.
2. **`newServerWithParamSet` helper** seeds both an instrument and a param set directly (bypassing HTTP) for the order-validation tests, keeping those tests focused on order-submission, not param-set CRUD.
3. **Store tests use the `store_test` package** (external black-box) — consistent with all prior store tests in the file.
4. **Handler tests use `package server`** (internal white-box) — shares `testTenant`, `doJSON`, `assertStatus`, `decodeBody` helpers from `handlers_instrument_test.go`.
5. **`TestOrderValidation_AllowedOrderType` uses `LIMIT` + price** because LIMIT requires a price > 0 and the instrument has tick_size=0.01; price=10.00 satisfies both lot-size (lotSize=1) and tick-size constraints.
6. **`TestOrderValidation_DisallowedOrderType` uses `STOP`** — STOP is in the global `validOrderTypes` set but absent from the param-set's `AllowedOrderTypes`, confirming param-set enforcement fires after global enum validation.
