## Decisions Made

1. **`tenantHeader` used for all 10 steps** — All steps use admin context with the MSE tenant header, consistent with all other admin operation steps in the file.

2. **`adv-1` `instrument_weights` uses computed keys** — Body function uses `state.apu_id` and `state.gov_id` as dynamic object keys (with fallback to `'APU'` and `'GOV'`). This matches the pattern of using state from prior steps (inst-1, inst-2 extract these IDs).

3. **`adv-2` URL is a function** — Uses `(state) => ...` to interpolate `state.index_id` (extracted from `adv-1`'s response), with fallback to `'MSE-TOP20'`.

4. **`adv-2` `validateResponse` checks for `current_value` field** — More specific than `okValidator` to verify the calculation endpoint returns a meaningful response shape.

5. **`adv-5` `validateResponse` checks array contains MINING folder** — Validates the hierarchical structure was created correctly, not just that the request succeeded.

6. **`adv-7` `validateResponse` checks array is non-empty** — Ensures the permissions were actually stored and returned.

7. **`adv-9` `validateResponse` checks response is a non-null object** — Surveillance dashboard may return varying shapes; checking for object type is the appropriate loose validation.

8. **`adv-10` `validateResponse` checks `Array.isArray`** — Warnings endpoint is expected to return an array (possibly empty); validates the response contract.

9. **`extractState` on `adv-1`** — Extracts `index_id` from the create-index response for use in `adv-2`. Falls back to `'MSE-TOP20'` if `b.id` is absent.
