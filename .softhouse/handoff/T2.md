## Summary

Implemented three features in the securities-service matching engine:

### Part A ‚Äî CircuitBreakerEngine (`circuit_breaker.go`)
- New file: `src/securities-service/internal/engine/circuit_breaker.go`
- `CircuitBreakerEngine` struct with `store.CircuitBreakerStore` dependency
- `NewCircuitBreakerEngine(store)` constructor
- `ValidatePrice(instrumentID, price)` ‚Äî checks static upper/lower and dynamic upper/lower thresholds against reference price and last traded price; returns `(bool, *CircuitBreakerEvent, error)`
- `OnTrade(instrumentID, price)` ‚Äî updates last traded price in the store after each trade

### Part B ‚Äî Circuit Breaker wired into MatchOrder (`engine.go`)
- Added `circuitBreakerEngine *CircuitBreakerEngine` field to `MatchingEngine`
- Extended `NewMatchingEngine` with 7th parameter (nil-safe)
- Before matching: if order is LIMIT and circuit breaker engine is configured, calls `ValidatePrice`; returns error `"circuit breaker triggered: {event.Type}"` on breach
- After each trade: calls `circuitBreakerEngine.OnTrade` to update last traded price
- `main.go` wires `NewCircuitBreakerEngine(store.NewInMemoryCircuitBreakerStore())`

### Part C ‚Äî Self-Trade Prevention in MatchOrder (`engine.go`)
- Before creating a trade, checks `order.ParticipantID == resting.ParticipantID`
- Gets STP mode from instrument (`inst.STPMode`), defaults to `STP_CANCEL_NEWEST`
- `STP_CANCEL_NEWEST`: skips the resting order, continues matching others
- `STP_CANCEL_OLDEST`: cancels resting order (status ‚Üí CANCELLED), continues matching
- `STP_CANCEL_BOTH`: cancels both orders, stops matching loop

## Decisions
- Circuit breaker `ValidatePrice` only runs for LIMIT orders (MARKET orders have no price to validate)
- STP default mode is `STP_CANCEL_NEWEST` when instrument has no STPMode set
- `OnTrade` errors are non-fatal (suppressed with `_ =`) to avoid breaking matching on store issues
- All existing callers pass `nil` for the new circuit breaker parameter, preserving backward compatibility
