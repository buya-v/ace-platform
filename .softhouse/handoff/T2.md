## Summary

Implemented three features in the securities-service:

### Part A — Permission Checks (`permissions.go`)
- New file: `src/securities-service/internal/engine/permissions.go`
- `CheckPermission(participantID, requiredPerm, store)` function
  1. Gets participant from store; returns "participant not found" if missing
  2. Checks `Status != PARTICIPANT_ACTIVE`; returns "participant is suspended"
  3. Scans `Permissions` slice for `requiredPerm`; returns "permission denied: requires <perm>" if absent
  4. Returns nil on success

### Supporting Types Added
- `types.go`: `ExchangeParticipant` struct, `ParticipantStatus` type (`PARTICIPANT_ACTIVE`, `PARTICIPANT_SUSPENDED`), 5 permission constants (`TRADE_EQUITY`, `TRADE_BOND`, `TRADE_ETF`, `MARKET_MAKER`, `SPONSORED_ACCESS`)
- `types.go`: `DayState` type (`DAY_CLOSED`, `DAY_PRE_OPEN`, `DAY_TRADING`, `DAY_POST_CLOSE`)
- `types.go`: Iceberg fields on `SecurityOrder` — `VisibleQuantity`, `HiddenQuantity`
- `store.go`: `ParticipantStore` interface + `InMemoryParticipantStore` implementation

### Part B — Iceberg Matching (`engine.go`)
- Modified `MatchOrder` matching loop to handle iceberg orders:
  - **Resting iceberg**: matches against `VisibleQuantity` instead of full remaining quantity
  - **Incoming iceberg**: matches visible portion only
  - After fill: reduces `VisibleQuantity` by fill amount
  - Replenishment: when `VisibleQuantity == 0` and `HiddenQuantity > 0`, replenishes with `min(fillQty, HiddenQuantity)`
  - FILLED status only when both `VisibleQuantity` and `HiddenQuantity` are 0
- Non-iceberg orders (VisibleQuantity=0, HiddenQuantity=0) follow the original code path unchanged

### Part C — Day Lifecycle Manager (`day_manager.go`)
- New file: `src/securities-service/internal/engine/day_manager.go`
- `DayManager` struct: `sessionManager`, `instrumentStore`, `currentState`, `mu sync.RWMutex`
- `NewDayManager(sessionManager, instrumentStore)` — starts as `DAY_CLOSED`
- `StartDay()`: CLOSED -> PRE_OPEN, transitions all ACTIVE instruments to PRE_OPEN
- `StartTrading()`: PRE_OPEN -> TRADING, transitions all instruments to CONTINUOUS (triggers opening auctions)
- `EndTrading()`: TRADING -> POST_CLOSE, transitions all to CLOSING_AUCTION then CLOSED
- `EndDay()`: POST_CLOSE -> CLOSED
- `GetState()` returns current DayState
- Invalid transitions return descriptive errors

## Decisions
- Iceberg detection: an order is iceberg if `VisibleQuantity > 0 || HiddenQuantity > 0`; orders with both zero follow the original non-iceberg path
- Replenish amount uses `min(fillQty, HiddenQuantity)` — the fill quantity serves as a proxy for the original display size
- DayManager passes empty tenantID to `TransitionTo` — the session manager uses it for Kafka events only, and day lifecycle transitions are infrastructure-level
- ParticipantStore is a standalone interface, not wired into MatchingEngine constructor (keeps existing constructor backward-compatible)
