## Decisions Made

- **Nil-safe everywhere**: all three wiring functions and the engine check `if p == nil { return nil }` so tests work without any Kafka infrastructure
- **Non-fatal publish errors**: publish failures are silently swallowed in both engine and handlers â€” order submission and matching must succeed even if event publishing is degraded
- **Producer on both engine and server**: the engine publishes per-trade events; the server handler publishes per-order lifecycle events (created, cancelled)
- **ChannelProducer buffer = 256**: generous buffer for dev/test; production swaps in a real Kafka wire-protocol adapter behind the `Producer` interface
- **Exact matching-engine API shape**: `Publish(topic, key string, event *Event)` matches the template for cross-service consistency
