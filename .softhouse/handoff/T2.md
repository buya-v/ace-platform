## Summary

Implemented three feature sets in `securities-service` plus gateway route updates:

### Part A — Session Extend / Shorten
- Added `SessionExtension` struct to `internal/types/types.go`
- Added `POST /api/v1/securities/sessions/{instrument_id}/extend` and `/shorten` handlers in `internal/server/handlers_session.go`
- Both endpoints require `{duration_minutes, reason}` body, return `{instrument_id, action, duration_minutes, new_end_time, reason, created_at}`
- `new_end_time` is calculated as `now + duration_minutes * time.Minute` (UTC, RFC3339)
- Routed through `handleSessionOrAdjustment` wildcard dispatcher (replaces direct `handleSessionTransition` route)

### Part B — Participant Lock / Unlock
- Added `ParticipantLocked ParticipantStatus = "PARTICIPANT_LOCKED"` constant to `types.go`
- Added `LockedAt string` and `LockReason string` fields (json: `locked_at`, `lock_reason`, omitempty) to `ExchangeParticipant`
- Added `Lock(id, reason string) error` and `Unlock(id string) error` to `ParticipantStore` interface in `store.go`
- Implemented both in `InMemoryParticipantStore` (store.go) and `PgParticipantStore` (pg_store_ops.go)
- Lock: sets status=PARTICIPANT_LOCKED, lock_reason, locked_at; orders are preserved (not cancelled)
- Unlock: sets status=PARTICIPANT_ACTIVE, clears lock_reason and locked_at
- Both handlers emit best-effort audit log entries via `s.auditStore.Log(...)`
- Added handlers `handleLockParticipant` and `handleUnlockParticipant` in `handlers_participant.go`

### Part C — Firm View (Surveillance)
- Created `internal/server/handlers_firm_view.go`
- `GET /api/v1/securities/surveillance/firm-view/{firm_id}`
- Aggregates: orders (count by status across firm's participants), trades (total + buy/sell volume), positions (all positions for firm's participants), alerts (count by status from surveillance store)
- Returns last 10 recent orders and last 10 recent trades, sorted by `created_at` descending
- Tenant-aware: requires `X-GarudaX-Tenant` header

## Decisions Made

1. **Session adjust does not persist** — no `SessionExtensionStore` interface exists; the handler calculates and returns `new_end_time` but does not store the extension. If persistence is needed, add a store and call it here.
2. **Lock preserves orders** — unlike suspend (which could cancel open orders), lock only changes participant status. Existing orders remain in place for manual review.
3. **Firm view uses participant→order linkage** — aggregates orders by looking up all participants with `firm_id` match, then filtering orders by `participant_id`. This is O(participants × orders) — acceptable for current in-memory store scale.
4. **Audit log is best-effort** — `if s.auditStore != nil { s.auditStore.Log(...) }` pattern; nil auditStore silently skips, matching existing patterns in the codebase.
5. **`handleSessionOrAdjustment` replaces the previous wildcard** — the old server.go registered `/sessions/` → `handleSessionTransition` directly; this is now a dispatcher that also handles extend and shorten.

## Suggested Follow-ups

- **SessionExtension persistence** — create `SessionExtensionStore` interface + in-memory + PgStore if audit trail of session adjustments is required.
- **Participant lock enforcement** — the lock status is set but the order submission handler (`handleSubmitOrder`) does not check for `PARTICIPANT_LOCKED`. A locked participant can still submit orders through the current code. Add a check in `handleSubmitOrder` mirroring the existing `PARTICIPANT_SUSPENDED` check.
- **Firm view `recent_orders` sort** — currently sorted by `CreatedAt` string comparison (lexicographic). This works correctly with RFC3339 format but should be noted as an assumption.
- **Firm view alert integration** — the `SurveillanceStore.ListAlerts` is called without firm-specific filtering; all alerts for the tenant are counted. If firm-scoped alerts are required, add a `FirmID` filter to `SurveillanceFilters`.
- **Tests for new endpoints** — `handlers_session_test.go` does not yet have tests for extend/shorten; `handlers_participant_test.go` does not have tests for lock/unlock; `handlers_firm_view_test.go` does not exist. These would follow the existing patterns in sibling test files.
