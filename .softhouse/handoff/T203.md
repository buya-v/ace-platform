# T203 — Bot Orchestrator: Nano Routing + Claude Escalation

## Status: DONE

## Summary

Created the bot orchestrator at `src/admin-bot/orchestrator/` that routes admin chat requests between GPT-nano (simple tasks) and Claude CLI (complex analysis).

## Files Created

- `src/admin-bot/orchestrator/package.json` — Package config with openai + node-cron deps
- `src/admin-bot/orchestrator/tsconfig.json` — ES2022/NodeNext TypeScript config
- `src/admin-bot/orchestrator/src/types.ts` — Shared types (ChatRequest, ChatResponse, Action, Suggestion, etc.)
- `src/admin-bot/orchestrator/src/router.ts` — Request classifier: GPT-nano (gpt-5.4-nano / gpt-4o-mini fallback) or keyword-based classification
- `src/admin-bot/orchestrator/src/nano-handler.ts` — Handles simple categories (status_query, ticket_create, ticket_update, admin_action, report_generate) via gateway API calls + GPT-nano function calling for extraction
- `src/admin-bot/orchestrator/src/claude-handler.ts` — Handles complex_analysis by spawning `claude -p` with 60s timeout, 2 retries, 5s backoff
- `src/admin-bot/orchestrator/src/daily-admin.ts` — DailyScheduler with node-cron: morning health (9:00), EOD report (17:00 M-F), aging tickets (10:00)
- `src/admin-bot/orchestrator/src/api.ts` — HTTP server (Node built-in, no Express): POST /chat, GET /suggestions, GET /health, CORS headers
- `src/admin-bot/orchestrator/src/index.ts` — Entry point: starts server on PORT 3100, starts scheduler, graceful SIGTERM/SIGINT shutdown

## Key Design Decisions

1. **Zero Express dependency** — Uses Node's built-in `http` module to keep deps minimal (only openai + node-cron)
2. **Graceful fallbacks everywhere** — No OPENAI_API_KEY: keyword classification; no claude binary: fallback message; API errors: human-readable error responses
3. **GPT-nano function calling** — Used for structured ticket extraction (create_ticket tool) with gpt-5.4-nano primary, gpt-4o-mini fallback
4. **Claude CLI isolation** — Spawns as child process with project root CWD, 60s kill timeout, max $0.10 budget per request
5. **Page-aware suggestions** — GET /suggestions?page= returns context-specific quick actions for dashboard, tickets, trading, settlement, participants
