# T207 — MCP Server: Ticket + Daily Admin Tools Extension

## Status: DONE

## Summary
Extended the GarudaX MCP server with 10 new tools and 2 new resources for ticket management and daily administration workflows.

## Files Created
- `src/admin-bot/mcp-server/src/tools/tickets.ts` — 6 ticket management tools
- `src/admin-bot/mcp-server/src/tools/daily_admin.ts` — 4 daily admin tools
- `src/admin-bot/mcp-server/src/resources/tickets.ts` — 2 MCP resources

## Files Modified
- `src/admin-bot/mcp-server/src/index.ts` — Added imports and registrations for new tools and resources

## Tools Added

### Ticket Tools (tickets.ts)
1. `create_ticket(title, description, category, priority)` — POST /api/v1/tickets
2. `list_tickets(status?, category?, limit?)` — GET /api/v1/tickets with query filters
3. `get_ticket(id)` — GET /api/v1/tickets/{id}
4. `update_ticket(id, status?, assignee_id?, priority?)` — PATCH /api/v1/tickets/{id}
5. `add_ticket_comment(ticket_id, body)` — POST /api/v1/tickets/{id}/comments
6. `get_ticket_stats()` — GET /api/v1/admin/tickets/stats

### Daily Admin Tools (daily_admin.ts)
7. `run_daily_health_check()` — Comprehensive health report (services, circuit breakers, halted instruments)
8. `run_eod_report()` — End-of-day summary (settlement cycles, margin stats, trades)
9. `get_aging_tickets()` — Open tickets older than 24 hours
10. `get_unresolved_alerts_summary()` — Compliance alerts grouped by severity with counts

### Resources (resources/tickets.ts)
11. `open-tickets` (garudax://tickets/open) — List of open tickets
12. `ticket-stats` (garudax://tickets/stats) — Ticket statistics

## Decisions
- Followed the exact same pattern as existing tool files (compliance.ts, settlement.ts, etc.)
- Daily admin tools aggregate multiple API calls into single reports with error resilience (try/catch per endpoint)
- `get_aging_tickets` filters client-side for tickets >24h old since the API doesn't support age filtering
- `get_unresolved_alerts_summary` groups alerts by severity field, defaulting to "unknown" if missing
- All tools return formatted JSON in MCP content blocks
