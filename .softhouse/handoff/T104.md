# T104 — Clearing Engine PostgreSQL Persistence

## Status: DONE

## Summary
Replaced clearing-engine in-memory stores with PostgreSQL persistence for obligations, positions, and netting results. The implementation uses `database/sql` with `pgx/v5/stdlib` driver and falls back to in-memory storage when `DB_HOST` is not set.

## Artifacts Created

### 1. Migration: `infrastructure/db/migrations/V12__clearing_engine_tables.sql`
- `clearing.obligations` — stores novated trade obligations with indexes on trade_id, participant_id, instrument_id, status
- `clearing.positions` — composite PK (participant_id, instrument_id) for net positions per participant/instrument
- `clearing.netting_results` — persists multilateral netting run outputs with indexes on run_id, participant_id, instrument_id

### 2. PostgreSQL Store: `src/clearing-engine/internal/store/postgres.go`
- `PostgresObligationStore` — implements `ObligationStore` interface (Append, ByTrade, ByParticipant, ByInstrument, ByStatus, All)
- `PostgresPositionStore` — provides SavePosition (UPSERT via INSERT ON CONFLICT UPDATE), GetPosition, GetPositionsByParticipant, GetPositionsByInstrument, SaveNettingResult
- `OpenDB()` — connection factory with pool settings (25 max open, 5 idle, 5min lifetime)
- Decimal values serialized via `types.Decimal.String()` and deserialized via `types.ParseDecimal()`
- Side and Status stored as SMALLINT matching Go const values (SideBuy=1, SideSell=2, etc.)

### 3. Updated: `src/clearing-engine/cmd/clearing-engine/main.go`
- Checks `DB_HOST` env var at startup
- If set: connects to PostgreSQL using DB_HOST, DB_PORT (default 5432), DB_USER (default "clearing"), DB_PASSWORD, DB_NAME (default "garudax"), DB_SSLMODE (default "disable")
- If not set: falls back to `InMemoryObligationStore` (existing behavior)

### 4. Tests: `src/clearing-engine/internal/store/postgres_test.go`
- 7 unit tests covering decimal serialization, store construction, field mapping for obligations/positions, netting result ID format
- All 8 existing test packages continue to pass (engine, kafka, netting, novation, position, store, types)

### 5. Updated: `src/clearing-engine/go.mod`
- Added `github.com/jackc/pgx/v5 v5.7.4` dependency with transitive deps

## Decisions Made
- Schema uses `SMALLINT` for Side and Status columns (matching Go int constants) rather than VARCHAR status strings — avoids string parsing overhead and enables efficient indexing
- Obligations table uses `BIGINT` for quantity (matching Go `uint64`) rather than `DECIMAL` — quantities are always whole units
- Positions use composite PK `(participant_id, instrument_id)` with UPSERT pattern — matches the logical key used by the in-memory `position.Manager`
- `PostgresPositionStore` is a separate type from `PostgresObligationStore` (not behind the same interface) because positions require UPSERT semantics not present in the `ObligationStore` interface
- The `position.Manager` still handles in-memory position calculation logic; the Postgres store is for persistence, not for replacing the Manager
