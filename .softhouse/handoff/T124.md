# T124 — Trading UI: Advanced Order Entry, Position Management, Trade History

## Status: DONE

## Summary

Enhanced the web-ui trading interface with production-grade features across four areas:

### 1. Advanced Order Entry
- Added order type dropdown: Limit, Market, Stop-Limit, Stop-Market, Iceberg
- Added time-in-force selector: DAY, GTC, GTD (with datetime picker), IOC, FOK
- Conditional fields: stop_price shown for stop orders, display_qty for iceberg, expiry for GTD
- Replaced the old button-based type selector with dropdown `<select>` elements
- Migrated IOC/FOK from OrderType to TimeInForce (where they belong semantically)

### 2. Enhanced Position Panel
- Added current price column (populated via ticker polling)
- Added P&L formatting with +/- prefix and color coding (green/red/neutral)
- Added total P&L summary in the title bar
- Added "Close Position" button that submits opposing market order
- Grid updated to 7 columns (added Current, Close action)

### 3. Trade History with Filters
- Refactored TradeHistory from simple real-time trade feed to full filterable history table
- Filter panel: date range (start/end), instrument dropdown, side (buy/sell)
- Columns: Time, Instrument, Side, Qty, Price, Total Value
- CSV Export button (builds CSV via pure `buildTradesCsv` function)
- Trade count footer
- Trading page now has tabbed bottom panel: Positions | Trade History

### 4. Instrument Info Panel
- New InstrumentInfo component below order entry
- Expandable "Contract Specs" panel with grid display
- Shows: symbol, commodity, delivery month, contract size, tick size, lot size, position limit, settlement type, delivery location, status
- Fetches from GET /api/v1/instruments/{id}

### 5. New Types
- Extended `InstrumentDetail` type with contractSize, positionLimit, currency, exchange, etc.
- Added `TradeRecord` type (with instrument context and totalValue)
- Added `TradeHistoryFilter` type

## Decisions Made

1. **IOC/FOK moved to TimeInForce**: In the original code, IOC and FOK were listed as order types. They are properly time-in-force qualifiers, so they were moved. The OrderType enum now has: limit, market, stop-limit, stop-market, iceberg.

2. **Pure function extraction**: All business logic extracted as pure functions in `services/tradingUtils.ts`: calculateUnrealizedPnl, calculateTotalPnl, formatPnl, pnlColorClass, filterTrades, buildTradesCsv, formatTradeTime, calculateTradeValue. This enables 100% testability without RTL.

3. **Tabbed bottom panel**: Added a tab bar (Positions / Trade History) to the bottom area instead of cramming both into the same space.

4. **TradeHistory is now dual-purpose**: The component accepts `TradeRecord[]` (historical trades with instrument info) instead of `Trade[]` (real-time WebSocket trades). The real-time trade feed is now handled by the Trading page's WebSocket integration, while TradeHistory shows the full filterable history.
