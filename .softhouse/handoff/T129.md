# T129 — Warehouse Receipt Financing: Pledge-as-Collateral Integration with Margin

## Status: DONE

## Summary

Connected warehouse receipts to the margin system so physical commodity stored in certified warehouses can serve as collateral. This is a key feature for agricultural commodity exchanges (ECX, EAX model): farmer stores grain, gets receipt, pledges receipt, margin requirement is reduced by haircut-adjusted value.

## Files Created/Modified

### New Files
- `src/margin-engine/internal/collateral/warehouse_source.go` — `WarehouseCollateralSource` that fetches pledged receipts from warehouse-service HTTP API and calculates collateral value as `quantity * settlement_price * haircut`
- `src/margin-engine/internal/collateral/warehouse_source_test.go` — 18 tests covering: receipt valuation, haircut application, unknown commodity handling, HTTP success/error/unreachable/invalid JSON scenarios, fractional quantities
- `src/margin-engine/internal/collateral/composite_source.go` — `CompositeCollateralSource` that aggregates collateral from multiple sources (clearing positions + warehouse receipts + future cash). Each source is best-effort with panic recovery.
- `src/margin-engine/internal/collateral/composite_source_test.go` — 9 tests covering: empty/single/multiple sources, panic recovery, per-participant routing, source count

### Modified Files
- `src/margin-engine/internal/params/params.go` — Added `GetSpotPrice(instrumentID string) (Decimal, bool)` method to support commodity price lookups for receipt valuation
- `src/margin-engine/cmd/margin-engine/main.go` — Updated to use `CompositeCollateralSource` when `WAREHOUSE_SERVICE_ADDR` env var is set. Supports `WAREHOUSE_HAIRCUT` env var for custom haircut.

## Design Decisions

1. **Haircut default 80%** — Standard for physical commodity collateral. Configurable via `WithHaircut()` option or `WAREHOUSE_HAIRCUT` env var.
2. **PriceProvider function** — Rather than hardcoding price lookups, the warehouse source accepts a `PriceProvider func(commodityID string) (Decimal, bool)`. In production, this is wired to `paramStore.GetSpotPrice()`.
3. **Composite pattern** — `CompositeCollateralSource` sums collateral from all registered sources. Each source is independent and best-effort (errors/panics return zero, don't fail the calculation). This allows adding cash collateral in the future without modifying existing code.
4. **Zero-dep approach** — No new external dependencies. Uses existing `net/http` for warehouse-service communication.
5. **Graceful degradation** — If warehouse-service is unreachable, collateral contribution is zero (margin calculation still works with clearing positions only).
