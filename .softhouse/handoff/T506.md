## Key decisions and findings

### Conversational phrasing
- Conversational prefixes like "can you …", "I need to …", "I want to …" do NOT trigger the regex rules (all regex rules require the verb at position 0). These correctly fall to `keywordAction`. Tests document expected outcomes based on keyword presence.
- "I want to see the order book for corn" → `unknown` (not `instruments`) because neither "instrument", "commodity", nor "contract" appears in the phrase. Test asserts `unknown`.
- "give me a market summary" → `generate_report` / entity `market_summary` because `reReport` scans anywhere in the trimmed message.
- "what is the audit trail showing?" → `unknown` (no keyword match for "audit").

### Typo resilience
- `reHalt` (`(?i)^(halt|stop|pause)\s+(.+)`) uses `\s+` so double/triple spaces are absorbed without issue.
- Misspelled verb "approv" (not "approve") does not trigger `reApprove` which anchors on `^approve`.
- "halt wheet" still matches `halt_instrument`; the unrecognised entity "wheet" is passed through unchanged (not empty).

### Edge cases
- Empty string → `unknown` (keywordAction default branch).
- "halt" alone → does NOT match `reHalt` because `(.+)` requires at least one character after the whitespace.
- "buy wheat" (no qty/price) → does NOT match `reOrder`.
- "buy 0 wheat at 325" → DOES match (qty="0" is valid for the regex `\d+`).
- "sell -5 corn at 450" → does NOT match (reOrder requires `\d+(?:\.\d+)?`, a bare minus before digits blocks the match).
- Unicode "小麦" → `halt_instrument` action (reHalt verb matches), but entity is NOT WHT-HRW-2026M07-UB.
- 500-char message → no crash, graceful unknown return.

### ResolveInstrumentAlias
- "w" (single char) → resolves to `WHT-HRW-2026M07-UB` because "w" is a prefix of "wheat" in `prefixGroups`. Documented this as the actual behaviour (task spec said "no match" but the implementation disagrees).
- "abc-def-xyz" (unknown full-ID format with dashes) → `ABC-DEF-XYZ` (passthrough, uppercased).
- "so" → `SBN-NO2-2026M11-UB` (prefix of "soybeans"/"soybean"/"soy").
- "li" → `LVS-CATTLE-2026M10-UB` (prefix of "livestock").
