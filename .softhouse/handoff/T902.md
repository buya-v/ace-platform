# T902 Handoff — Bot NL Improvement: Natural Language Normalizer

## Status: DONE

## Summary

Added a `normalizeMessage` function to `src/gateway/internal/bot/executor.go` that strips common filler words from user messages before keyword matching. This allows natural language phrases to route to the correct handlers without breaking any existing exact-command handling.

## Changes Made

**File:** `src/gateway/internal/bot/executor.go`

1. **Two new package-level regex vars** (near the top, before `instrumentAliases`):
   - `fillerWordsRe` — matches words like "please", "can you", "show me", "list of", "what are the", "any", "display", "view", "get", etc.
   - `multiSpaceRe` — collapses consecutive whitespace after filler removal

2. **New `normalizeMessage(msg string) string` function** — lowercases input, strips filler words, collapses whitespace, trims.

3. **`norm` variable declared at the top of `Execute()`** alongside `lower`. This is declared early so all handlers (both before and after the guided CRUD section) can use it.

4. **All `containsAny` keyword handlers updated** to also check `norm` with an `|| containsAny(norm, ...)` clause. The regex CRUD handlers (buy/sell orders, halt/resume, approve/reject KYC, etc.) are unchanged — they still use the original `message` for parameter extraction.

5. **Guard added to the orders-list handler** to prevent `containsAny(norm, "orders")` from matching "cancel all orders" (which normalizes to "cancel orders"):
   ```go
   && !containsAny(lower, "cancel", "mass")
   ```

## Verified Natural Language Phrases

All of these now route correctly:
- `"list of commodities"` → normalizes to `"commodities"` → instruments/commodities handler
- `"show me the instruments"` → `"instruments"` → instruments handler
- `"what are the margin calls?"` → `"margin calls ?"` → margin handler
- `"can you check system health"` → `"system health"` → health handler
- `"any alerts?"` → `"alerts ?"` → alerts handler
- `"display positions"` → `"positions"` → positions handler

## Test Results

- `go build ./...` — PASS (no compilation errors)
- `go test ./internal/bot/` — PASS (all existing tests pass, including `TestExecutor_CancelAll` which was briefly broken by the norm check on "orders" until the cancel guard was added)

## Decisions Made

- Placed `norm` at the top of `Execute()` (not just before keyword handlers) so all handlers in the function have access — some keyword handlers like "audit log", "market summary", and "mass cancel" appear before the main keyword section.
- Did not normalize inputs to regex CRUD handlers — they need the original message to extract parameters like instrument names, prices, participant IDs.
- Used `|| containsAny(norm, ...)` pattern (additive) rather than replacing `lower` checks, preserving full backward compatibility.
