## Part 2 — Health Check Audit

**Finding: All 8 Go services already have `/healthz` and `/readyz` endpoints.**

Each service's `internal/server/server.go` implements `StartHealthServer()` with:
- `/healthz` — always returns `200 OK` (liveness probe)
- `/readyz` — returns `200 OK` when `readyFlag`/`ready` atomic is set, `503` otherwise

| Service | readyFlag mechanism | /healthz | /readyz |
|---|---|---|---|
| matching-engine | atomic.Int32 on Server | YES | YES |
| clearing-engine | atomic.Bool on Server | YES | YES |
| margin-engine | atomic.Bool on Server | YES | YES |
| settlement-engine | atomic.Bool on Server | YES | YES |
| auth-service | atomic.Bool on Server | YES | YES |
| compliance-service | atomic.Bool on Server | YES | YES |
| market-data-service | atomic.Int32 on Server | YES | YES |
| warehouse-service | atomic.Int32 on Server | YES | YES |

The gateway follows the same pattern but uses a package-level `readyFlag int32` in `src/gateway/internal/handler/health.go` with `SetReady()` / `IsReady()` helpers.

No code changes were required for Part 2.
