## Summary

Created `src/securities-service/internal/store/pg_store_test.go` with three groups of tests, none requiring a live database.

### 1. Compile-time interface compliance (10 tests)
One test per Pg store type ‚Äî assigns `&PgXxxStore{}` to the corresponding interface variable. The compiler rejects the file if any method is missing or has the wrong signature. Zero-value struct literals work from the `store_test` package because all PG store fields are unexported.

Tests:
- `TestPgInstrumentStore_ImplementsInterface` ‚Üí `InstrumentStore`
- `TestPgOrderStore_ImplementsInterface` ‚Üí `OrderStore`
- `TestPgTradeStore_ImplementsInterface` ‚Üí `TradeStore`
- `TestPgPositionStore_ImplementsInterface` ‚Üí `PositionStore`
- `TestPgMarketStore_ImplementsInterface` ‚Üí `MarketStore`
- `TestPgSegmentStore_ImplementsInterface` ‚Üí `SegmentStore`
- `TestPgFirmStore_ImplementsInterface` ‚Üí `FirmStore`
- `TestPgParticipantStore_ImplementsInterface` ‚Üí `ParticipantStore`
- `TestPgSettlementStore_ImplementsInterface` ‚Üí `SettlementStore`
- `TestPgAuditStore_ImplementsInterface` ‚Üí `AuditStore`

### 2. `TestNewPool_InvalidURL` (1 test)
Calls `db.NewPool("invalid://url")` and asserts a non-nil error is returned. `NewPool` issues a `db.Ping()` which fails immediately on an invalid/unreachable URL, so no real Postgres is required.

### 3. `TestPgStoreConstructors` (1 test, 10 sub-tests)
Passes a nil `*sql.DB` to each `New*` constructor and verifies:
- The returned pointer is non-nil.
- The returned concrete type satisfies the interface (runtime assignment check).

Nil `*sql.DB` is safe here because constructors only store the pointer without dereferencing it.

## Root Cause

The tenant selector UI was never added to `TopBar.tsx`. The `TenantContext` and `useTenant` hook
existed and worked correctly, and the securities pages (Instruments, Orders, Positions) correctly
consumed `currentTenant` for titles and conditional rendering. However:

- TopBar had no `useTenant` import and rendered no `<select>` element
- DashboardHome had no tenant banner and no `useTenant` import
- No toast was fired on tenant switch (the helper `buildTenantSwitchToast` existed only in tests)
- The `TenantContextType` had no `fetchError` field ‚Äî errors were silently swallowed
