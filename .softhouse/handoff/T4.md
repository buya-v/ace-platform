## Root Cause

The tenant selector UI was never added to `TopBar.tsx`. The `TenantContext` and `useTenant` hook
existed and worked correctly, and the securities pages (Instruments, Orders, Positions) correctly
consumed `currentTenant` for titles and conditional rendering. However:

- TopBar had no `useTenant` import and rendered no `<select>` element
- DashboardHome had no tenant banner and no `useTenant` import
- No toast was fired on tenant switch (the helper `buildTenantSwitchToast` existed only in tests)
- The `TenantContextType` had no `fetchError` field — errors were silently swallowed

---

## Fixes Applied

### 1. `src/admin-ui/src/components/TopBar.tsx`

Added tenant selector between breadcrumb and right section:

- Imports `useTenant` and `useToast`
- Renders `<label htmlFor="topbar-tenant-select" className={styles.srOnly}>Active tenant</label>` — visually hidden but accessible
- Renders `<select id="topbar-tenant-select" aria-label="Select active tenant" aria-busy={tenantLoading} data-testid="tenant-select">`
- Shows `"Loading tenants…"` option while `isLoading === true`
- Shows `"Error loading tenants"` option when `fetchError` is set
- Shows `"No tenants available"` option when 0 tenants
- `disabled` when loading, error, or 0 tenants
- `onChange` handler calls `setCurrentTenant(id)` and fires `showToast(`Switched to ${tenant.name}`, 'info')`

### 2. `src/admin-ui/src/components/TopBar.module.css`

Added `.tenantSelector`, `.tenantSelect`, and `.srOnly` styles:

- `.tenantSelect`: `max-width: 180px`, custom SVG chevron, focus ring with `box-shadow`

### 3. `src/admin-ui/src/contexts/TenantContext.tsx`

- Added `fetchError: string | null` to `TenantContextType`

- `fetchError` passed through `TenantContext.Provider`

### 4. `src/admin-ui/src/pages/DashboardHome.tsx`
