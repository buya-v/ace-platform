## Fixes Made

### Fix 1 — Error state: SecuritiesOrders instrument fetch (Item 3)

**File:** `src/admin-ui/src/pages/SecuritiesOrders.tsx`

**Problem:** `fetchSecuritiesInstruments` in `useEffect` had a `.catch(() => { // Silently handle })` — user received no feedback if the API call failed, leaving the instrument selector empty with no explanation.

**Fix:** Changed to expose the error via `showToast`:
```ts
.catch((err) => {
  if (err?.name === 'AbortError') return;
  showToast(err instanceof Error ? err.message : 'Failed to load instruments', 'error');
});
```

### Fix 2 — Responsive CSS: All 3 module files (Item 4)

**Files:**
- `src/admin-ui/src/pages/SecuritiesInstruments.module.css`
- `src/admin-ui/src/pages/SecuritiesOrders.module.css`
- `src/admin-ui/src/pages/SecuritiesPositions.module.css`

**Problem:** No `@media` queries existed in any of the three CSS files. On narrow viewports, the toolbar would overflow, modals would be clipped, and tables would not scroll.

**Fixes applied:**
- Instruments: toolbar `flex-wrap`, full-width `select`/`createBtn`, modal as bottom-sheet, `actionBtns` wraps
- Orders: header wraps, `submitBtn` full-width, selector wraps, `section` gets `overflow-x: auto`, trades table sets `min-width: 420px`, modal as bottom-sheet
- Positions: `section` gets `overflow-x: auto`

### Fix 3 — Accessibility: Toolbar inputs (Item 6)

**File:** `src/admin-ui/src/pages/SecuritiesInstruments.tsx` + `.module.css`

**Problem:** The search `<input>` and asset class `<select>` in the toolbar had no `<label>` association — screen readers would only read the placeholder text.

**Fix:**
- Added `id="inst-search"` to input, with a visually-hidden `<label htmlFor="inst-search">` before it
- Added `id="inst-asset-class"` to select, with a visually-hidden `<label htmlFor>` and `aria-label` fallback
- Added `.srOnly` utility class to CSS module (standard screen-reader-only pattern)

## Decisions Made

1. **Cancel modal toast omitted intentionally.** The checklist mentions "cancel actions show toast" — interpreted as cancelling an active operation (cancel order button), not dismissing a create form. Showing a toast on every modal close would be excessive noise. `cancelSecuritiesOrder()` (the real cancel action) already shows a toast.

2. **AbortError guard in instruments fetch.** The `useEffect` cleanup aborts the fetch on unmount. Without the guard, every navigation away from the page would flash an error toast.

3. **Bottom-sheet modal pattern on mobile.** Chose `position: fixed; bottom: 0` with `border-radius` only on top corners for modals on narrow viewports — this is the standard mobile pattern and avoids the modal being clipped or requiring horizontal scroll.
