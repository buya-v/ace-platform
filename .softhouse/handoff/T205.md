# T205 — Admin UI Floating Bot Button + Chat Panel

## Status: DONE

## Summary
Added a floating bot button and chat panel to the admin-ui for interacting with the GarudaX bot. The implementation follows existing patterns (CSS Modules, Context + useReducer, pure function extraction for testability).

## Files Created
- `src/admin-ui/src/services/botApi.ts` — Bot API client (sendBotMessage, getBotSuggestions, createTicket)
- `src/admin-ui/src/contexts/BotContext.tsx` — BotProvider with reducer, useBot hook, pure utility functions
- `src/admin-ui/src/components/BotButton.tsx` + `BotButton.module.css` — Fixed-position circular button with unread badge + pulse animation
- `src/admin-ui/src/components/BotChatPanel.tsx` + `BotChatPanel.module.css` — Chat panel with messages, typing indicator, suggestions, shortcuts
- `src/admin-ui/src/components/BotTicketForm.tsx` + `BotTicketForm.module.css` — Inline ticket form (Report Bug / Request Feature)
- `src/admin-ui/src/__tests__/bot-ui.test.ts` — 49 tests covering all pure functions

## Files Modified
- `src/admin-ui/src/pages/DashboardLayout.tsx` — Wrapped with BotProvider, added BotButton + BotChatPanel

## Decisions Made
- BotProvider placed inside KPIProvider but wrapping DashboardInner, so bot has access to current route via `useLocation()`
- Bot button hidden when chat panel is open (avoids visual overlap)
- Ticket form renders inline within chat panel (not a separate modal)
- All business logic extracted as pure exported functions for Vitest testing without RTL
- z-index 1100 chosen to match existing floating UI (Toast is 9999, so bot sits below toasts)

## Test Results
- 49 new tests added (bot-ui.test.ts)
- 254 existing tests still pass
- Total: 303 tests passing
