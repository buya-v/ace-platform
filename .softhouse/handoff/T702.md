## Decisions made
- Used `strings.ToUpper(id[:1]) + id[1:]` instead of deprecated `strings.Title()`
  to avoid any external dependency.
- `create instrument` regex requires the full pattern
  (`<ID> <commodity> <month> <year> contract <size> tick <tick>`) to be
  unambiguous and not collide with `update instrument`.
- Set tier supports two natural-language forms (`set tier X for Y` and
  `set Y tier to X`) since both are plausible user inputs.
- `batch screen` / `screen all` placed before the broader `containsAny` handlers
  to ensure "screen all" doesn't fall through to the participants handler.
- `list commodities` / `show commodities` placed before the generic `instrument/commodity/contract`
  containsAny handler that would otherwise intercept "commodities".
