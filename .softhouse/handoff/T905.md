# T905 Handoff — Playwright UAT: Bot NLP + Master Data

## Status
DONE

## Summary
Created `tests/playwright/uat/bot-nlp.spec.ts` with 12 Playwright UAT tests
covering two areas:

1. **Master data visibility** — verifies that gateway `SeedDefaults` data
   (6 commodities, 6 instruments, 1 default fee schedule with 5 rules) is
   returned by bot responses in human-readable form.

2. **NL variation coverage** — verifies the `normalizeMessage()` filler-word
   stripper and `IsCRUDCommand()` prefix matcher handle common alternative
   phrasings without falling through to the "unknown command" handler.

## Tests produced (12 total)

### Master Data (3)
| # | Command | Assertions |
|---|---------|------------|
| 1 | `show commodities` | Mentions wheat/corn/barley/etc, not empty JSON, not raw `{` |
| 2 | `show instruments` | Mentions WHT-HRW/CRN-YEL/2026 IDs, not empty, not raw JSON |
| 3 | `show fees` | Mentions fee/schedule/bps/farmer/speculator |

### NL Variations (5)
| # | NL phrase | Expected resolution |
|---|-----------|---------------------|
| 4 | `list of commodities` | commodity list handler |
| 5 | `show me the instruments` | instruments list handler (filler words stripped) |
| 6 | `what are the margin calls` | margin calls handler |
| 7 | `check system health` | system health handler |
| 8 | `any alerts?` | compliance alerts handler |

### Response Quality (4)
| # | What is checked |
|---|-----------------|
| 9 | `show commodities` — reply does not start with `{` or `[` |
| 10 | `show margin calls` — reply contains no `<nil>`, `undefined`, `null null` |
| 11 | `show instruments` — reply does not contain `"data": []` or `"instruments": []` |
| 12 | `help` — reply is prose, mentions command categories, not raw JSON |

## Design Decisions
- All assertions use `expect.soft()` — tests accumulate failures rather than
  aborting on the first mismatch. This matches the convention in all other UAT
  spec files.
- `isPortalAvailable()` guard in `beforeEach` enables graceful skip in CI
  environments where the admin portal is not deployed.
- The spec does NOT create or mutate state — all tests are read-only queries
  so they are safe to run repeatedly without side effects.
- Screenshots captured on the three master-data tests and the health check for
  visual evidence in UAT runs.
