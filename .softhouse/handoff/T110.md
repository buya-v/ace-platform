## Decisions Made

1. **Used Store interface pattern** â€” allows mock-based testing without PostgreSQL. PgStore is the production implementation; tests use mockStore.
2. **Reference data routes registered after existing routes** â€” avoids conflicts with existing `/api/v1/instruments/list` and `/api/v1/instruments/{instrument_id}/book` routes since router matches in registration order.
3. **Graceful degradation** â€” if `DATABASE_URL` is not configured, reference data endpoints are simply not registered. The gateway continues to function normally.
4. **DECIMAL columns cast to TEXT in SQL** â€” avoids floating-point precision loss when serializing contract sizes and tick sizes to JSON.
5. **Commodity fetched with separate query in GetInstrument** â€” simpler than a JOIN scan with mixed nullable columns.
