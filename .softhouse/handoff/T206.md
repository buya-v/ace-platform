## Decisions Made
- Used lowercase priority/status values (critical/high/medium/low, open/in_progress/resolved/closed) to match typical REST API conventions
- Priority colors: critical=red, high=orange, medium=yellow, low=blue (matching Surveillance severity pattern)
- Status colors: open=blue, in_progress=yellow, resolved=green, closed=gray
- Inline row expansion for ticket detail (no modal) — consistent with table-based pages
- 30-second polling interval for auto-refresh via usePolling hook
- Bot comments shown with robot emoji prefix (is_bot=true)
- Extracted 10 pure functions for testability: priorityClassName, ticketStatusClassName, categoryLabel, priorityLabel, statusLabel, formatTicketTime, truncateId, filterTickets, sortTickets, computeStatusCounts
