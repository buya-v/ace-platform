# T903 — Bot Response Formatting — Handoff

## Summary

Replaced all raw JSON dumps in the bot executor with human-readable formatted text. Every list handler now produces structured, numbered output with counts and empty-state messages.

## Changes Made

### `src/gateway/internal/bot/executor.go`

1. **`safeStr(v interface{}) string`** — new helper that returns `"0"` for `nil` instead of `<nil>`, preventing broken margin output.

2. **`formatMarginResponse`** — switched `%v` format verbs to `safeStr()` calls, fixing nil-value display.

3. **`formatSettlementResponse`** — completely rewritten. Parses `data[]` or `cycles[]` arrays and formats as a numbered list: `#<id> — Status: <STATUS>, Date: <YYYY-MM-DD>`. Empty state: `"⚖️ No settlement cycles found."`.

4. **`formatAlertsResponse`** — completely rewritten. Parses `data[]` or `alerts[]`, counts by severity (CRITICAL/HIGH/MEDIUM/LOW), and formats as:
   ```
   🔍 Compliance Alerts: 5 total
     CRITICAL: 1, HIGH: 2, MEDIUM: 2
   ```
   Empty state: `"🔍 No active compliance alerts."`.

5. **`formatParticipantsResponse`** — completely rewritten. Parses `data[]`, `applications[]`, or `participants[]` keys, shows `email — Status: STATUS, Type: type` per row. Empty state: `"👥 No participants found."`.

6. **`formatInstrumentsResponse`** — completely rewritten. Parses `data[]` or `instruments[]` keys, shows numbered `ID — Name [STATUS]` rows (status omitted when `active`). Empty state with create-command hint.

7. **`formatCommoditiesResponse`** — new function. Parses `data[]` or `commodities[]` keys, shows numbered `ID — Name (category, unit)` rows. Empty state with create-command hint.

8. **List commodities handler** — changed from `prettyJSON(respBody)` to `formatCommoditiesResponse(respBody)`.

### `src/gateway/internal/bot/executor_test.go`

- Updated `TestExecutor_ShowInstruments` assertion from `"instrument"` substring check to `"WHT-HRW-2026M07-UB"` (ID check) — the old assertion was case-sensitive and failed against the capitalized "Instruments" header.
- Updated `TestExecutor_ShowAlertsEmpty` assertion from `"0"` (no longer in the new empty-state message) to checking for `"🔍"` emoji and `"no active"` substring.
- Added 18 new unit tests covering all formatting helpers: `TestFormatInstrumentsResponse_WithData`, `_Empty`, `_InstrumentsKey`, `TestFormatCommoditiesResponse_WithData`, `_Empty`, `TestFormatParticipantsResponse_WithData`, `_Empty`, `TestFormatTicketsResponse_WithData`, `_Empty`, `TestFormatAlertsResponse_WithSeverityBreakdown`, `TestFormatMarginResponse_NilValues`, `_WithValues`, `TestSafeStr_Nil`, `TestSafeStr_Value`, `TestListCommoditiesCommand`.

## Test Results

- `go build ./...` — clean, no errors
- `go test ./internal/bot/` — 295 tests, all PASS

## Decisions Made

- Accepted duplication of local `type` structs inside each `format*` function to keep parsing hermetic without introducing package-level types.
- Settlement cycles truncate timestamps to `YYYY-MM-DD` (first 10 chars) for readability.
- Ticket IDs truncated to 8 chars in display (`#abc12345`).
- Alerts severity breakdown uses a fixed priority order: CRITICAL > HIGH > MEDIUM > LOW > UNKNOWN.
- `formatInstrumentsResponse` omits `[STATUS]` suffix for `active` instruments (reduces noise), but shows it for `suspended`, `halted`, etc.

## Blockers Found

None.

Consider adding `formatWarehouseResponse`, `formatPositionsResponse`, `formatNettingResponse`, `formatFeesResponse` in a follow-up task.

- Market data responses (candles, trades, tickers, order book) return raw JSON — these would benefit from tabular formatting.
