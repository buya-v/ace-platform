# T701 — Admin CRUD APIs for instruments, commodities, fee rules, facilities

## Status: complete

## Summary

Added missing admin write (CRUD) endpoints to the gateway for reference data and fee management. All endpoints enforce `admin` or `exchange_admin` role via `middleware.ClaimsFromContext`. When `DATABASE_URL` is not set (nil `*sql.DB`), all write operations fall back to an in-process session store (Go maps protected by `sync.RWMutex`), so the bot always receives a success response in development.

## Deliverables

### `src/gateway/internal/refdata/store.go`
- Added `InstrumentInput` and `CommodityInput` input types.
- Extended `Store` interface with three new methods:
  - `CreateInstrument(ctx, InstrumentInput) (*Instrument, error)`
  - `UpdateInstrument(ctx, id string, updates map[string]interface{}) (*Instrument, error)`
  - `CreateCommodity(ctx, CommodityInput) (*Commodity, error)`
- Implemented all three on `PgStore` with SQL INSERT/UPDATE.
- Added helper `GetInstrumentByID` (used by `UpdateInstrument` to return the updated row).
- Added `inMemoryStore` fallback (process-scoped, `sync.RWMutex`-protected) invoked when `db == nil`.

### `src/gateway/internal/refdata/handlers.go`
- Added `RegisterAdminRoutes(rt)` registering:
  - `POST /api/v1/admin/instruments` → `CreateInstrument`
  - `PUT  /api/v1/admin/instruments/{id}` → `UpdateInstrument`
  - `POST /api/v1/admin/commodities`  → `CreateCommodity`
- Each handler validates required fields, auto-generates IDs via `crypto/rand` when absent, and checks `claims.HasAnyRole("admin", "exchange_admin")`.

### `src/gateway/internal/fees/store.go`
- Added `FeeScheduleInput` and `FeeRuleUpdate` types.
- Added `inMemoryFeeStore` fallback.
- Extended `Store` interface with:
  - `CreateSchedule(ctx, FeeScheduleInput) (*FeeSchedule, error)`
  - `UpdateRule(ctx, id string, FeeRuleUpdate) (*FeeRule, error)`
  - `SetParticipantTier(ctx, participantID, tier string) error`
- Implemented all three on `PgStore` with SQL; upsert for participant tiers.

### `src/gateway/internal/fees/handlers.go`
- Added `RegisterAdminRoutes(rt)` registering:
  - `POST /api/v1/admin/fees/schedules`          → `CreateSchedule`
  - `PUT  /api/v1/admin/fees/rules/{id}`          → `UpdateFeeRule`
  - `PUT  /api/v1/admin/fees/tiers/{participant_id}` → `SetParticipantTier`
- Role check (`admin` or `exchange_admin`) in every admin handler.

### `src/gateway/cmd/gateway/main.go`
- Imported `refdata` and `fees` packages.
- Instantiated `PgStore` with `nil` db (in-memory fallback), registered both public and admin routes for both packages:
  ```go
  refdataHandlers.RegisterRoutes(rt)
  refdataHandlers.RegisterAdminRoutes(rt)
  feesHandlers.RegisterRoutes(rt)
  feesHandlers.RegisterAdminRoutes(rt)
  ```

## Decisions made
- **In-memory fallback over nil checks**: Added `db == nil` guards on all PgStore write methods rather than a separate no-op store type — keeps the interface boundary clean and the bot always gets a 201/200 response.
- **Dynamic UPDATE via allowed-list**: `UpdateInstrument` and `UpdateRule` build SET clauses from an explicit allowed-field list (not user-controlled column names) to prevent SQL injection while still being flexible.
- **Auto-generate IDs with prefix**: `INST-<hex>`, `COMM-<hex>`, `SCHED-<hex>` — clearly namespaced, 8 bytes of entropy, hex-encoded.
- **No duplicate `writeJSON`/`writeError`**: Existing helpers in each package reused as-is.
