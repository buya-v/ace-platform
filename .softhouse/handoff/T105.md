# T105 — Margin Engine PostgreSQL Persistence + Real Collateral Source

## Summary

Replaced margin-engine in-memory stores with PostgreSQL persistence and implemented a real HTTP-based collateral source that queries the clearing-engine.

## Artifacts Created

### 1. Migration: `infrastructure/db/migrations/V13__margin_engine_tables.sql`
- Creates `margin` schema with 3 tables:
  - `margin.portfolio_margins` — snapshot of each margin calculation per participant
  - `margin.margin_calls` — margin call lifecycle tracking (ISSUED/SATISFIED/BREACHED)
  - `margin.margin_parameters` — per-instrument margin percentages
- Indexes on `participant_id`, `status`, and `calculated_at`

### 2. PostgreSQL Store: `src/margin-engine/internal/store/postgres.go`
- `PortfolioMarginStore` interface + `PostgresPortfolioStore` implementation
- `MarginCallStore` interface + `PostgresMarginCallStore` implementation
- `OpenDB` helper using `pgx/v5/stdlib` driver with connection pooling
- Follows the same pattern as clearing-engine's postgres.go

### 3. HTTP Collateral Source: `src/margin-engine/internal/collateral/http_source.go`
- `HTTPCollateralSource` implements `engine.CollateralSource`
- Makes HTTP GET to `clearing-engine:8082/positions?participant_id=X`
- Calculates collateral as `sum(|net_qty * avg_entry_price|)` — simplified proxy
- Falls back to zero on any error (timeout, non-200, JSON decode failure)
- 5-second HTTP timeout to prevent blocking margin calculations

### 4. Updated `cmd/margin-engine/main.go`
- PostgreSQL persistence when `DB_HOST` env var is set (with `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_SSLMODE`)
- HTTP collateral source when `CLEARING_ENGINE_ADDR` env var is set
- Falls back to in-memory mode for both when env vars are unset
- Margin handler persists `PortfolioMargin` to PostgreSQL
- Margin call handler persists `MarginCall` to PostgreSQL

### 5. Tests
- `internal/store/postgres_test.go` — utility function tests + interface conformance (11.4% — PostgreSQL methods need live DB)
- `internal/collateral/http_source_test.go` — 11 tests, 100% coverage: success, error, unreachable, invalid JSON, empty, multi-position, zero quantity, invalid price

## Decisions Made

- **Collateral formula**: `sum(|net_qty * avg_entry_price|)` — simplified from true collateral accounting. This gives a proportional proxy of position value without needing a separate collateral/deposit ledger.
- **Best-effort collateral**: Returns zero when clearing-engine is down. This means margin calculations proceed (conservatively — zero collateral triggers margin calls) rather than blocking.
- **No position store**: The margin-engine doesn't store positions directly — it receives them via `CalculateMargin()` calls. Only the computed margins and margin calls are persisted.
- **go.mod**: Added `github.com/jackc/pgx/v5` dependency, matching clearing-engine's version.
