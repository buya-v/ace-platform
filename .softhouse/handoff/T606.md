## Design Decisions

### `toHaveScreenshot()` vs `page.screenshot()`
`toHaveScreenshot()` is Playwright's built-in visual comparison tool:
- **First run** (`--update-snapshots`): saves `.png` baseline files to
  `tests/playwright/uat/visual-regression.spec.ts-snapshots/`
- **Subsequent runs**: diffs against stored baselines and fails if pixel diff
  exceeds the threshold
- No external image-diff library required

### Diff thresholds
- Desktop: `maxDiffPixelRatio: 0.03` (3%) — tight tolerance for 1080p layouts
- Tablet: `maxDiffPixelRatio: 0.05` (5%) — wider tolerance for responsive
  reflows that vary slightly between runs (e.g., flexbox rounding)

### Monitoring test has `timeout: 15000`
The monitoring/dashboard page makes multiple service health API calls on load.
An explicit `timeout: 15000` (15s) is set on the first test only because it has
the highest chance of being the "cold" first navigation after login and may
take slightly longer than the 10s global `expect.timeout` in `playwright.config.ts`.

### SPA navigation pattern preserved
All tests use `navigateTo(page, section)` from `helpers.ts` (sidebar click
navigation), not `page.goto()`, to preserve in-memory React auth tokens — per
the validated pattern from T067 Playwright tests.

### Skip guard pattern
`beforeEach` calls `isPortalAvailable(page)` and uses `test.skip()` when the
portal is unreachable. This matches the graceful-skip pattern used across all
UAT tests (T055, T067) — zero failures when the portal is down, tests simply
skip.

### `waitForData()` before every screenshot
All tests call `waitForData(page)` before `toHaveScreenshot()` to wait for
loading spinners and skeleton placeholders to disappear. This prevents
screenshots of partial/loading states, which would cause false-positive diff
failures on subsequent runs.

## Suggested Follow-ups
- **Run `--update-snapshots` once** against the live portal to create the
  initial `.png` baseline files before adding to CI:
  ```
  cd tests/playwright
  npx playwright test uat/visual-regression.spec.ts --update-snapshots
  ```
- **Commit the snapshot files** to git so CI has baseline images to compare
  against. Snapshots land in:
  `tests/playwright/uat/visual-regression.spec.ts-snapshots/`
- **Consider adding more tablet pages** (circuit-breakers, settlement) if
  responsive layout regressions are a concern
- **Freeze dynamic content** before screenshots: consider masking timestamps,
  chart candles, or live price tickers with `mask` option if these cause
  spurious diffs:
  ```typescript
  await expect(page).toHaveScreenshot('...', {
    mask: [page.locator('[class*="timestamp"], [class*="price"]')],
  });
  ```
