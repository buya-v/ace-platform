# T103 — Matching Engine PostgreSQL Persistence

## Status: DONE

## Summary

Added PostgreSQL persistence to the matching-engine for trades and execution reports. The order book remains in-memory for maximum matching performance. All writes to PostgreSQL are async via a buffered channel and background batch-insert goroutine.

## Deliverables

### 1. Migration: `infrastructure/db/migrations/V11__matching_engine_tables.sql`
- `exchange.orders` — order state for audit/recovery (VARCHAR(64) PKs, DECIMAL(18,4) prices/quantities)
- `exchange.trades` — append-only matched trade records with UPDATE/DELETE rules blocked
- `exchange.matching_execution_reports` — append-only execution reports with UPDATE/DELETE rules blocked
- Indexes on instrument_id, account_id, traded_at, order_id, trade_id
- Conditional GRANT to `garudax_exchange_svc` role

### 2. Async Trade Writer: `src/matching-engine/internal/store/postgres.go`
- `PostgresTradeStore` implements the existing `TradeStore` interface
- Uses a buffered channel (`writeItem`) to decouple matching from DB writes
- Background goroutine batch-inserts trades and execution reports
- Configurable buffer size (default 10,000), batch size (default 100), flush timeout (default 100ms)
- In-memory `InMemoryTradeStore` backs all read operations for consistency
- `AppendExecutionReport()` method for execution report persistence
- `ConnectPostgres()` and `BuildDSN()` helpers for connection setup
- `Close()` for graceful shutdown (flushes pending writes)
- Buffer-full handling: logs warning and drops async write (in-memory copy preserved)

### 3. Tests: `src/matching-engine/internal/store/postgres_test.go`
- Interface compliance check
- In-memory read path tests (Trades, LastTrade, TradesBySequence)
- Append queues to write channel verification
- Execution report queueing
- Buffer-full graceful degradation
- Concurrent append thread safety (10 goroutines x 50 trades)
- DSN building with defaults
- ExecType string conversion
- NullableString helper

### 4. Main.go update: `src/matching-engine/cmd/matching-engine/main.go`
- If `DB_HOST` env var is set, connects to PostgreSQL and uses `PostgresTradeStore`
- Otherwise falls back to `InMemoryTradeStore` (existing behavior preserved)
- Execution reports are persisted to PostgreSQL when pgStore is active
- Graceful shutdown flushes pending writes before exit
- Env vars: `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `DB_SSLMODE`

## Decisions Made
- Used `github.com/jackc/pgx/v5/stdlib` as the PostgreSQL driver (as specified in task)
- go.mod bumped to go 1.25.0 (required by pgx v5.9.1)
- Named the execution reports table `exchange.matching_execution_reports` to avoid conflict with existing `exchange.execution_reports` from V6 migration
- Batch insert uses `ON CONFLICT (id) DO NOTHING` for idempotency
- 5-second context timeout on batch inserts to prevent hanging
