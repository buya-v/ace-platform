## Decisions Made

1. **No @types/node dependency** — Used ambient `declare` statements for `process`, `__dirname`, `require` to avoid adding @types/node to a project that doesn't have it. TypeScript compiles clean with zero errors.

2. **getBodyText() helper in page-checks** — Named separately to avoid variable shadowing (each checker function declares `const bodyText`).

3. **Soft assertions throughout** — All `expect.soft()` used so a failing check doesn't abort the test suite. Appropriate for UAT where partial validation is useful.

4. **SPA navigation via sidebar** — All `navigateTo()` clicks sidebar links instead of `page.goto()` to preserve the admin-ui's in-memory React auth tokens (same pattern as admin-verification.spec.ts).

5. **Graceful timeouts** — All helpers catch errors and warn rather than throw. `isPortalAvailable()` enables the skip pattern for running tests without infrastructure.

6. **ADMIN_BASE_URL env var** — Playwright UAT project uses `process.env.ADMIN_BASE_URL || 'https://admin.garudax.asla.mn'` for portability.
