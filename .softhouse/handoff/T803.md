## Summary
Created `tests/playwright/uat/bot-guided.spec.ts` with 8 Playwright UAT tests that verify the GarudaX Bot correctly guides users through CRUD operations when they provide incomplete commands, and that fully-formed / list commands still execute normally.

## Tests Created

### File: `tests/playwright/uat/bot-guided.spec.ts`

| # | Test name | What it verifies |
|---|-----------|-----------------|
| 1 | `create new instrument — guides with format` | Incomplete instrument create returns format guide, NOT instrument list |
| 2 | `create commodity — guides with format` | Incomplete commodity create returns category/unit guidance |
| 3 | `add fee — shows fee management options` | Bare "add fee" returns fee schedule/rule/tier menu |
| 4 | `new receipt — guides receipt creation` | "new receipt" returns issue-receipt format with holder/quantity |
| 5 | `create order — guides order placement` | "create order" returns buy/sell syntax, no INTERNAL_ERROR |
| 6 | `show instruments — still lists (not guided)` | List command returns data, not a format guide |
| 7 | `create commodity testuat grain kg — executes (not guided)` | Fully-formed command executes (✅/❌), not guided |
| 8 | `unknown command — shows helpful suggestions` | "xyzzy" returns fallback help, no INTERNAL_ERROR |

## Decisions Made

- Used `test.skip(!(await isPortalAvailable(page)), ...)` in `beforeEach` for graceful CI skip (matches pattern from `bot-agent.spec.ts`)
- All assertions use `expect.soft()` so a single failed check does not abort the test
- Screenshots captured for tests 1, 2, 8 (most visually notable guided prompt scenarios)
- Negative assertion on tests 1 & 2 confirms guided prompts do NOT fall through to the instrument list handler (`📊.*Active instruments` pattern)
- Test 7 (fully-formed create) uses a `testuat` commodity ID so it won't collide with production data but will still exercise the CRUD execution path
- Assertions are anchored to the executor's actual guided-prompt reply text (lines 653-716 of `executor.go`)
