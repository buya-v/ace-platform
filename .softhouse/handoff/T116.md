# T116 — Real Kafka Event Streaming: Replace Channel Stubs with Wire Protocol

## Status: DONE

## Summary

Added real Kafka producer and consumer implementations using `github.com/segmentio/kafka-go` v0.4.47 to all 9 Go services. The existing `Producer` and `Consumer` interfaces remain unchanged -- the new `KafkaProducer` and `KafkaConsumer` types implement them using real Kafka wire protocol (kafka-go `Writer` and `Reader` respectively).

## What Was Done

### New Files (per service, in `internal/kafka/`):
- **`kafka_writer.go`** -- `KafkaProducer` struct implementing `Producer` interface via kafka-go `Writer`. Features: connection pooling, hash-based partitioning, configurable retries/timeouts, graceful shutdown with flush on Close.
- **`kafka_reader.go`** -- `KafkaConsumer` struct implementing `Consumer` interface via kafka-go `Reader` (consumer group mode). Features: per-topic goroutine consumption, idempotency tracking, retry with exponential backoff, DLQ forwarding, commit-after-process.
- **`kafka_writer_test.go`** -- Tests for KafkaProducer lifecycle (create, close idempotent, publish-after-close error) and `NewProducerFromEnv` factory (real producer when KAFKA_BROKERS set, channel producer when not).
- **`kafka_reader_test.go`** -- Tests for KafkaConsumer lifecycle (create, close idempotent, subscribe, start-with-no-handlers).

### Modified Files (per service):
- **`wiring.go`** -- Added `NewProducerFromEnv()` factory function that reads `KAFKA_BROKERS` env var. When set and non-empty, creates `KafkaProducer`; otherwise falls back to existing `ChannelProducer`. For market-data-service and gateway, also added `NewConsumerFromEnv()`.
- **`go.mod`** -- Added `github.com/segmentio/kafka-go v0.4.47` dependency (+ transitive deps: `klauspost/compress`, `pierrec/lz4/v4`).

## Decisions Made

1. **Template duplication pattern** -- Each service gets its own copy of `kafka_writer.go` and `kafka_reader.go` (identical implementations). This follows the established zero-shared-dependency pattern from the Decimal type and original channel-based Kafka packages.

2. **Environment-driven factory** -- `NewProducerFromEnv()` / `NewConsumerFromEnv()` check `KAFKA_BROKERS` env var. Empty or unset = channel-based (backward compatible). Set = real Kafka. No code changes needed in existing tests or main.go.

3. **kafka-go Writer (not individual Produce calls)** -- Writer handles connection pooling, batching, and per-message topic routing. This is simpler and more performant than managing individual connections.

4. **kafka-go Reader in consumer-group mode** -- Uses `GroupID` for consumer group coordination, automatic partition assignment, and offset tracking.

5. **Gateway is consumer-only** -- No `NewProducerFromEnv` for gateway since it doesn't publish events. Only `NewConsumerFromEnv` for receiving margin calls and compliance status changes for WebSocket push.
