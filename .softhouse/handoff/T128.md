# T128 — Secrets Management: JWT Key Rotation, DB Credentials from Env/Vault

## Status: DONE

## Summary

Hardened secrets management across auth-service and gateway by:

1. **RS256 JWT Support** — auth-service can now sign tokens with RSA-SHA256 (asymmetric) in addition to the existing HS256 (symmetric). Gateway can validate both HS256 and RS256 tokens based on the `alg` header field.

2. **Key Rotation** — `KeySet` struct holds current + previous signing key. Validation tries current key first, falls back to previous. This enables zero-downtime key rotation (new tokens use new key, old tokens remain valid until expiry).

3. **Production Mode Guard** — `PRODUCTION_MODE=true` env var requires explicit `AUTH_JWT_SIGNING_KEY` or `JWT_RSA_PRIVATE_KEY_PATH`. In dev mode, a warning is logged when using the default dev secret.

4. **RSA Key File I/O** — Supports both PKCS#1 and PKCS#8 private key formats, PKIX public key format. Includes `WriteRSAKeyPairPEM` for dev/test key generation with 0600 permissions on private keys.

## Environment Variables Added

| Variable | Service | Description |
|---|---|---|
| `JWT_RSA_PRIVATE_KEY_PATH` | auth-service | Path to RSA private key PEM file (enables RS256) |
| `JWT_RSA_PUBLIC_KEY_PATH` | auth-service, gateway | Path to RSA public key PEM file |
| `PRODUCTION_MODE` | auth-service, gateway | If `true`, requires explicit secrets (no defaults) |

## Decisions Made

- RS256 and HS256 are dispatched based on the JWT header `alg` field, not configuration. This means the gateway can validate tokens from both HS256 and RS256 auth-service instances during a migration.
- KeySet keeps exactly 2 keys (current + previous). After a second rotation, the first key is lost. This is intentional — the rotation window should be shorter than the token TTL.
- `GenerateRSAKeyPair` uses 2048-bit keys, which is the minimum recommended for production RSA.
- DB credentials remain env-var based (no changes needed — they already use `DB_HOST`, `DB_PASSWORD`, etc. with no hardcoded defaults for sensitive values).
