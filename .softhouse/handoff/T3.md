## Decisions Made

### SecuritiesOrders page
- Used `usePolling(10000)` for orders, one-time `useEffect` for instruments (instruments don't change often)
- Instrument selector fetches on mount; first instrument auto-selected
- Modal form uses controlled state with inline validation via exported `validateOrderForm(form, orderType)`
- Price field disabled (not hidden) for MARKET orders — clearer UX than removing the field
- Recent Trades section: shows trades from `ordersResult.data.trades` if present; falls back to placeholder text — this is the correct approach since the API may embed trades in the order response
- Cancel button only shown for `PENDING` and `PARTIALLY_FILLED` orders, as specified
- DataGrid keyField: `_key` (synthetic, guaranteed unique)

### SecuritiesPositions page
- `usePolling(30000)` — 30s interval as specified
- `exportFilename="securities-positions"` wired to DataGrid for CSV export
- `unrealized_pnl` rendered with green (positive) / red (negative) based on numeric value
- `formatPnl(value)` exported as pure function — always shows sign prefix (`+250.00` or `-100.00`)
- `formatMoney(value)` exported as pure function — no sign prefix, 2 decimal places

### Test strategy
- Pure function extraction enables 100% test coverage without React Testing Library
- All business logic exported: `validateOrderForm`, `normalizeInstruments`, `normalizeOrders`, `formatOrderDate`, `formatPnl`, `formatMoney`, `normalizePositions`
- API integration tests verify correct endpoints and HTTP methods using mock fetch
