## Key Decisions

1. **Balance check gate**: Only activates when the from-account already has at least one balance record. Accounts with no records bypass the check (new-account semantics). This preserves backward compatibility with all existing CSD transfer tests.

2. **SSE test design**: Test reads only the initial "connected" event then closes the response body — avoids blocking the test goroutine waiting for a 15-second heartbeat.

3. **NodeStore ErrNotFound fix**: The store was silently returning `([], nil)` for unknown node IDs in `GetEffectivePermissions`. This caused the handler to return 200 with an empty permissions list instead of 404. Added a guard at the top of the function.

4. **CSV partial-fail test**: Uses an empty ticker cell (`,No Ticker Corp,...`) which passes the CSV parser but fails the `validateBulkItem` check, cleanly producing `failed=1`.

## Suggested Follow-ups

- **SSE fan-out**: Current SSE handler serves one client with no real events. Implement a broadcast channel (fed by the matching engine or a pub/sub store) so market events are streamed to all connected SSE clients.
- **Balance validation**: Add a `CheckBalance(accountID, instrumentID string, qty int) error` method to `CustodyBalanceStore` for a cleaner contract than the current list-and-scan approach.
- **Coverage improvement**: `internal/server` is at 54%. The node, SSE, and new CSD balance paths are covered; the remaining gap is in the many other handler files. A targeted coverage pass targeting 70%+ is recommended.
- **NodeStore.Update**: The PUT-permissions handler uses a type-assertion to `SetPermissions` on the concrete store. Consider promoting `SetPermissions` (or an `UpdatePermissions` method) into the `NodeStore` interface so it is testable without type assertions.
