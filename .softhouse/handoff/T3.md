## Summary

Two changes:

**Part A** wires corporate action processing through the CSD by updating custody balances when dividends and stock splits are processed. For DIVIDEND, each participant's custody balance is credited with the entitlement value. For STOCK_SPLIT, each participant's custody balance quantity is multiplied by the split ratio.

**Part B** adds accrued interest calculation for bond instruments during settlement. When `ProcessSettlementCycle` processes obligations, it checks if the instrument is a bond and calculates accrued interest based on the bond's day count convention (ACT/365, ACT/360, 30/360), adding it to the obligation's `NetAmount` (dirty price).

## Changes Per File

### `src/securities-service/internal/types/types.go`
- Added `AccruedInterest float64 json:"accrued_interest,omitempty"` field to `SettlementObligation` struct

### `src/securities-service/internal/store/store.go`
- Added `Update(obligation *types.SettlementObligation) error` to `SettlementStore` interface
- Added `Update` method to `InMemorySettlementStore` implementation

### `src/securities-service/internal/server/handlers_corporate_actions.go`
- `processDividend`: after creating each entitlement, finds the participant's custody account via `findCustodyAccountForParticipant` and calls `custodyBalanceStore.GetOrUpdate` to credit the entitlement value
- `processStockSplit`: after adjusting each position, retrieves the participant's custody balance and applies the split ratio delta via `custodyBalanceStore.GetOrUpdate`
- Added `findCustodyAccountForParticipant` helper: participant -> firm (via participantStore) -> first custody account (via custodyAccountStore.ListByFirm)
- All CSD operations are nil-guarded (custodyAccountStore, custodyBalanceStore, participantStore must be non-nil)

### `src/securities-service/internal/settlement/engine.go`
- Added `bondStore store.BondStore` field to `SettlementEngine` struct
- Added `SetBondStore(store.BondStore)` setter method
- In `ProcessSettlementCycle`: after affirmation (step 2), added step 2b that calculates accrued interest for bond instruments
- Added `calculateAccruedInterest` function implementing three day count conventions:
  - `ACT/365`: accrued = days * couponRate * parValue / 365
  - `ACT/360`: accrued = days * couponRate * parValue / 360
  - `30/360`: accrued = days * couponRate * parValue / 360
  - Uses 30 days as default since last coupon date (MVP simplification)
  - Result is multiplied by quantity

---

## Decisions Made

- CSD balance updates are best-effort (errors silently ignored) to avoid failing the entire corporate action if CSD is unavailable
- `findCustodyAccountForParticipant` uses the first custody account for the participant's firm; in production, account selection would be more sophisticated
- Accrued interest calculation uses `bond.CouponRate` directly as a fraction (e.g., 0.05 for 5%) per the type documentation
- Default 30 days since last coupon is an MVP simplification; a real implementation would track actual coupon dates
- The `SettlementStore.Update` method was added to support updating AccruedInterest and NetAmount on obligations
