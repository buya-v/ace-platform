## Design Decisions

1. **VARCHAR IDs instead of UUID**: The Go service generates string IDs like "app-1", "scr-2" via atomic counters. The V15 migration uses VARCHAR(64) primary keys to match, rather than forcing UUID conversion. Tables use `_v2` suffix to coexist with the V7 UUID-based tables during migration.

2. **UPSERT pattern**: All Save methods use `INSERT ... ON CONFLICT (id) DO UPDATE` for idempotent writes. Risk scores and SAR filings use `DO NOTHING` on conflict since they are append-only.

3. **Nullable columns**: Empty Go strings and zero `time.Time` values are stored as SQL NULL via `nullableString()` and `nullableTime()` helpers. This avoids storing empty strings where NULL is semantically correct.

4. **N+1 query pattern for List methods**: `ListApplications` and `ListAlerts` first query IDs, then call `GetApplication`/`GetAlert` per row. This trades query efficiency for code simplicity and ensures consistent field mapping. Acceptable for compliance workloads (low QPS).

5. **No pgxpool**: Task specified `database/sql` with `pgx/v5/stdlib` driver, not the pgxpool wrapper. The `OpenPostgres` function configures `sql.DB` with sensible pool defaults (10 max open, 2 idle, 30m lifetime).

6. **Driver registration via blank import**: `main.go` imports `_ "github.com/jackc/pgx/v5/stdlib"` to register the "pgx" driver with `database/sql`.

## Suggested Follow-ups

- **T108+**: Other services (matching, clearing, margin, settlement, etc.) should follow the same pattern: create `internal/store/postgres.go` implementing their store interfaces, and switch on `DB_HOST` in main.go.
- **Integration tests**: When a PostgreSQL instance is available, add integration tests that exercise the full store CRUD cycle against a real database.
- **V7 table deprecation**: Once all services use the V15 `_v2` tables, create a migration to drop the V7 UUID-based tables and rename `_v2` tables to remove the suffix.
- **Connection pooling tuning**: The current defaults (10 max conns) are conservative. For production, tune based on expected compliance workload.
