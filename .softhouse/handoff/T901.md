# T901 Handoff — Seed master data in gateway in-memory store on startup

## Status: Done

## Summary

Added idempotent startup seeding for refdata and fees in-memory stores so the gateway
is non-empty from the moment it starts, even when `DATABASE_URL` is not set.

## Changes

### `src/gateway/internal/refdata/store.go`
Added `SeedDefaults(ctx context.Context) error` on `*PgStore`:
- Calls `ListCommodities` — returns early (no-op) if count > 0
- Seeds 6 commodities: WHT-HRW, CRN-YEL, SBN-NO2, BRL-MALT, CSH-RAW, LVS-CATTLE
- Seeds 6 instruments: one per commodity, all PHYSICAL/MNT futures for 2026
- Works for both in-memory (db == nil) and PostgreSQL paths via existing `CreateCommodity`/`CreateInstrument`

### `src/gateway/internal/fees/store.go`
Added `SeedDefaults(ctx context.Context) error` on `*PgStore`:
- Calls `ListAllSchedules` — returns early if count > 0
- Seeds schedule `default` / "GarudaX Default Fee Schedule" / effective 2026-01-01 / ACTIVE
- Seeds 5 rules: farmer 10bps, hedger 15bps, speculator 25bps, market_maker 5bps, clearing 5bps

### `src/gateway/cmd/gateway/main.go`
After `feesHandlers.RegisterAdminRoutes(rt)`, added two seed calls:
```go
seedCtx := context.Background()
if err := refdataStore.SeedDefaults(seedCtx); err != nil { ... }
if err := feesStore.SeedDefaults(seedCtx); err != nil { ... }
```
Both log a warning on error (non-fatal) and log info on success.

## Key Design Decisions

- **Idempotent by count check**: both methods check `len(existing) > 0` before inserting,
  so a restart against a populated DB will not duplicate rows.
- **Seed via existing Store methods**: re-uses `CreateCommodity`, `CreateInstrument`,
  `CreateSchedule`, `CreateRule` — no raw SQL in the seed layer.
- **Non-fatal on error**: seed failures emit a `Warn` log but don't prevent the gateway
  from starting. This is intentional so a DB permission error at boot doesn't kill the server.
- **`json.Number` for contract/tick sizes**: consistent with `InstrumentInput` field types.
