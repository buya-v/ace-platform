# T202 — Ticket System Backend API (Gateway)

## Summary
Added a complete ticket/issue tracking system to the gateway service, following the existing handler + store pattern used by fees, reporting, and refdata packages.

## Artifacts Created

1. **Migration**: `infrastructure/db/migrations/V25__tickets.sql`
   - `tickets.tickets` table with category, priority, status, reporter_id, assignee_id, tags, metadata
   - `tickets.comments` table with ticket_id FK, author_id, is_bot flag
   - Indexes on status, reporter, category, priority, and comment ticket_id

2. **Package**: `src/gateway/internal/tickets/`
   - `store.go`: Store interface, InMemoryStore (for tests/fallback), PgStore (PostgreSQL), types (Ticket, Comment, TicketStats, ListFilters)
   - `handlers.go`: HTTP handlers for all 6 endpoints, validation helpers
   - `handlers_test.go`: 41 tests covering all endpoints, auth, access control, validation, and store edge cases

3. **Wiring**: `src/gateway/cmd/gateway/main.go`
   - Added `tickets` import and route registration with InMemoryStore default

## API Endpoints

| Method | Path | Access | Description |
|--------|------|--------|-------------|
| POST | /api/v1/tickets | Any authenticated user | Create ticket |
| GET | /api/v1/tickets | Authenticated (own tickets for users, all for admin) | List tickets with filters |
| GET | /api/v1/tickets/{id} | Owner or admin | Get ticket with comments |
| PATCH | /api/v1/tickets/{id} | Admin only | Update status/assignee/priority |
| POST | /api/v1/tickets/{id}/comments | Reporter or admin | Add comment |
| GET | /api/v1/admin/tickets/stats | Admin only | Aggregate stats |

## Decisions Made
- Followed the exact same pattern as `fees` and `reporting` packages (Store interface, PgStore, handlers with RegisterRoutes, writeJSON/writeError helpers)
- Used `middleware.ClaimsFromContext(r)` for user identity, falling back from ParticipantID to Sub
- Admin role check uses `HasAnyRole("admin", "exchange_admin")` matching the existing convention
- ID generation uses `crypto/rand` hex (32 chars) with prefix (TKT- or CMT-)
- InMemoryStore is the default; PgStore can be injected when DATABASE_URL is set
