# T108 — Market Data Service TimescaleDB Persistence

## Status: DONE

## Summary

Replaced market-data-service in-memory-only stores with a dual-mode architecture: in-memory (default) or TimescaleDB-backed when `DB_HOST` is set. All existing tests pass unchanged.

## Changes Made

### 1. Migration: `infrastructure/db/migrations/V16__market_data_tables.sql`
- Created `market_data.trades` table as a TimescaleDB hypertable (partitioned by `traded_at`)
- Created `market_data.candles` table with composite PK `(instrument_id, interval, bucket)`
- Created `market_data.tickers` table with single-row-per-instrument design for UPSERT
- Added indexes on `(instrument_id, traded_at DESC)` and `(instrument_id, sequence_number)` for trades

### 2. Store Interfaces: `internal/store/interfaces.go`
- Defined `TradeRepository`, `CandleRepository`, `TickerRepository` interfaces
- All existing in-memory stores (`TradeStore`, `CandleStore`) satisfy these interfaces
- New `TickerStore` (in-memory) also satisfies `TickerRepository`

### 3. PostgreSQL Implementations: `internal/store/postgres.go`
- `PGTradeStore`: Implements `TradeRepository` with `database/sql` + pgx driver
  - `Append()` uses `INSERT ... ON CONFLICT DO NOTHING` for idempotent writes
  - `LastN()` orders by `traded_at DESC, sequence_number DESC`
  - All query methods have 5-second context timeouts
- `PGCandleStore`: Implements `CandleRepository`
  - `Store()` uses `INSERT ... ON CONFLICT DO UPDATE` (upsert)
  - `Query()` returns candles ordered by bucket ASC
  - `DeleteBefore()` for retention policy enforcement
- `PGTickerStore`: Implements `TickerRepository`
  - `Upsert()` uses `INSERT ... ON CONFLICT DO UPDATE` with `updated_at = NOW()`
  - `GetAll()` supports filtering by instrument IDs or returning all
- `OpenDB()` helper: configures connection pool (25 max open, 5 idle, 5min lifetime)

### 4. In-Memory Ticker Store: `internal/store/tickerstore.go`
- New `TickerStore` for in-memory ticker persistence (used when no DB)
- Thread-safe with `sync.RWMutex`

### 5. Server Refactoring: `internal/server/server.go`
- Changed store fields from concrete types to interfaces (`TradeRepository`, `CandleRepository`, `TickerRepository`)
- Added `NewServerWithStores()` constructor accepting custom store implementations
- `NewServer()` now delegates to `NewServerWithStores()` with nil (in-memory defaults)
- `IngestTrade()` now also persists ticker snapshot after each trade

### 6. Retention Policy: `internal/retention/policy.go`
- Changed `Enforce()` parameter from `*store.CandleStore` to `CandleDeleter` interface
- Eliminates circular dependency between retention and store packages

### 7. Main.go: `cmd/market-data-service/main.go`
- Added DB_HOST detection: when set, opens PostgreSQL connection and uses PG stores
- Reads DB_PORT, DB_USER, DB_PASSWORD, DB_NAME, DB_SSLMODE from environment
- Falls back to in-memory stores when DB_HOST is not set

### 8. Driver Registration: `internal/store/pgdriver.go`
- Side-effect import of `github.com/jackc/pgx/v5/stdlib` to register the "pgx" driver

## Tests
- 49 total tests pass (all existing + 14 new)
- New tests: TickerStore (3), interface compliance compile-time checks (6 var assertions), interface integration tests (3), helper function tests (4)
- PG store methods are not unit-tested (require real database) but have compile-time interface checks

## Dependencies Added
- `github.com/jackc/pgx/v5 v5.7.4` (PostgreSQL driver)

## Decisions
- Candle aggregation stays in-memory (Builder), only flushed/closed candles persist to DB
- Ticker is computed by the in-memory Engine but persisted to DB after each trade ingestion
- Used `database/sql` interface (not pgx native) for easier testing and driver swapping
- In-memory stores remain the default; PostgreSQL is opt-in via DB_HOST
