### #1 — Gateway 404-before-auth (RouteChecker) — ✅ FIXED (verified live)
Direct evidence on the fresh gateway:
- `GET /api/v1/does-not-exist` (no auth, no tenant) → **404** `{"code":"NOT_FOUND"}` (was 401 under stale image).
- `GET /api/v1/orders` (no tenant) → **401** `TENANT_REQUIRED`.

- `TestGatewayHealth`, `TestMethodNotAllowed`, and the full `TestTenantEnforcement_*` suite (R011/R012) all PASS.

R011's `RouteChecker` returns 404 for unregistered paths *before* tenant enforcement, so the "reject-before-routing masks 404" bug does not recur at the tenant layer.

### #2 — Compliance KYC object response (array vs object) — ✅ FIXED (verified live)
`TestComplianceKYCFlow` PASS (submit / get / list / approve), `TestComplianceScreening` PASS. The KYC submit endpoint returns a JSON object that unmarshals into `map[string]interface{}` (the 2026-03-29 failure was "cannot unmarshal array into object"). Verified against the freshly-rebuilt `compliance-service`.

### #3 — Cross-service event propagation over real Kafka — ❌ NOT FIXED / NOT IMPLEMENTED
With a real broker running and `KAFKA_BROKERS=kafka:9092` injected into every engine:
- **No application Kafka topics exist** after the full e2e run (`kafka-topics --list` shows only `__*` internals).
- **No Kafka activity** in any engine log (no producer/consumer/broker lines).
- **Source confirms why:** no engine `cmd/*/main.go` references Kafka at all. The `internal/kafka/` packages (including the real `segmentio/kafka-go` `kafka_writer.go`/`kafka_reader.go`) are **never wired into the running binaries** — they are exercised only by unit tests. R007 added fail-fast to `NewProducerFromEnv`/`NewConsumerFromEnv`, but nothing in `main()` calls those constructors.
- `TestFullTradingLifecycle` **PASSES** but is vacuous for this purpose: the demo matching-engine only registers `WHT-HRW-2026M07-UB`, so the lifecycle's `WHEAT-2026-07` orders 400 ("instrument not found") and every downstream cross-service assertion (positions/netting/margin/settlement) tolerates an empty result via the graceful pattern. No trade is ever produced to propagate.

**Conclusion:** cross-service event propagation does not happen — over Kafka or otherwise — because the engine binaries publish/consume nothing. This is a real wire-up gap, distinct from the (now-removed) channel-stub limitation. See follow-up **R024**.

## Blockers / Infrastructure defects discovered (all real, all outside test-writer scope)

A genuinely *fresh* full-stack bring-up surfaced three defects that block reproducible build/deploy. Each was worked around at orchestration time (temp `/tmp` artifacts only — **no repo source changed**) so the e2e run could proceed; each needs a real fix in `infrastructure/` or `src/*/Dockerfile`.

1. **Migration files are not zero-padded → Postgres `initdb.d` applies them lexicographically.** `V10__…`–`V19__…` sort *before* `V1__core_schemas.sql`/`V2__exchange_schema.sql`, so `V11__matching_engine_tables.sql` runs before the `exchange` schema exists → `ERROR: schema "exchange" does not exist`, postgres exits, whole stack fails. Fix: rename to `V001__`…`V033__` (zero-pad) in `infrastructure/db/migrations/`. → **R025**
2. **Duplicate/conflicting `market_data.trades` definition (V8 vs V16).** `V8__market_data_timescaledb.sql` creates `market_data.trades(executed_at …)` and `V16__market_data_tables.sql` re-creates it as `…(traded_at …)` + `create_hypertable(…, 'traded_at')`. On a fresh DB V8 wins the `CREATE`, then V16's `IF NOT EXISTS` no-ops and its `create_hypertable`/index on `traded_at` fails (`column "traded_at" does not exist`). The **service code uses `ace_market_data.trades.traded_at`** (V16 shape), so V8 is the stale duplicate. Workaround: neutralized V8 in the orchestration copy. Fix: reconcile/remove the duplicate V8 (V32's own header already flags this conflict). → **R025**
3. **Per-service Docker build context excludes the shared modules → fresh images cannot be built with the committed Dockerfiles.** Every Dockerfile builds context `./src/<svc>` only, but after R002 (`shared/pkg/types/decimal`) and R010 (`shared/pkg/tenant`) the modules use relative `replace ../shared/...`, which is outside the context: `replacement directory ../shared/pkg/tenant does not exist`. This is *why every on-disk image is stale (06-22)* — the image build broke the moment the shared-module refactor merged. Workaround: host `go build` (where the sibling dirs exist) baked into minimal alpine images. Fix: change each service's build context to `./src` (or add a root builder Dockerfile / vendored shared modules) so the shared modules are in scope. → **R026**
