## Failures

### 1. gateway — `internal/middleware` tenant enforcement (3 failures) — REAL DEFECT

```
--- FAIL: TestTenantMiddleware_ValidTenant   (expected tenant in context, got "")
--- FAIL: TestTenantMiddleware_MissingHeader  tenant_test.go:32 handler should not have been called
                                              tenant_test.go:40 expected 401, got 200
--- FAIL: TestTenantMiddleware_UnknownTenant  tenant_test.go:47 handler should not have been called
                                              tenant_test.go:56 expected 403, got 200
FAIL  github.com/garudax-platform/gateway/internal/middleware
```

**Root cause:** `src/gateway/internal/middleware/tenant.go` `tenantBypassPrefixes` whitelists
essentially every tenant-scoped API path (`/api/v1/orders`, `/api/v1/margin`,
`/api/v1/participants`, `/api/v1/tickets`, plus clearing/settlement/warehouse/market-data).
The tests exercise `/api/v1/orders`, which matches a bypass prefix, so `TenantMiddleware`
skips header validation and calls the downstream handler with **no tenant in context** —
returning 200 where 401/403 is required.

**Impact:** This defeats tenant scoping at the gateway and contradicts the platform invariant
("Tenant ID is never optional" / "no service accepts traffic with an unresolved tenant",
2026-04-23 multi-tenancy directive). It is a security-relevant correctness defect, not test
drift. Tracked by deferred backlog tasks R011/R012. **Not fixed by this agent (report-only).**

## Recommendation

- **P1 (correctness/security):** Fix gateway `tenantBypassPrefixes` — narrow it to genuinely
  platform-level paths only (`/platform/`, `/api/v1/platform/`, `/api/v1/auth/`, health) and
  require `X-GarudaX-Tenant` on all tenant-scoped routes (deferred tasks R011/R012). This is a
  multi-tenancy invariant violation and should precede further feature work.
- **P2:** Reconcile admin-ui `market-phase`/`bot-integration` and demo-runner `sections`/`t066`
  suites with their merged implementations — determine source-of-truth (code looks newer than
  tests) and update the lagging side.
- Playwright UAT specs under `tests/playwright/` were **not executed** — they need a live
  full stack + browsers (out of scope for this static build/test run).

**Overall health:** Build pipeline is healthy (19/19 Go modules + 3/3 SPAs compile). Test suite
is **FAIL** — 1 real gateway tenant-enforcement defect plus 26 frontend assertions out of sync
