## Summary

Produced a comprehensive specification document for the ACE Admin Dashboard at `docs/T049_admin_dashboard_spec.md`. The spec defines a React 18 + TypeScript + Vite SPA for exchange operators and compliance officers, covering 8 feature areas (System Monitoring, Participant Management, Margin Calls, Settlement Status, Circuit Breakers, Warehouse Receipts, Compliance Alerts, Audit Log) with complete route definitions, component hierarchy, API endpoint mapping, role-permission matrix, and polling intervals.

## Key Decisions

1. **Two-role model (admin vs compliance_officer)**: Rather than a granular permission system, the dashboard uses two broad roles matching the auth-service JWT claims. Admin sees everything; compliance_officer sees only Participants, Compliance Alerts, and Audit Log. This aligns with the gateway's existing RBAC roles (`exchange_admin`, `compliance_officer`).

2. **Polling-first, no WebSocket initially**: All dashboard data uses polling at defined intervals (10s-60s depending on urgency). WebSocket streams are noted as a future enhancement. This simplifies the initial implementation while still providing near-real-time data.

3. **Context + useReducer over Redux**: React Context with `useReducer` for auth state, plus per-page custom hooks for data fetching. This avoids Redux complexity for a dashboard that doesn't need cross-page shared data state.

4. **Aggregated health endpoint**: Proposed a new `GET /api/v1/admin/health` gateway endpoint that proxies health checks to all 9 services, rather than having the browser make 9 separate requests. This reduces CORS complexity and network overhead.

5. **Runtime config injection**: Build artifact is environment-agnostic; config is injected via `window.__ACE_CONFIG__` script tag, enabling the same Docker image across dev/staging/prod.
