## Summary
Added a `useWebSocket` React hook to the admin dashboard that provides real-time WebSocket connectivity with auto-reconnect and exponential backoff. Integrated it into the OrderBook page (with polling fallback) and added a WS connection status badge to the TopBar.

## Key Decisions
1. The hook uses exponential backoff (1s, 2s, 4s, ... capped at 30s) and resets the backoff counter on successful connection, matching the task spec exactly.
2. OrderBook uses WS data when connected and falls back to existing `usePolling` when disconnected — polling is disabled while WS is active to avoid redundant requests.
3. TopBar uses a standalone WS health check connection (`/health` path) rather than shared state, keeping it decoupled from page-specific WS connections.
4. WS status dot in OrderBook uses inline `style` for the color to avoid CSS module class proliferation for a simple 3-state indicator.
