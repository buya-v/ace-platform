## What it fixes (the three P0 bugs)
1. **Silent overflow** — `MulDecimal`/`MulInt64`/`MulUint64` use a 128-bit
   intermediate (`bits.Mul64`/`bits.Div64`); overflow returns `ErrOverflow`
   (Try*) or panics. Test `TestMulDecimalOverflowNotSilent` proves the old
   margin-engine `(a*b)/scale` wrap is gone.
2. **Truncation bias** — `MulDecimal` and `DivInt64`/`DivInt` round half-to-even.
   `ParseDecimal` rounds (not truncates) >4 fractional digits. Tests:
   `TestDivRoundsHalfEven`, `TestMulDecimalRoundsHalfEven`, `TestParseRoundsHalfEvenNotTruncate`.
3. **Silent divide-by-zero** — returns `ErrDivideByZero` / panics, never `0`.
   Test `TestDivByZeroIsError`.

## API (superset of all six old copies)
Constructors: `NewDecimal`, `DecimalFromInt`, `DecimalZero`, `Zero`, `DecimalFromRaw`,
`ParseDecimal`, `MustParse`. Arithmetic (each with a `Try*` sibling): `Add`, `Sub`,
`MulInt64`, `MulUint64`, `MulDecimal`, `DivInt64`, `DivInt`, `Negate`, `Abs`.
Comparison: `Cmp`, `LessThan`, `GreaterThan`, `Equal`, `GreaterThanOrEqual`,
`LessThanOrEqual`, `Max`, `Min`. Predicates/accessors: `IsZero`, `IsNeg`, `IsPos`,
`Raw`, `String`.

## Notes / behavioural deltas for migrators (R003/R004)
- Results that previously **truncated** now **round half-even** — expect a few
  existing engine test expectations to shift by 1 ulp; update them to the rounded value.
- Previously-silent overflow/÷0 now **panic** (convenience) or **error** (`Try*`).
  At untrusted/unbounded money boundaries, prefer the `Try*` variants.
- `Cmp` returns -1/0/1 exactly as before; `Raw()`/`DecimalFromRaw()` preserve the
  scaled int64 so no DB/proto migration is needed.
