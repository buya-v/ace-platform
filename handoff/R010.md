# Handoff — R010: Adopt shared tenant module

**Status:** DONE
**Agent:** Coder
**Phase:** R2 (tenancy)

---

## Summary
Promoted the shared tenant primitives out of `src/shared/internal/tenant` into a new
standalone **zero-dependency** module `github.com/garudax-platform/tenant`
(`src/shared/pkg/tenant/`), mirroring the validated decimal-module pattern, so any
service can adopt them via a filesystem `replace` without pulling in the heavy
`shared` module (pgx/grpc). Wired gateway and securities-service to that module so
they stop carrying their own duplicate `TenantID` type + context key, and deleted
the orphaned `src/platform-control` registry. Row-level isolation and the gateway
bypass-list narrowing (R011) are deliberately untouched.

## Key Decisions
1. **Standalone zero-dep core module, not the whole `shared` module.** The tenant
   package lived under `internal/`, which Go's internal rule makes un-importable
   from other modules (gateway/securities are not under `…/shared/`). I moved the
   core files (`context.go`, `middleware.go`, `observability.go` + their two tests)
   to `src/shared/pkg/tenant/` as `module github.com/garudax-platform/tenant`,
   `go 1.22`. This matches the decimal precedent and keeps the core stdlib-only so
   even the zero-dep engines can adopt it later. Consumed via
   `replace github.com/garudax-platform/tenant => ../shared/pkg/tenant`.
2. **gRPC interceptors stay in the `shared` module.** `grpc.go`/`grpc_test.go`
   depend on `google.golang.org/grpc`. Keeping them in the core would force grpc as
   a transitive dep onto gateway/securities (HTTP-only). Instead `internal/tenant`
   now holds `grpc.go` (unchanged) + a new `alias.go` that re-exports the core
   symbols via type alias (`type TenantID = core.TenantID`, wrapped
   `WithTenant`/`TenantFromContext`/`MustTenant`, `HeaderName`, local
   `buildTenantSet`). Because it's a **type alias**, the context key is shared:
   grpc.go, the kafka tenant router, observability, gateway and securities all read
   and write one and the same `TenantID`/context value. `grpc.go`, `grpc_test.go`,
   `kafka/tenant_router.go`, and `internal/observability/*` needed **zero edits**.
3. **Securities-service fully adopts shared.** Its middleware was behaviorally
   identical to the shared one (health-only bypass, same error shapes/codes), so
   `internal/middleware/tenant.go` is now a thin shim delegating to
   `tenant.TenantMiddleware`/`WithTenant`/`TenantFromContext`/`MustTenant` and
   aliasing `TenantID`. The local `ValidTenantsFromEnv` env helper is kept (not part
   of the shared package). Public API of the `middleware` package is unchanged, so
   all ~60 call sites and `tenant_test.go` compile/pass untouched.
4. **Gateway adopts the shared type + context only; its policy middleware is left
   intact for R011.** The gateway's `TenantMiddleware` carries a `tenantBypassPrefixes`
   list and CORS handling that are gateway-specific routing policy (and the subject
   of R011), not a duplicate of the shared package. I replaced only the duplicated
   primitives: removed the local `tenantContextKey`/`TenantID`, aliased
   `TenantID = tenant.TenantID`, store via `tenant.WithTenant`, read via
   `tenant.TenantFromContext`, and read the header via `tenant.HeaderName`. Behavior
   is byte-for-byte preserved (verified — see Blockers).
5. **platform-control deleted, platform-service kept.** `src/platform-control` had no
   Go importers and no compose/k8s/Dockerfile wiring (only `tasks*.json` + its own
   Dockerfile + descriptive docs reference it). Removed the directory; the live
   registry `src/platform-service` (port 8095/9095) is untouched.

## Blockers Found
- **None for R010.** The 3 failing gateway tests (`TestTenantMiddleware_ValidTenant`,
  `_MissingHeader`, `_UnknownTenant`) are the **pre-existing R011 defect**: the
  `tenantBypassPrefixes` list contains `/api/v1/orders`, so those requests bypass
  enforcement. I verified these exact 3 tests fail identically on committed `HEAD`
  (before my change) and that my change adds **no new failures** — the rest of the
  gateway suite and all other affected packages are green. Fixing the bypass list is
  R011's scope, not R010's.
- `tasks.json` / `tasks-remediation.json` and several root docs
  (`docs/platform-architecture.md`, `GarudaX_Strategy_Directive.md`,
  `CODEBASE_REVIEW_2026-06-26.md`, `REMEDIATION_PLAN_2026-06-26.md`) still mention
  `platform-control`. These are outside my allowed scope (tasks.json is forbidden;
  docs belong to R013-class). Flagged for follow-up.

## Suggested Follow-ups
- **R011 (gateway tenant enforcement):** narrow `src/gateway/internal/middleware/tenant.go`
  `tenantBypassPrefixes` to genuinely platform-level paths only
  (`/platform/`, `/api/v1/platform/`, `/api/v1/auth/`, health) and require
  `X-GarudaX-Tenant` on all tenant-scoped routes. The 3 failing tests in
  `tenant_test.go` already encode the desired behavior. The type/context plumbing is
  now shared, so R011 only needs to touch the bypass policy, not the primitives.
- **R012 (tenant tests):** add gateway/securities gRPC + HTTP enforcement coverage;
  the shared `internal/tenant` gRPC interceptors (`NewGRPCInterceptors`,
  `Unary/StreamClientInterceptor`) are now ready to wire onto any gRPC server.
- **R013/R017 (docs + hygiene):** scrub the lingering `platform-control` references in
  `tasks*.json` and the four root docs noted above.
- Future services should adopt tenant the same way: `require github.com/garudax-platform/tenant v0.0.0`
  + `replace => ../shared/pkg/tenant` (zero-dep, no go.sum, no network).
