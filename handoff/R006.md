# Handoff — R006: Money-path audit

**Status:** DONE
**Agent:** Coder
**Phase:** R0 financial-correctness remediation

## Summary

Static audit of every Go money path (price / quantity / cash / P&L / margin /
settlement) across all 21 `src/` modules, confirming the state after R003 (6
engines → shared decimal) and R004 (securities-service float→decimal). The
R003/R004 **defined scope is clean**: all six engines and securities-service's
core order/position/P&L/settlement/accrued-interest math now run on the shared
`decimal.Decimal` type with checked overflow, half-even rounding, and
error-on-divide-by-zero. One live, in-scope truncation was found and fixed
(margin-engine haircut). The audit additionally surfaced **four out-of-scope
modules that still carry residual float64 money math** (corporate-actions,
compliance MCSD/FRC, gateway fees/risk, fix-gateway) — these were never part of
R003/R004 and need their own remediation tasks.

## Audit method

Grepped (non-test `.go`, excluding the shared module which legitimately owns
`float64`) for every token named in the task plus the underlying defect class:

| Token | What it finds | Result |
|---|---|---|
| `float64` | float money fields/arithmetic | classified per-service below |
| `/365`, `/360` | accrual-basis truncation | none in arithmetic — only string constants, comments, and one SQL `EXTRACT(EPOCH)/3600` (ticket-SLA hours, non-money) |
| `*PriceScale` / `PriceScale` | old price scaling | only `securities-service/internal/protocol` (dead/test-only, see below) |
| `DivInt` / `DivInt64` | truncating division | **all call sites are the shared `decimal` method** (checked, half-even) — verified at market-data VWAP, securities avg-cost/auction/bond/settlement |
| `MulDecimal` | fixed×fixed multiply | all the shared `decimal` method |
| `int64(float * N)` / `uint64(float * N)` | silent float→int truncation | 3 sites: margin haircut (**fixed**), securities protocol (dead), compliance round2 (out of scope) |

Also confirmed: the 6 per-engine `internal/types/decimal.go` files are now
**23-line alias shims** (byte-identical, all → `decimal.Decimal`), no leftover
full implementations, and no leftover `decimal_test.go` asserting old truncation.

## Fix applied (the one live, in-scope truncation)

`margin-engine/internal/collateral/warehouse_source.go:127`

```go
// before — truncates toward zero:
haircutDecimal := types.DecimalFromRaw(int64(s.haircut * 10000))
// after — half-even via the shared decimal, with error handling:
haircutDecimal, err := decimal.NewFromFloat(s.haircut)
if err != nil { … return DecimalZero() }
```

The haircut multiplies collateral value (`qty.MulDecimal(price).MulDecimal(haircut)`),
so a truncated haircut understates/overstates every pledged-receipt credit.
`int64(haircut*10000)` truncated toward zero; `decimal.NewFromFloat` rounds
half-to-even. `haircut` is bounded `(0,1]` by `WithHaircut`/`DefaultHaircut`, so
the error path is defensive only. `go build`, `go vet`, `gofmt`, and the full
margin-engine test suite (`go test ./...`) all pass.

## Residuals noted but NOT changed (justified / deferred)

1. **securities-service deferred floats** (R004 Suggested Follow-ups #2): reference
   prices (`ReferencePrice`, `OffBookTrade.Price`, `TradeCorrection.Original/
   CorrectedPrice`), `CustodyBalance.AvgCost`, `Entitlement.EntitlementValue`
   (dividend cash), `TradingParameterSet.MaxOrderValue`, `Index.BaseValue/
   CurrentValue`, market-data display (`PriceLevel.Price`, `TickerData.*Price`).
   Genuine ratios stay float by design: `TickSize`, `CouponRate`, circuit-breaker
   `*Pct` bands, `PenaltyRatePct`. These were explicitly scoped out of R004 — a
   known, documented residual, not a regression.
2. **securities-service `pg_store.go` DB boundary** (R004 Follow-up #3):
   `decFromFloat` scans `price/stopPrice/avgCost/marketValue/unrealizedPnl` as
   `float64` from numeric columns. Precision-at-rest gap deferred to a future
   `*_raw int64` column migration. Conversion itself is safe (half-even
   `NewFromFloat`), but the round-trip through float64 is not exact-by-construction.
3. **securities-service `internal/protocol/message.go`**: `PriceToFixed` uses
   `uint64(price*PriceScale)` (truncating) and `FixedToPrice` returns float64.
   **Dead code** — the `protocol` package is imported by nothing; only its own
   tests call these. On-wire `Price` is integer `uint64` fixed-point (sound).
   Recommend deleting the float helpers or routing them through decimal.
