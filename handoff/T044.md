## Gaps Identified

- `CalculateMargin` at 93.3%: the uncovered path is the error return from `callService.Evaluate()` — the current `margincall.Service.Evaluate()` never returns an error, so this path is unreachable without mocking the service. This is an acceptable gap.
- `./...` cross-package numbers show lower coverage (32-38%) because Go counts all transitively imported code. The isolated per-package coverage is the accurate measure.
