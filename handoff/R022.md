## Summary

Re-ran the R006 money-path audit across the four modules remediated by R018–R021
(corporate-actions, compliance-service MCSD/FRC, gateway fees+risk, fix-gateway)
and re-confirmed the six engines + securities-service. **All four R006 FAIL
residuals are now CLOSED** — every money path they flagged is on the shared
`github.com/garudax-platform/decimal` type, and all three float `round2`/`roundTo4`
money helpers in those paths are deleted. The engines and securities remain clean
(alias shims intact; only R004-documented deferred floats remain in securities).

**However, the platform is NOT yet 100% zero-residual.** The re-audit surfaced
**one previously-misclassified residual** that R006 wrongly lumped under "gateway
reporting = non-money": `gateway/internal/reporting/generator.go` computes
settlement-statement P&L, fee totals, net amount, market-summary OHLC, and
large-trader positions in `float64` with a float `roundTo4`. It is **latent**
(the `Generate*` functions have no live caller — GET handlers serve pre-stored
reports; only tests call the generator), exactly like corporate-actions was
before R018. Per the audit-task pattern (minimal diff, named follow-ups), it is
**not** migrated here — it is emitted as scoped follow-up **R023**.

## Audit method

Replicated R006's method on non-test, non-`kafka/` `.go` (the shared module
legitimately owns `float64`):
- `float64` token sweep across the 4 remediated modules + 6 engines + securities + all N/A modules.
- `int64(float*N)` / `uint64(float*N)` silent-truncation hunt platform-wide.
- `func round2|roundTo4|roundTo2|round(` float-rounding-helper sweep.
- Verified each engine's `internal/types/decimal.go` is still the 23-line
  `type Decimal = decimal.Decimal` alias shim.
- `go build ./...` + targeted `go test` on every remediated money package.

  ratios `TickSize`/`CouponRate`/`*Pct`) and the dead `internal/protocol`

### ⚠️ NEW residual — gateway/internal/reporting/generator.go (latent, R006 mis-classified)
Genuine `float64` money math, **not** previously remediated:
- `GenerateSettlementStatement` — unrealized P&L `(MarkPrice-AvgPrice)*Quantity`,
  `totalFees` sum, `NetAmount = totalUnrealized - totalFees` (generator.go:72-88).
- `GenerateMarketSummary` — OHLC, volume, settlement price (generator.go:132-159).
- `GenerateLargeTraderReport` — net/gross position (generator.go:202-214).
- float `roundTo4` (generator.go:231) — the **only** money-rounding float helper
  left in the four audited modules.
- DB-at-rest: `reporting/store.go` `DailyStatement.NetAmount` (:19) and the
  `MarketSummary` OHLC columns (:28-34) are scanned/stored as `float64` — same
  deferred class as securities `pg_store` residual #2.

**Why not fixed here:** the `Generate*` functions have **no live caller** — the
GET routes (`/api/v1/reports/settlement-statement`, `/trade-summary`,
`/admin/reports/market-summary`, `/admin/reports/large-traders`) serve
pre-stored reports from the store; the generator is exercised only by its tests.
This makes it a **latent** residual (analogous to corporate-actions before R018),
not a live correctness defect. Per the learned audit-task pattern, the audit keeps
its own diff minimal and emits a named, scoped follow-up rather than performing a
sizeable generator+store+DTO migration inside an audit task.
