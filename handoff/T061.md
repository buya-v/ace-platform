# T061 — Domain Configuration (ace.asla.mn + demo.ace.asla.mn)

**Status:** Done
**Date:** 2026-03-28

## Summary

Configured the ACE Platform to use production domains `ace.asla.mn` (main platform) and `demo.ace.asla.mn` (demo runner). Changes span nginx reverse proxy, Docker Compose, Kubernetes ingress/TLS, frontend Dockerfiles, CORS middleware, and all documentation/scripts.

## Changes Made

### New Files
- **`infrastructure/nginx/nginx.conf`** — Reverse proxy with two server blocks (ace.asla.mn, demo.ace.asla.mn), proper proxy headers, WebSocket upgrade support, commented TLS blocks ready for certificate provisioning.
- **`src/web-ui/Dockerfile`** — Multi-stage build (node:22-alpine build, nginx:alpine serve on port 3000).
- **`src/admin-ui/Dockerfile`** — Multi-stage build (node:22-alpine build, nginx:alpine serve on port 3001).
- **`src/demo-runner/Dockerfile`** — Multi-stage build (node:22-alpine build, nginx:alpine serve on port 3002).
- **`deploy/k8s/base/ingress.yaml`** — Kubernetes Ingress with TLS for both domains, cert-manager annotations, WebSocket support annotation.
- **`deploy/k8s/base/certificates.yaml`** — ClusterIssuers (Let's Encrypt prod + staging), Certificate resources for both domains.
- **`src/gateway/internal/middleware/cors.go`** — CORS middleware allowing `https://ace.asla.mn` and `https://demo.ace.asla.mn`, with env-var override via `CORS_ALLOWED_ORIGINS`.

## Decisions Made

1. **HTTP-first for Docker Compose** — nginx.conf listens on port 80 by default with TLS blocks commented out. This allows local development without certificates. Production TLS is handled by K8s Ingress + cert-manager.
2. **CORS via Go middleware** — Created a standalone `cors.go` middleware rather than modifying `auth.go`. This follows the existing middleware chain pattern (requestid, logging, ratelimit, auth) and keeps concerns separated.
3. **Environment-extensible CORS** — Hardcoded production origins with `CORS_ALLOWED_ORIGINS` env var for additional origins (e.g., localhost during development).
4. **Frontend Dockerfiles use nginx:alpine** — Matches the reverse proxy pattern and is production-ready. Port remapping via sed in the Dockerfile.
