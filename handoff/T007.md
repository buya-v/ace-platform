## Summary

Designed the complete CLOB (Central Limit Order Book) architecture for the ACE matching engine. Produced a detailed spec document covering order book data structures, price-time priority matching algorithm, auction-based price discovery, trade ledger design, and circuit breakers. Created gRPC protobuf API contracts (OrderService, MarketDataService, AdminService) and a SQL migration for three new tables (instruments, execution_reports, circuit_breaker_events).

## Key Decisions

1. **Single-threaded matching per instrument** — guarantees deterministic order-to-trade mapping; instruments parallelized across engine instances.
2. **Price-time priority (strict FIFO)** — standard CLOB algorithm; simpler than pro-rata and appropriate for an agriculture commodity exchange with moderate volume.
3. **Event-sourced book state** — the order book is rebuilt from the event log on recovery, with periodic snapshots for speed. No shared state between engine instances.
4. **Dual auction sessions** — opening auction (09:00-09:05) and closing auction (15:25-15:30) using equilibrium price maximization algorithm for price discovery.
5. **Append-only execution reports** — follows T004's append-only pattern for trades; execution_reports table has DELETE/UPDATE rules blocked.
6. **Cancel-replace for order modifications** — ModifyOrder uses cancel-replace semantics (loses time priority) which is simpler and safer than in-place modification.
7. **Self-trade prevention modes** — three STP modes (cancel newest/oldest/both) configurable per participant account.
8. **Decimal-as-string in protobuf** — prices are string-encoded decimals to avoid floating-point precision issues in gRPC messages.
