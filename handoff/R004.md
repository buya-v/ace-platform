# R004 — Remove float64 Money Math in securities-service — Handoff

**Status:** prerequisite done; full conversion routed to the pipeline (worktree + review)

## Why this is not a single manual pass
Measured blast radius: `float64` is the wire/JSON type for `Price` and money fields
across `Order`, `Trade`, `Position`, `Settlement`, `Instrument`, ~25 handler DTO
files, plus the DB-scan layer (`pg_store.go`, `store.go`) which scans straight into
`float64`. That is ~66 money-field declarations + ~17 arithmetic sites — a 60–100
edit refactor of the **flagship** service that also changes the external JSON API.
With no live DB/stack on this host, a manual pass can only be unit-validated; a
half-converted flagship (mixed float/Decimal boundaries) would be worse than today's
consistent-but-imprecise state. This is the canonical case for the pipeline's
isolated-worktree + reviewer loop.

## What WAS completed (safe, tested prerequisite)
The shared decimal type (`github.com/garudax-platform/decimal`) gained the support
needed to convert `Price` to `Decimal` **without breaking the wire contract** the
three SPAs and e2e tests depend on:
- `MarshalJSON` → emits a bare JSON number (`12.5`, not `"12.5"`).
- `UnmarshalJSON` → accepts number, string, or null; rounds >4 dp half-even.
- `Float64()` / `NewFromFloat()` → explicit, documented boundary for legitimately
  float-typed interop (never for money arithmetic).
- Tests added (`TestJSONRoundTrip`, `TestFloatInterop`); module still green at
  **92.1% coverage**. The 6 migrated engines (R003) re-verified green against the
  enhanced module (additive change).

The speculative `require`/`replace` added to `securities-service/go.mod` while
probing was reverted to keep the tree clean — the conversion task adds it back.

## Conversion plan (encoded in tasks.json R004 description)
1. `go.mod`: `replace github.com/garudax-platform/decimal => ../shared/pkg/types/decimal`.
2. Convert money fields to `decimal.Decimal` (list in the task): Order/Trade/Position/
   Settlement/Instrument. Leave non-money floats (surveillance scores, ratios,
   latency, percentages) as `float64`.
3. Fix arithmetic sites with shared ops:
   - `engine.go:678-681` AvgCost/MarketValue/UnrealizedPnl → `MulInt64`/`MulDecimal`/`DivInt64`.
   - `settlement/engine.go:79` `NetAmount = Price.MulInt64(qty)`.
   - `settlement/equities.go:186`; bond accrual `engine.go:207-220` via `MulDecimal`/`DivInt64`.
4. DB-scan/store: scan into `Decimal` (or scan float then `NewFromFloat` at the edge).
5. Handler DTOs: `Decimal` fields marshal as numbers automatically — minimal churn.
6. Verify: `go test ./...`; R005 reconciles securities settlement cash vs settlement-engine.
