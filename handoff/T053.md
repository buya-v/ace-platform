## Key Decisions
1. **Health checks use `wget --spider`** — Alpine images don't include `curl` or `grpc_health_probe`, but `wget` is available by default. All Go services expose HTTP `/healthz` endpoints, so wget works for all 9 services.
2. **YAML anchors for DRY config** — Used `x-common-env` for shared env vars (KAFKA_BROKERS, BIND_ADDRESS) and `x-healthcheck-http` for shared healthcheck timing to reduce duplication across 9 service blocks.
3. **Dependency chain reflects trading pipeline** — `matching-engine` starts first, then `clearing-engine` depends on it, `margin-engine` and `settlement-engine` depend on `clearing-engine`, and `gateway` depends on all core services. All services depend on `postgres` and `kafka`.
4. **Gateway health port is 8090** — Gateway's config uses `HEALTH_PORT` defaulting to 8090 (separate from its main HTTP port 8080), matching the source code.
5. **Dockerfiles follow auth-service pattern** — Multi-stage build with `golang:1.22-alpine` builder and `alpine:3.19` runtime, non-root `appuser`, CGO disabled. Zero-dep services skip `go mod download` and `COPY go.sum` since they have no external dependencies.
6. **DB init scripts mounted from `infrastructure/db/migrations/`** — SQL migrations are mounted to `docker-entrypoint-initdb.d` for automatic schema creation on first run.
