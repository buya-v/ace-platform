## Summary

Created 5 Terraform modules covering VPC, EKS, RDS, MSK, and security groups.
State backend configured with S3 + DynamoDB in eu-west-1. Three workspaces: dev/staging/prod.

## Key Decisions

1. **Module-per-service** pattern — each AWS service gets its own module for independent lifecycle
2. **VPC endpoints** — S3 gateway + 7 interface endpoints to avoid NAT costs and improve security
3. **IRSA OIDC provider** created at EKS module level — all service accounts bind here
4. **Karpenter IRSA role** pre-provisioned for T003 to deploy autoscaler
5. **MSK topics stored in SSM** — 12 topic definitions, services discover via Parameter Store
6. **Security groups are least-privilege** — data tier has no internet route; RDS/Redis/MSK only accept traffic from EKS node SG
7. **Required tags enforced** — Project, Environment, Phase, TaskID, ManagedBy, CostCenter
