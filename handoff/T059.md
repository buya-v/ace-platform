## Summary

Built a Demo Runner web application as a React 18 + TypeScript + Vite SPA at `src/demo-runner/`. The app provides a visual button-driven interface for walking through the 9 demo runbook sections (32 API steps + 1 production readiness checklist). Dark theme trading UI aesthetic with zero heavy dependencies — uses React Context + useReducer for state, CSS modules for styling, and fetch() for API calls.

## Key Decisions

1. **Zero heavy dependencies** — No Redux, Material UI, Tailwind, or axios. State managed via React Context + useReducer. CSS modules for scoped styling. fetch() for all HTTP calls.
2. **Step definitions as data** — All 32 API steps defined declaratively in `data/sections.ts` with typed validators, state extractors, and dynamic URL resolvers. This makes adding/modifying steps trivial.
3. **Token flow via appState** — Login steps extract JWT tokens into shared appState via `extractState` callbacks. Downstream steps (trading, clearing, etc.) automatically pick up tokens via `headers` callbacks that read from state.
4. **Production readiness as interactive checklist** — Section 9 uses a different component (`ReadinessChecklist`) with toggleable checkboxes and percentage calculation, separate from the API step cards.
5. **Vite proxy for CORS** — Dev server proxies `/api` to configurable gateway URL (default `http://localhost:8080`).
6. **Port 3002** as specified, avoiding collision with web-ui (3000) and admin-ui (3001).
