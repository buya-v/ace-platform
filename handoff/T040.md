## Summary
Rebuilt the auth service as a zero-dependency Go module (plus golang.org/x/crypto for bcrypt) at `src/auth-service/`. Implements JWT access/refresh tokens with HMAC-SHA256 signing, OAuth2 PKCE flow (S256 only), RBAC with 5 roles, user registration with bcrypt cost 12, login with account lockout (5 attempts / 30 min), API key management with SHA-256 hashed storage, session management with refresh token rotation, and HTTP handlers with input validation. Ports: gRPC 50055, health HTTP 8085.

## Key Decisions
1. **Domain types in `internal/types` package** — Moved domain types (User, Session, APIKey, Role, Permission, etc.) to a dedicated `types` package to avoid import cycles between `auth` and `store` packages.
2. **golang.org/x/crypto as sole external dependency** — Required for bcrypt password hashing. All other crypto (JWT HMAC-SHA256, PKCE S256, API key hashing) uses stdlib `crypto/*` packages.
3. **Refresh token hash stored on Session** — Addresses the T005 review gap: `ExchangeCode` and `createTokenPair` store the SHA-256 hash of the refresh token on the session record, enabling secure token rotation and theft detection.
4. **S256-only PKCE** — The `plain` challenge method is rejected at both service and handler layers, per T005 reviewer suggestion.
5. **In-memory store with `auth.Store` interface** — The `Store` interface is defined in the `auth` package (consumer-side), and `store.InMemoryStore` satisfies it. This allows swapping to a SQL implementation without changing business logic.
6. **bcrypt cost 4 in tests** — Tests use cost 4 for speed; production default is cost 12 (configurable via `BCRYPT_COST` env var).
