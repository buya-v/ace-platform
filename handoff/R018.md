# Handoff — R018: corporate-actions float→decimal

**Status:** DONE
**Agent:** Coder
**Phase:** R0-extension (financial-correctness remediation, follow-up to R006 audit)

---

## Summary
Migrated the standalone `corporate-actions` Go module off `float64` money math onto the
shared fixed-point `github.com/garudax-platform/decimal` type, and fixed the parallel
`float64` dividend-entitlement defect in `securities-service`. All dividend cash, rights
subscription cost, split-adjusted price, and theoretical ex-rights price are now exact
Decimal arithmetic with banker's (half-even) rounding; the float `round2` helper is gone.
Both modules build, vet clean, and pass their full suites (corporate-actions 100% coverage).

## Key Decisions
1. **Wired the shared module via the validated relative `replace` directive.** Added
   `require github.com/garudax-platform/decimal v0.0.0` + `replace ... => ../shared/pkg/types/decimal`
   to `src/corporate-actions/go.mod`, keeping `go 1.22` so it builds with the host toolchain
   and stays zero-`go.sum` / zero-network (per the R002/R003 learned pattern). securities-service
   already had this wiring from R004.
2. **Removed `round2` entirely rather than re-implementing it as 2dp banker's rounding.**
   The platform Decimal standard is 4dp (`Scale=10000`); cents-rounding is a settlement-boundary
   concern, not an engine concern. Money is now carried at native 4dp with half-even rounding
   built into `MulInt64`/`DivInt64`. This is the more-precise, platform-consistent choice and
   matches the R002 migration guidance ("update expectations to the rounded value").
   - Behavioral deltas (test expectations updated): dividend 333×0.1250 now `41.6250` (was the
     lossy `41.63`); TERP 58/6 now `9.6667` (was `9.67`). All exact-value cases (150, 375, 160,
     330, 50, 20) are unchanged.
3. **Converted integer-result paths to exact integer arithmetic, not Decimal.** Split share
   counts (`SplitAdjustedQuantity` whole shares, `CalculateRights` rights count) use
   `qty*NewShares/OldShares` integer floor — exact and overflow-equivalent to the old float
   floor for the non-negative (eligible) quantities the engine ever sees.
4. **Made `SplitAdjustedQuantity`'s fractional remainder a `Decimal`** (was `float64`). It is the
   basis for cash-in-lieu, so a precise fixed-point fraction (`rem/OldShares`) is correct; this
   removes the last float from the split path.
5. **Kept `SplitTerms.Ratio() float64`** — it is a dimensionless share-ratio multiplier (e.g. 1.5×),
   not money, and is now only used for the invalid-ratio validation check. Left as public API.
6. **securities-service dividend fix:** `Entitlement.EntitlementValue` is now `types.Decimal`
   (`entitlement_value` JSON unchanged — Decimal marshals as a bare number). The handler converts
   `dividend_amount` (JSON float64/int) to Decimal at the boundary via `decimal.NewFromFloat`/
   `DecimalFromInt`, then `entitlementValue = dividendAmount.MulInt64(qty)`. The custody-balance
   integer delta uses `int(entitlementValue.Float64())`.
