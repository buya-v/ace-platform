## Summary

Implemented Kafka producer/consumer adapters for all 9 ACE platform services following the zero-dependency Go module pattern. Each service has its own `internal/kafka/` package with shared core infrastructure (event envelope, channel-based producer/consumer with retry, idempotency, and DLQ support) plus service-specific wiring (typed payloads, partition key routing, handler factories). All tests pass with 82.4-82.9% coverage across all services.

## Key Decisions

1. **Interface-based Producer/Consumer** — `Producer` and `Consumer` are Go interfaces backed by `ChannelProducer`/`ChannelConsumer` implementations using Go channels. This allows swapping in a real Kafka wire protocol client at deployment without changing service code.
2. **Zero external dependencies** — the entire kafka package uses only Go stdlib (`crypto/rand`, `encoding/json`, `context`, `sync`, `time`, `fmt`, `log`, `os`, `strconv`, `strings`, `net` — no third-party imports).
3. **Copied per service (zero-dep pattern)** — consistent with the existing Decimal type duplication pattern. Each service owns its kafka package independently.
4. **Event envelope matches T051 spec exactly** — `id` (UUID v4), `type`, `timestamp`, `source`, `correlation_id`, `schema_version`, `payload` (json.RawMessage).
5. **Typed payload structs per service** — each service defines only the payload types it produces or consumes, mirroring the JSON schemas from T051. Consumer payloads are mirrored copies (not shared).
6. **Idempotency via in-memory processed ID set** — extends the clearing-engine's `processedTrades map[string]bool` pattern. Capped at 100K entries with LRU-style eviction.
7. **3 retries with exponential backoff before DLQ** — per T051 spec. DLQ topics follow `ace.dlq.{original-topic-without-prefix}` convention.
8. **Partition keys match T051 spec** — instrument_id for trade/clearing/settlement/market-data/delivery topics, participant_id for margin-call/compliance/warehouse-receipt/auth topics.
9. **Environment-based configuration** — `KAFKA_BROKERS`, `KAFKA_MAX_RETRIES`, `KAFKA_RETRY_BACKOFF_MS`, `KAFKA_MAX_TRACKED_IDS` read from env vars with sensible defaults.
