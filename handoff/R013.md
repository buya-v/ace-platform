## Reality verified against code (not just prior docs)
- **Registry is in-memory, not `platform.tenants`-backed:** `src/platform-service/
  internal/store/store.go` uses `InMemoryTenantStore` seeded with 2 hardcoded
  tenants. The `platform.tenants` table DDL exists (V29) but is unused by the
  service.
- **Provisioner is schema-only + dry-run by default:** `src/platform-service/
  internal/provisioning/provisioner.go` only `CREATE SCHEMA`s + computes Kafka
  topic prefixes; no IAM/Redis/dashboards; `db == nil` ⇒ dry-run.
- **Row-level tenant_id absent on domain tables:** only V32 annotates immutable
  audit/event rows with `tenant_id`; orders/trades/positions have none.
- **`mse_*` schemas do not exist:** only referenced in a comment in V33.
- **`ace_*` schema renames shipped:** V30/V32/V33.
- **Shared tenant module shipped (R010):** `src/shared/pkg/tenant/` (context,
  middleware, observability), `src/shared/internal/tenant/grpc.go` interceptors,
  `src/shared/kafka/tenant_router.go`.
- **Gateway enforcement (R011):** `tenantBypassPrefixes` = `/platform/`,
  `/api/v1/platform/`, `/api/v1/auth/`; valid tenants
  `[]string{"ace-commodities","mse-equities"}` in `main.go`.
