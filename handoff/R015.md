## Blockers found (real infra defects, out of test-writer scope → follow-up tasks)
- **R025 (P1):** Migration files are not zero-padded → Postgres `initdb.d` applies them
  lexicographically (`V11` before `V1`) → `schema "exchange" does not exist`; AND a
  duplicate/conflicting `market_data.trades` between `V8__` (executed_at) and `V16__`
  (traded_at, the shape the service uses). **A clean DB cannot initialize without both
  fixes.** Worked around by mounting a zero-padded ordered copy with V8 neutralized.
- **R026 (P1):** Per-service Dockerfiles build context `./src/<svc>` only, which excludes
  the shared modules added by R002/R010 (`../shared/pkg/types/decimal`, `.../tenant`) →
  `replacement directory does not exist`. **Fresh images cannot be built with the
  committed Dockerfiles** — this is why all images were 4 days stale. Worked around with
  host `go build` baked into minimal alpine images. Fix: build context `./src` (or
  vendor/root-builder).
- **R024 (P2):** Wire `internal/kafka` into engine `cmd/main.go` so cross-service events
  actually flow over the real broker (bug class #3), then add a topic/propagation
  assertion to `TestFullTradingLifecycle` against a seeded instrument.
