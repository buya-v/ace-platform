# Handoff — R021: fix-gateway price mapper → decimal

**Status:** DONE
**Agent:** Coder
**Phase:** R0 (financial-correctness remediation)

---

## Summary
Migrated fix-gateway's FIX protocol price/stop-price handling off `float64` onto
the shared `github.com/garudax-platform/decimal` type. Inbound price tags are now
parsed with `decimal.ParseDecimal` and outbound prices rendered with
`Decimal.String()`, so price values never touch binary floating point — closing
the float-drift residual flagged by the R006 audit (`mapper.go:39,40,121,204`,
`parser.go:223`).

## Key Decisions
1. **`GetFloatTag` → `GetDecimalTag`** (parser.go). FIX tags are strings on the
   wire, so the new helper round-trips through `decimal.ParseDecimal` and returns
   `decimal.Zero()` for absent/invalid tags — preserving the old "0 on missing"
   semantics that callers (`MapNewOrderSingle`) rely on, without the silent float
   conversion. Replaced rather than added-alongside because the only two callers
   are both price-bearing (`TagPrice`, `TagStopPx`).
2. **`InternalOrder.Price` / `StopPrice` are now `decimal.Decimal`.** The shared
   type's `MarshalJSON` emits a bare JSON number (e.g. `275.5`, `0`), identical in
   shape to the previous `float64`, so `server/tcp.go`'s `orderPayload` map sent
   to the securities-service needs **no change** and stays wire-compatible.
3. **`formatPrice` / `MapExecutionReport` take `decimal.Decimal`.** `formatPrice`
   now returns `price.String()`. Note the AvgPx wire format changed from fixed
   4-dp (`"275.5000"`, `"0.0000"`) to trailing-zero-trimmed (`"275.5"`, `"0"`).
   Both are valid FIX price strings (FIX floats have no fixed scale); this is the
   exact, drift-free representation. `MapExecutionReport` is only called from
   tests today — `server/tcp.go:sendExecutionReport` builds the fields map
   directly with a literal `"0.0000"`, which was left untouched (no float math).
4. **Module wiring via filesystem `replace`** (the validated repo pattern):
   `require github.com/garudax-platform/decimal v0.0.0` +
   `replace ... => ../shared/pkg/types/decimal`. No `go.sum` needed (zero-dep,
   local path). No alias shim added — fix-gateway has no `internal/types` package,
   so the `fix` package imports `decimal` directly.

## Behavioural deltas for reviewers / downstream
- ExecutionReport `AvgPx(6)` is now trailing-zero-trimmed (`"275.5"` not
  `"275.5000"`). If a FIX counterparty requires fixed 4-dp formatting, add a
  zero-padding variant of `formatPrice`; the current output is spec-valid.
- `GetFloatTag` no longer exists. Any future price tag should use
  `GetDecimalTag`; any genuinely non-money numeric tag should add its own helper
  rather than re-introducing float money.

## Suggested Follow-ups
- **R0 audit closure:** this clears the fix-gateway item from R006's residual
  list. The remaining out-of-scope `float64` money sites (corporate-actions,
  compliance MCSD/FRC, gateway fees/pre-trade risk) are tracked separately

- Consider tightening `GetDecimalTag` to surface parse errors (return

  silently becoming zero — current code preserves the prior silent-zero
  behaviour to avoid changing order-rejection semantics in this scoped task.
