# Handoff — R031: Add signed-cookie auth gate to architect-bot (demo portal, hard 7-day cutoff)

**Status:** DONE
**Agent:** Coder
**Phase:** R-series (demo portal)

## Summary
Added an additive, dependency-free signed-cookie login gate to the architect-bot
(`src/architect-bot/server.js`) so the demo portal (`demo.garudax.asla.mn`) can be
put behind a shared password with a HARD time cutoff. nginx enforces the gate via
`auth_request` against `GET /api/architect/_gate/check`. The existing
`GET /api/health` and `POST /api/chat` are unchanged and NOT auth-checked here
(nginx gates `/api/chat`). Everything uses Node 22 built-ins (`node:crypto` for
HMAC, manual `Cookie` header parsing, `express.urlencoded` for the form). 15
`node --test` tests added (all green).

## nginx auth_request wiring contract (read this to write the nginx config)

### Public paths (what the browser / nginx hit)
nginx rewrites `/api/architect/` → `/api/` before proxying to the bot, so:

| Purpose | Public path (browser/nginx) | Express route (after rewrite) | Method |
|---|---|---|---|
| Password page | `/login` | `/login` (served directly, **not** rewritten) | GET |
| Submit password | `/api/architect/_gate/login` | `/api/_gate/login` | POST |
| auth_request check | `/api/architect/_gate/check` | `/api/_gate/check` | GET |

- `GET /login` — self-contained HTML password page (inline CSS, no external assets).
  The form `POST`s `application/x-www-form-urlencoded` `{password}` to
  `/api/architect/_gate/login`. Shows an error banner on `?error=1`, and shows
  **"This demo access window has ended."** (no form) once `now >= DEMO_AUTH_EXPIRES_AT`.
  Shows **"Demo access is not currently available."** (no form) if the gate is unconfigured.
- `POST /api/_gate/login` — parses the urlencoded `password`.
  - gate unconfigured → **503**
  - `now >= DEMO_AUTH_EXPIRES_AT` → **403**
  - wrong password → **302** redirect to `/login?error=1` (no cookie)
  - correct password (constant-time compare) → sets the cookie (below) and **302** to `/`
- `GET /api/_gate/check` — the `auth_request` endpoint. Reads the cookie from the
  `Cookie` header (parsed manually). **204** when valid, **401** when
  missing/invalid/expired. Side-effect-free, no body, `Cache-Control: no-store`.

### Cookie
- **Name:** `demo_auth`
- **Format:** `<exp>.<hmacHex>` where `exp = DEMO_AUTH_EXPIRES_AT` (epoch seconds)
  and `hmacHex = HMAC-SHA256(DEMO_AUTH_SECRET, String(exp))` as lowercase hex (64 chars).
- **Attributes:** `Path=/; HttpOnly; Secure; SameSite=Lax; Max-Age=max(0, exp - now)`.
- **Validity (in `/api/_gate/check`):** valid IFF the value is exactly `<exp>.<hmac>`,
  the hmac recomputes and matches via **constant-time** compare, `exp ===
  DEMO_AUTH_EXPIRES_AT` (binds the cookie to the current cutoff so changing the
  cutoff invalidates old cookies), AND `now < exp`.

### Suggested nginx sketch (for the separate deploy task — NOT in this task)
```nginx
location = /api/architect/_gate/check {
    internal;
    proxy_pass http://architect-bot:3010/api/_gate/check;
    proxy_pass_request_body off;
    proxy_set_header Content-Length "";
    proxy_set_header Cookie $http_cookie;
}

location / {
    auth_request /api/architect/_gate/check;
    error_page 401 = @login_redirect;   # send unauthenticated users to /login
    # ... proxy to demo-runner SPA / architect-bot ...
}

location @login_redirect { return 302 /login; }

# /login and the login POST must be PUBLIC (no auth_request):
location = /login { proxy_pass http://architect-bot:3010/login; }
location = /api/architect/_gate/login { proxy_pass http://architect-bot:3010/api/_gate/login; }
```
(`/api/architect/` is rewritten to `/api/` on the proxy_pass — keep the existing
rewrite rule; the bot routes are `/api/_gate/login` and `/api/_gate/check`.)

## Env vars (all read at startup; never hardcoded; never logged)
| Var | Meaning |
|---|---|
| `DEMO_PORTAL_PASSWORD` | the shared password users type |
| `DEMO_AUTH_SECRET` | HMAC signing secret |
| `DEMO_AUTH_EXPIRES_AT` | hard cutoff as Unix epoch **seconds** integer (parsed strictly; NaN/float/negative/garbage → gate disabled) |

**Fail-closed:** if any of the three is unset/empty (or the cutoff is unparseable),
`/api/_gate/check` returns **401** and `/api/_gate/login` returns **503** — a
misconfigured deploy denies access rather than opening it.

**Startup log line** (no secrets printed):
- enabled: `Demo portal auth gate: ENABLED — access cutoff <ISO> (epoch <n>)`
- disabled: `Demo portal auth gate: DISABLED (fail-closed) — set DEMO_PORTAL_PASSWORD, DEMO_AUTH_SECRET and DEMO_AUTH_EXPIRES_AT ...`

## Key Decisions
1. **Constant-time compare hashes both operands first** (`SHA-256` → 32-byte
   buffers → `crypto.timingSafeEqual`). This satisfies "equal-length buffers,
   handle length mismatch without throwing" and additionally avoids leaking the
   length of the password/hmac. Used for both the password check and the cookie
   hmac check.
2. **Cookie binds `exp` to the configured cutoff** (`exp === DEMO_AUTH_EXPIRES_AT`
   in `cookieValid`). Combined with `now < exp` this enforces the HARD window even
   if the operator shortens the cutoff after cookies were issued — old cookies stop
   validating immediately.
3. **`DEMO_AUTH_EXPIRES_AT` parsed with a strict `^\d+$` + positive-integer check**
   so floats, negatives, `0`, and non-numeric values disable the gate (fail-closed)
   rather than producing a bogus cutoff.
4. **Gate is purely additive.** No auth check was added to `/api/chat` or
   `/api/health` (nginx gates `/api/chat`). `express.urlencoded({extended:false})`
   was added only for the login form; `express.json()` is untouched.
5. **`app.listen` guarded by `require.main === module`** and the gate helpers +
   `app` are exported, so the test suite can drive real HTTP requests and unit-test
   the helpers without spawning the production listener. No behavior change when run
   normally (`node server.js`).
6. **Zero new npm dependencies.** Only `node:crypto` (built-in) was added.
   `package-lock.json` was intentionally left untouched (reverted the unrelated
   stale `@google/generative-ai` prune that `npm install` produced).
