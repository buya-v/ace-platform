## Summary

Implemented market-data-service as a zero-dependency Go module at `src/market-data-service/`. The service provides OHLCV candle aggregation across 6 intervals (1m, 5m, 15m, 1h, 4h, 1d), ticker/summary computation with 24h rolling stats, trade tape storage, real-time streaming via pub/sub hub, TimescaleDB DDL generation for continuous aggregates, and data retention policy enforcement. All 35 tests pass with 68.7% coverage on the candle builder (core business logic).

## Key Decisions

1. **Zero-dependency Go module** — Followed the established pattern from matching-engine, clearing-engine etc. Duplicated Decimal type locally with added `Add()` and `DivInt()` methods needed for turnover/VWAP computation.
2. **In-memory candle builder with callback pattern** — Builder emits candle updates via a callback function, which the server wires to both the streaming hub (for real-time subscribers) and the candle store (for closed candles). This keeps the builder testable without depending on server infrastructure.
3. **Hierarchical bucket computation** — `BucketStart()` computes bucket boundaries for all 6 intervals using UTC-aligned truncation. 4h buckets align to midnight (0, 4, 8, 12, 16, 20).
4. **Streaming hub with non-blocking publish** — Slow subscribers get messages dropped rather than blocking the ingestion pipeline. Buffer size is 64 per subscription.
5. **DDL generation as Go code** — TimescaleDB continuous aggregate DDL is generated programmatically in `internal/ddl/`, matching the V8 migration from T035 but available for runtime use.
6. **Ports 50057/8087** — Following the established port allocation convention.
7. **Separate candle store and trade store** — CandleStore holds closed (historical) candles; the builder holds in-progress candles. Both are in-memory for now (production would use TimescaleDB).
