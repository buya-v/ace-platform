## Summary
Created `scripts/demo.sh`, an executable bash script that automates the full ACE Platform demo flow (registration, authentication, KYC, trading, post-trade verification, market data) against a running gateway. The script uses curl for all HTTP calls, prints colored PASS/FAIL/SKIP output per step, and exits 0 only if all checks pass. It includes a test suite (`tests/demo/demo_test.sh`) with 28 tests covering CLI flags, unreachable gateway, healthy mock flow, unhealthy gateway, and file properties.

## Key Decisions
1. **Used `/api/v1/` prefix** — read actual gateway routes from `src/gateway/internal/handler/routes.go` instead of guessing `/v1/` paths. All endpoints match the codebase exactly.
2. **Dual JSON field name handling** — auth login response may return `access_token` or `AccessToken` (PascalCase from Go structs without JSON tags). Script tries both, matching the e2e test pattern.
3. **jq optional with grep/sed fallback** — JSON parsing uses jq when available, falls back to grep/sed for zero-dependency operation.
4. **Service health checks SKIP, not FAIL** — individual backend service health checks on ports 8081-8088 gracefully skip when services aren't reachable, since the demo may run with only the gateway available.
5. **No warehouse receipt or settlement cycle POST endpoints** — these don't exist in gateway routes, so they're omitted rather than faking a SKIP for non-existent endpoints.
6. **Mock-server-based testing** — tests use a Python HTTP mock server to validate all script paths without requiring a running platform.
