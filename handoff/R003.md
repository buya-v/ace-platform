# R003 — Migrate Engines to Shared Decimal — Handoff

**Status:** done
**Services migrated (6):** matching-engine, clearing-engine, margin-engine, settlement-engine, market-data-service, warehouse-service

## What changed
For each engine:
- Deleted the local `internal/types/decimal.go` (and per-engine `decimal_test.go`).
- Replaced it with a 1-import **alias shim**: `type Decimal = decimal.Decimal` plus
  re-exported constructors (`NewDecimal`, `DecimalFromInt`, `DecimalZero`,
  `DecimalFromRaw`, `ParseDecimal`, `MustParseDecimal`). All `types.Decimal` /
  `types.NewDecimal(...)` call sites (334 NewDecimal, 240 DecimalFromInt, etc.)
  compile unchanged.
- Wired the dependency in `go.mod`:
  `require github.com/garudax-platform/decimal v0.0.0` +
  `replace github.com/garudax-platform/decimal => ../shared/pkg/types/decimal`
  (filesystem replace → no go.sum/network needed).

The six divergent copies (different APIs, silent overflow in margin's `MulDecimal`,
truncating `DivInt`/`DivInt64`, zero-on-div-by-zero) are gone. All engines now share
one checked, half-even-rounding, error-on-div-zero implementation.

## One behavioural-delta fix
`clearing-engine/internal/novation/novation_coverage_test.go` had
`TestDecimalParseTruncation` asserting the OLD truncation bug
(`1.123456789` → `1.1234`). Renamed to `TestDecimalParseRounding` and updated to the
corrected half-even result (`1.1235`). This was the only test that relied on the old
truncation behaviour — confirming the migration's behavioural surface is small.
