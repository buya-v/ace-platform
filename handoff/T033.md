## Summary

Designed the API gateway service that provides REST-to-gRPC translation for all 6 exchange platform backend services. Produced a comprehensive architecture spec (`docs/T033_gateway_spec.md`) covering 52 endpoints across orders, market data, admin, clearing, margin, settlement, auth, and compliance. Also produced a complete OpenAPI 3.0 specification (`docs/T033_openapi.yaml`) with all request/response schemas derived from the actual protobuf definitions and Go type definitions in the codebase.

## Key Decisions

1. **URL-path versioning (`/api/v1/`)** rather than header-based — simpler for clients, easier to route at the load balancer level, and aligns with industry standard for financial APIs.
2. **Public market data endpoints (no auth)** — order book and trade data are public per standard exchange practice. Only order submission, positions, and admin endpoints require authentication.
3. **Token bucket rate limiting in Redis** with per-user and per-endpoint-group limits — 50 req/s for order submission matches the matching engine's per-participant rate limit (T007 spec section 4.4).
4. **Cursor-based pagination** instead of offset-based — better performance at scale and no issues with data changes between pages.
5. **Enums as lowercase strings in JSON** (e.g., `"buy"` not `1`) — better developer experience for REST clients while proto integers are used on the gRPC side.
6. **WebSocket for streaming** (trades, order book, executions) rather than SSE — bidirectional capability needed for future features (e.g., client sending subscription updates), and better browser/library support.
7. **Circuit breaker per backend** (3 failures → 30s open) — prevents cascade failures when a backend is down without blocking other backend services' endpoints.
8. **Gateway on port 8080, health on 8090** — follows the established port convention from CLAUDE.md learned patterns (gateway=8080 HTTP).
9. **Fail-open on Redis unavailability** — rate limiting is a safety feature but must not be a single point of failure; log and allow requests when Redis is down.
