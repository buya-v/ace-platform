## Failures (details)

### 1. TestGatewayHealth/unknown_endpoint_returns_404
```
expected status 404, got 401; body: {"error":{"code":"UNAUTHENTICATED","message":"Missing authorization token"}}
```
**Root cause:** Gateway auth middleware runs before the router. Unknown paths hit auth check and return 401 before the router can return 404. Known issue — see learned pattern "Gateway auth middleware runs before routing."

### 2. TestComplianceKYCFlow/submit_KYC_application
```
json: cannot unmarshal array into Go value of type map[string]interface {}; raw: []
```
**Root cause:** Compliance service KYC submission endpoint returns a JSON array `[]` but the e2e test expects a JSON object. API contract mismatch between spec and implementation.

### 3. TestFullTradingLifecycle — 4 subtest failures
- `verify_clearing_positions`: response is array, expected object
- `verify_netting`: response is array, expected object
- `check_margin_calls`: response is array, expected object
- `check_settlement_cycles`: response is array, expected object

**Root cause:** Cross-service endpoints (clearing, netting, margin, settlement) return arrays where e2e tests expect objects. Additionally, channel-based Kafka stubs don't propagate events across separate processes, so cross-service data flow doesn't work in e2e mode.
