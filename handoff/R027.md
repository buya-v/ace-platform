  correlation ID with strict `==`, but the settlement engine carries the deterministic
  idempotency key `cycle-<tradeID>` as the `settlement.completed` correlation ID (R024

  propagation, not the exact correlation encoding. Root cause was proven on the broker

  is untouched. The repo `docker-compose.yml` is unchanged.

  R016 baseline / `run-20260626-033236`). R027's purpose is the live deploy tier.
