# Handoff ‚Äî T041: Fix PostMortem Context Size Bug

**Status:** DONE
**Agent:** Coder
**Phase:** 4 (bug fix)

## Summary
Fixed `build_postmortem_context` in `pipeline/lib/context.sh` to write assembled context to a temp file instead of returning it as a shell variable. The caller in `pipeline/run.sh` now pipes the temp file via stdin to `claude` instead of passing it as a `-p` argument, avoiding the Linux ARG_MAX (~2MB) limit that caused E2BIG errors after Phase 3. Also added task ID filtering so only handoff files from the current run are included, keeping context focused.

## Key Decisions
1. **Temp file approach (Option 1 from task description):** `build_postmortem_context` now returns a file path instead of content. This is the minimal change that fixes the E2BIG error while preserving the existing prompt assembly logic.
2. **Added `_collect_handoffs` helper with optional filtering:** When task IDs are passed, only those tasks' handoff and review files (plus integration reports) are collected. When no IDs are passed, all files are collected (backward-compatible).
3. **Caller passes filtered task IDs:** `run_postmortem` in `run.sh` extracts non-pending task IDs from `tasks.json` and passes them to `build_postmortem_context`, so only relevant handoffs are included.
4. **Trap-based cleanup:** The temp file is cleaned up both explicitly after use and via a trap on EXIT as a safety net.

## Blockers Found
- The `pipeline/` directory was never committed to git (exists only as untracked files in the main repo). Had to copy the files into the worktree to make the fix. The pipeline directory should be committed to git in a follow-up.

## Suggested Follow-ups
- **Commit the pipeline directory to git.** Currently it's untracked in the main repo, which means worktrees and new clones don't have it.
- Consider applying the same temp-file pattern to other large context builders (`build_worker_context`, `build_reviewer_context`) proactively, before they hit similar limits.
- The `_claude` wrapper's `-p` flag is used by all agent runners ‚Äî if any other context grows large, the same E2BIG bug will recur.
