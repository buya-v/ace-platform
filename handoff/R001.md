# R001 — Shared Decimal Type Spec — Handoff

**Status:** done
**Deliverable:** `docs/specs/R001_shared_decimal_spec.md`

## Summary
Specified one shared fixed-point money type to replace the six divergent `Decimal`
copies and the `securities-service` `float64` math. Derived the API as a strict
**superset** of all six existing copies (union of: matching, clearing, margin,
settlement, market-data, warehouse) so migration is largely an import swap.

## Decisions
- Representation unchanged: `int64` scaled by `10000` (Decimal(18,4)) → wire/DB
  `*_raw` columns and protobufs are compatible via `Raw()`/`DecimalFromRaw()`.
- Three correctness rules made non-negotiable: no silent overflow (128-bit check),
  banker's (half-to-even) rounding, no silent divide-by-zero.
- Two API styles: `Try*` (returns error) for money boundaries; convenience methods
  (old names/signatures) that **panic** instead of silently corrupting — keeps R003
  a drop-in rename.
