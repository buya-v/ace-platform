## Summary

Created 5 Flyway migrations (V1–V5) covering 8 schemas: reference, participants, exchange,
clearing, compliance, warehouse, auth, market_data. Includes seed data for commodities,
delivery locations, and trading calendar.

## Key Decisions

1. **Append-only trades** — DELETE/UPDATE blocked by Postgres RULES on `exchange.trades`
2. **SHA-256 audit hash chain** — `compliance.audit_events` is tamper-evident; each row hashes previous
3. **eWR integrity** — `warehouse.warehouse_receipts` has `hash_chain` for receipt chain-of-custody
4. **`remaining_quantity` is a generated column** — computed as `quantity - filled_quantity`, cannot desync
5. **TimescaleDB hypertables** — 1-day chunks, 90-day raw retention, 7-day compression policy
6. **Continuous aggregates** — auto-refreshed 1m/1h/1d OHLCV candles from tick data
7. **Service roles per domain** — `ace_exchange_svc`, `ace_clearing_svc`, etc. for least-privilege DB access
8. **Circuit breakers** — ±5% triggers 15-min halt, ±10% triggers 60-min halt
