## Failures (6 e2e test failures)

### 1. TestGatewayHealth/unknown_endpoint_returns_404
- **Expected:** HTTP 404 for `GET /api/v1/nonexistent`
- **Actual:** HTTP 401 with `{"error":{"code":"UNAUTHENTICATED","message":"Missing authorization token"}}`
- **Root cause:** Gateway auth middleware runs before routing. Unknown paths hit auth check before router can return 404.
- **Fix:** Reorder middleware so routing (or path existence check) occurs before auth enforcement.

### 2. TestComplianceKYCFlow/submit_KYC_application
- **Expected:** JSON object response (`map[string]interface{}`)
- **Actual:** JSON array response (`[]`)
- **Root cause:** Compliance KYC endpoint returns array format; e2e test expects object format.
- **Fix:** Either change endpoint to return object, or update e2e test to match actual API contract.

### 3-6. TestFullTradingLifecycle (4 subtests)
- **Subtests:** verify_clearing_positions, verify_netting, check_margin_calls, check_settlement_cycles
- **Expected:** JSON object responses for clearing/netting/margin/settlement queries
- **Actual:** JSON array responses (empty `[]`)
- **Root cause:** Cross-service event propagation requires Kafka. Channel-based Kafka stubs only work in-process; each service runs as a separate process in e2e mode, so events don't bridge across services.
- **Fix:** Either (1) run real Kafka for e2e tests, or (2) accept array responses in e2e tests when no cross-service events have flowed.
