## Summary

Fixed the 3 blocking bugs identified in T054-review.md: (1) duplicated the `service-discovery` ConfigMap into both `ace-exchange` and `ace-services` namespaces so engine pods can resolve it, (2) deleted the duplicate `namespace.yaml` and added `ace-infra` to the existing `namespaces.yaml`, (3) added `ace-jwt-signing-key` secret to `ace-exchange` namespace for engine JWT validation. Added 5 new cross-resource validation tests.

## Key Decisions

1. Duplicated the full ConfigMap into both namespaces rather than using a shared mechanism — keeps manifests simple and K8s-native with no external tooling required.
2. Kept `ace-exchange` as the first ConfigMap definition since engines are the primary consumers; `ace-services` is the second copy.
3. Updated existing test class `TestNamespaces` to assert `namespace.yaml` does NOT exist (regression guard against re-introducing the duplicate).
4. Added `TestCrossResourceValidation` test class with 5 tests that verify configMapRef/secretRef resolvability, JWT secret presence, and no duplicate namespace definitions.

## Blockers Found

- `kustomize` is not available in the build environment, so `kustomize build` validation was skipped. Python-based cross-resource tests provide equivalent coverage for the 3 fixed bugs.

## Suggested Follow-ups
