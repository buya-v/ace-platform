## Summary

Created complete Kubernetes manifests for all ACE platform services using Kustomize at `deploy/k8s/`. The base layer defines deployments, services, HPAs, PDBs, and service accounts for 8 gRPC services + gateway + 2 UIs + 4 infrastructure components (PostgreSQL, Redis, Kafka, Zookeeper). Three overlays (dev/staging/prod) customize replica counts, resource limits, storage sizes, and selectively enable/disable HPA and PDB.

## Key Decisions

1. **Three namespaces**: `ace-exchange` (engines), `ace-services` (supporting services + UIs), `ace-infra` (postgres/redis/kafka/zookeeper) — matches the existing namespace definitions in `namespaces.yaml`.
2. **Port assignments from source code**: Used actual port defaults from each service's `config.go` — matching=50051/8081, clearing=50052/8082, margin=50053/8083, settlement=50054/8084, auth=50055/8085, compliance=50056/8086, market-data=50057/8087, warehouse=50058/8088, gateway=8080/8090.
3. **IRSA annotations use placeholder ACCOUNT_ID**: Real AWS account IDs come from Terraform (T002); manifests use `arn:aws:iam::ACCOUNT_ID:role/ace-<service>-irsa` pattern for substitution.
4. **Engines get 2x resources in prod**: Per task spec, engine deployments get double the standard service resources (512Mi/400m requests, 2Gi/2000m limits).
5. **HPA removed in dev, PDB removed in dev+staging**: Only prod gets full HPA + PDB. Staging gets HPA but no PDB.
6. **Service discovery via ConfigMap**: Single `service-discovery` ConfigMap with full cluster DNS addresses, mounted to all services via `envFrom`.
7. **Secrets use placeholder values**: DB credentials and JWT signing keys have `CHANGE_ME_USE_EXTERNAL_SECRET_MANAGER` values — intended to be replaced by ExternalSecrets or similar operator.
8. **Kafka replication factors scaled per overlay**: Dev=1, prod=3 (with min.isr=2).
