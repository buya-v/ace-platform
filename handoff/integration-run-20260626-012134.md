## Failures

### 1. gateway — `internal/middleware` tenant enforcement (3 failures) — REAL DEFECT

```
--- FAIL: TestTenantMiddleware_ValidTenant   (expected tenant in context, got "")
--- FAIL: TestTenantMiddleware_MissingHeader  (expected 401, got 200 — handler called)
--- FAIL: TestTenantMiddleware_UnknownTenant  (expected 403, got 200 — handler called)
```

**Root cause:** `src/gateway/internal/middleware/tenant.go` defines `tenantBypassPrefixes`
that whitelists essentially every tenant-scoped API path, including `/api/v1/orders`
(line 36), `/api/v1/margin` (38), `/api/v1/participants` (41), `/api/v1/tickets` (47),
plus clearing/settlement/warehouse/market-data/etc. The tests exercise `/api/v1/orders`,
which matches a bypass prefix, so `TenantMiddleware` skips header validation and calls the
downstream handler with **no tenant in context** — returning 200 where 401/403 is expected.

**Impact:** This is not merely test drift — it defeats tenant scoping at the gateway and
directly contradicts the platform invariant ("Tenant ID is never optional" / "no service
accepts traffic with an unresolved tenant", per the 2026-04-23 multi-tenancy directive).
The bypass list appears to have been used as a routing allow-list and has grown to cover
tenant-scoped business paths. Recommend narrowing `tenantBypassPrefixes` to genuinely
platform-level paths only (`/platform/`, `/api/v1/platform/`, `/api/v1/auth/`, health),
and requiring `X-GarudaX-Tenant` on all tenant-scoped routes. **Not fixed by this agent
(report-only).**
