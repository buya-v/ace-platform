## Key Decisions

1. **Scope boundary honored strictly — tests live in `tests/`, not `src/`.**
   The task asks for tests exercising `ClearTrade`, `RegisterInstrument`, and
   handler callbacks. That code is in `internal/` packages, which cannot be
   imported from a separate `tests/` module (Go internal-package rule), and the
   scope rule forbids modifying `src/`. R008 already placed the in-package
   concurrency tests inside each engine (its handoff explicitly anticipated
   R009 would "fold these into the CI race-gate"). So R009's contribution is the
   runnable race-gate harness + the pre-fix soundness proof + docs that drive
   those tests — not duplicate test files in `src/`.

2. **Two run modes.** Focused mode runs only the named concurrency tests (fast
   developer loop). `--full` runs whole packages under `-race` (true regression
   gate — also covers margin `internal/engine` `CalculateMargin`/`SetMarginHandler`).

3. **`GOTOOLCHAIN=auto`** is exported by the scripts because the engine modules
   declare `go 1.25.x` while the host may be older; this preserves the repo's
   relative-`replace`, no-`go.sum` build model.
