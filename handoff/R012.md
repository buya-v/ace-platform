## Summary
Added a full-chain e2e test file — `tests/e2e/tenant_enforcement_test.go` — that proves
the gateway enforces the platform invariant *"Tenant ID is never optional"* at the
edge. The tests drive a **live gateway** through the exact production middleware
chain (RequestID → Tenant → Auth → RateLimit → Router) and assert:

1. **Missing `X-GarudaX-Tenant` → 401 `TENANT_REQUIRED`** on every core
   tenant-scoped route (orders, clearing, margin, settlement, warehouse,
   compliance, market-data, admin).
2. **Invalid/unknown tenant → 403 `UNKNOWN_TENANT`** on the same routes.
3. **Empty header value** is treated as missing (401 `TENANT_REQUIRED`).
4. **Valid tenant (`ace-commodities`, `mse-equities`) clears the tenant gate** — the
   request proceeds to auth/backend and is never rejected as a tenant error.
5. **Valid tenant is forwarded to the backend** — on an auth-public + tenant-scoped
   route (`/api/v1/market-data/...`) a valid tenant reaches the backend (status is a
   backend outcome, not a 401/403 edge rejection), while a missing tenant is blocked
   at the edge.
6. **Bypass whitelist still passes without a tenant** — `/healthz`, `/readyz`,
   `/metrics`, `/platform/...`, `/api/v1/platform/...`, `/api/v1/auth/login`,
   `/api/v1/auth/register` are never rejected as tenant errors; `/healthz` and
   `/readyz` return 200 even with no (or a bogus) tenant header.

The suite uses the established **graceful-skip** pattern: all tests `t.Skip` when no
gateway is reachable at `E2E_BASE_URL` (default `http://localhost:8080`), so they are
safe in CI without infrastructure and run for real under Docker Compose / a live stack.

## Key Decisions
1. **e2e module, not in-package.** Scope is `tests/` + `handoff/` only, and the tenant
   logic lives in an `internal/` package a separate module cannot import (Go internal
   rule). The right home is `tests/e2e/` (live-HTTP, graceful-skip), matching R011's
   own "R012 should add full-stack e2e assertions" follow-up.
2. **Tenant-before-auth makes the rejections deterministic.** Because `TenantMiddleware`
   runs before `Auth`, the missing/unknown-tenant assertions need no JWT and no live
   backend — the request is rejected at the edge. This keeps the assertions stable in a
   partial stack (the common CI/Compose reality).
3. **Enforcement routes are all registered.** The middleware's `RouteChecker` returns
   404 for unregistered paths *before* tenant enforcement, so every path used for the
   401/403 assertions is a real registered route (verified against
   `internal/handler/routes.go`) — otherwise a typo'd path would 404 and mask the test.
4. **"Forwarded" asserted via behaviour, not header inspection.** e2e can't read the
   `X-GarudaX-Tenant` the gateway injects into the *backend* call (that is covered by
   the R011 handler unit test). Instead R012 proves forwarding behaviourally: a valid
   tenant on a public+tenant-scoped route reaches the backend (503 backend-down), a
   missing one is blocked at the edge (401).
5. **No shared-client changes.** Added a self-contained `tenantRequest` helper that
   controls the tenant header (omit / set / bogus) and sends no `Authorization`,
   leaving the existing `apiClient` (which always sends `ace-commodities`) untouched.

## Suggested Follow-ups
- **Wire this into the R014 CI gate.** Run `tests/e2e` with `E2E_BASE_URL` pointed at the
  Compose gateway so tenant enforcement is checked on every build (it currently only
  runs for real when a stack is up). A green unit suite with these e2e tests skipping
  should be reported `PARTIAL`, not `PASS`, per the e2e-canary pattern.
- **Backend-side enforcement (R011 follow-up still open).** These tests prove the gateway
  *forwards* the validated tenant; they do not prove backends *enforce* it. Add gRPC/HTTP
  interceptor tests on the backend services that the propagated `X-GarudaX-Tenant` is
  required and scoped (R011's "wire the shared gRPC interceptors onto backend servers").
- **WebSocket tenant resolution.** `/api/v1/ws/` is tenant-enforced but browsers can't set
  the header on the WS handshake (R011 follow-up). When a query-param/subprotocol resolver
  lands, add an e2e case for it. (Not tested here to avoid asserting a known gap.)
- **Demo-runner tenant header.** `/api/v1/admin/demo/` and `/api/v1/securities/demo/` now
  require a tenant; if the demo-runner is updated to send one, add a bypass/enforce case.
