# T043 — Clearing Engine Test Coverage Improvement

## Summary

Added comprehensive tests across the clearing-engine's business-critical packages,
raising coverage from well below the 60% floor to 95–100% on all target packages.

## Files Added

- `internal/novation/novation_coverage_test.go` — table-driven validation, multi-trade novation, value calculation, decimal price, timestamp checks, plus types coverage (Side, ClearingStatus, Position predicates, Decimal methods)
- `internal/netting/netting_coverage_test.go` — bilateral netting, multilateral (3-party ring), mixed buy/sell, single sell position, multi-instrument same participant, value accumulation, large obligation set, timestamp verification
- `internal/engine/engine_coverage_test.go` — ClearTrade error paths (novation failure, store buyer/seller failure, idempotency), GetPositions, GetPositionsByInstrument, end-to-end pipeline, handler verification, concurrent clearing with position verification
- `internal/types/clearing_test.go` — Side.String/Opposite, ClearingStatus.String, Position.IsLong/IsShort/IsFlat, NettingEfficiency, abs64
- `internal/types/decimal_test.go` — NewDecimal, ParseDecimal (valid + errors + truncation), IsZero, IsNeg, Negate, Add, Sub, MulUint64, Abs, Cmp, LessThan, GreaterThan, Equal, String (positive, negative, fraction, zero)

## Coverage Results (before → after)

Total tests: 116 (up from ~60)

## Engine 95% Gap

The remaining 5% in engine is the `ClearTrade` error path for position manager failures (`Apply` returning error on buyer/seller position update). These error paths require the position manager to fail, but since it's not injected as an interface (it's constructed internally), this would require source code changes to make it mockable. Not worth the coupling for 5%.

## Suggested Follow-ups

- Consider extracting `position.Manager` as an interface in the engine for full testability
- `internal/server/` has 0% coverage (not in scope — infrastructure package)
- `cmd/clearing-engine/` has 0% coverage (main entry point — would need integration test)
